# Frontend QA: Unstarted-course click behaviour — dedicated course detail page

Manual browser QA for the new canonical **course detail page** (`courses/<slug>/detail/`), the
not-registered card/row link behaviour (no more modal), the three new `Course` fields and their graceful
omission, and the `c-modal` regression check.

---

## 0. Start the dev server

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

If the styles look unstyled, build Tailwind first: `npm run tailwind_build`.

---

## 2. Click routing — no modal anywhere (success criterion 1)

### 2a. From the dashboard card

Go to `http://127.0.0.1:$PORT/` (dashboard).

- Find a **not-registered** course card (under "Recommended" or "Available courses").
- **Desktop (window ≥ md / ~≥768px):** click the card. **Expect:** it **navigates** to
  `…/courses/<slug>/detail/` — a full page. **No modal opens** (this is the behaviour change).
- **Mobile (window < md):** narrow the window (~375px), reload, click the card → same navigation to
  `…/courses/<slug>/detail/`.
- Keyboard: `Tab` to the card title link, press `Enter` → same navigation, focus ring visible.

### 2b. From the all-courses row

Go to `http://127.0.0.1:$PORT/courses/`.

- Find the **not-registered** row. Click it on **both** desktop and mobile widths.
- **Expect:** navigation to `…/courses/<slug>/detail/` every time. **No modal** on desktop.

> The old `…/preview/` URL no longer exists. Visiting it directly should 404 — the canonical path is now
> `…/detail/`.

## 3. Detail page content (success criterion 2) — Course WITH-FIELDS

Open the **WITH-FIELDS** course detail page (click it, or go to `…/courses/<with-fields-slug>/detail/`).

**Expect, top to bottom:**

- **Breadcrumbs**: first crumb "← All courses" links back to `/courses/`; last crumb is the course title
  (not a link).
- **Hero**: accent-coloured background; a large, **semi-transparent** course icon anchored bottom-right
  (it must not compete with the title); the **title**; the **subtitle** (if the course has one).
- **Stats strip** showing real values:
  - a lessons/modules count (a real number from the course structure);
  - the **difficulty** (e.g. "Beginner");
  - the **estimated duration** (e.g. "~1 hour 30 min");
  - an honest **"Free · open enrolment"** label.
  - **No** star rating, enrolment count, "self-paced", or "Certificate on completion".
- **About this course**: the course's rendered markdown (if it has content).
- **What you'll learn**: a tick-marked list, one item per `learning_outcomes` entry, in order.
- **Course content**: the table-of-contents list (reused ToC) showing the course structure — structure
  only, **no** ability to open/preview item content before enrolling.
- **Sign-up panel** (right column on desktop, near the top on mobile):
  - an honest "Free / open to everyone" line + microcopy (e.g. "One click. No credit card.");
  - a single high-contrast **primary CTA** reading **"Enrol & start"** (not just "Start");
  - a **"This course includes"** list: the lessons/modules count, and **"Includes assessments"** only if
    the course actually has a form/quiz. No rating / wishlist / certificate.

On **desktop**, scroll the page: the sign-up panel should remain **sticky** (stays in view) while the main
column scrolls. On **mobile**, the layout is a single column (hero → sign-up panel/CTA → main content) and
the CTA is reachable without scrolling to the very bottom.

## 4. Honesty / graceful omission (success criterion 3) — Course NO-FIELDS

Open the **NO-FIELDS** course detail page (`…/courses/<no-fields-slug>/detail/`).

**Expect:**

- **No** difficulty stat, **no** duration stat, **no** "What you'll learn" section at all.
- **No** placeholders anywhere — no `—`, "N/A", "TBD", "Beginner" default, or empty bullet.
- The lessons/modules count and "Free · open enrolment" still show (always-available data).
- The page still renders cleanly (hero, about if present, ToC, sign-up panel with "Enrol & start").

## 5. Enrolment flow (success criterion 4)

On the **WITH-FIELDS** detail page (as the not-registered learner):

- Click **"Enrol & start"**.
- **Expect:** the one-click enrol runs (`register_for_course`) and you land **inside the course** (course
  home / first item) — not back on a modal or the detail page.
- Go back to `/courses/` — that course is no longer "not registered" (it now shows a registered status and
  no longer routes to the detail page as an unstarted course).

## 6. `c-modal` regression (success criterion 6)

The modal component must still work for educator flows (the cleanup removed only the course-preview modal
usage, not `c-modal` itself).

- Log in as the **educator**.
- Trigger a flow that uses the delete-confirmation or modal-form (e.g. delete a cohort/object, or an
  edit-in-modal action in the educator interface).
- **Expect:** the modal **opens, functions, and closes** normally. Nothing is broken by the cleanup.
