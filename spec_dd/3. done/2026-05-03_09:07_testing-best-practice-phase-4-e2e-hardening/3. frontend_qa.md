# Frontend QA — Phase 4: E2E hardening

The implementation is mostly test-infrastructure work, but it includes one production template change: `course_minimal_toc.html` line 109 swaps the inline `data-storage-key="coursePart_{{ course.slug }}_{{ forloop.counter }}"` literal for a template-tag call that produces the same string. If the tag is wrong by even one character — different prefix, off-by-one counter, missing slug interpolation — the Alpine `coursePart` component will still render but state will no longer persist (the read key won't match the write key). A user would experience this as "the course-part collapses every time I navigate".

This QA exists to catch that regression in the browser, where the autouse localStorage-clearing fixture from the test suite isn't in play.

## Setup

Use the **DemoDev** site for QA test data (per project standing memory).

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

Log in as a student account that's enrolled in the `functionality_demo_course_parts` demo course on DemoDev. (That course has nested course-parts with children — the exact shape the `coursePart` Alpine component is designed for.) If no such enrolment exists, create one via the admin or through the student-management UI before starting the QA.

## Test 1 — Course-part expand/collapse on a topic detail page

**Goal:** Confirm the rewritten template tag still produces a working `data-storage-key` and the Alpine component reads it correctly.

1. Navigate to `http://127.0.0.1:$PORT/student/courses/functionality-demo-course-parts/1/` (the first item inside the course; the URL pattern is `student_interface:view_course_item`, kwargs `course_slug` and `index`).
2. In the table-of-contents sidebar, find a course-part heading (e.g. "Core Concepts" or whatever the demo course exposes). It should appear with a collapsed-state toggle by default.
3. Click the course-part toggle.

**Expected:** the toggle expands. The course-part's child topics appear inline under it. No console errors.

4. Open browser devtools → Application → Local Storage → site origin. You should see a key matching the pattern `coursePart_functionality-demo-course-parts_<N>` (where `<N>` is the course-part's position in the course).

**Expected:** the key exists, value is `"true"`. If the key is missing, or has a different prefix, the template-tag extraction is broken — fail this test and inspect `freedom_ls/student_interface/templatetags/course_storage_keys.py` and `course_minimal_toc.html`.

5. Click the toggle again.

**Expected:** it collapses. Children disappear. The same localStorage key now reads `"false"`.

## Test 2 — Persistence across navigation

**Goal:** Confirm state written by one page is readable by another (same key on both ends).

1. Continuing from Test 1 in an expanded state (key value `"true"`), navigate to the course home page: `http://127.0.0.1:$PORT/student/courses/functionality-demo-course-parts/`.
2. Locate the same course-part in the TOC there.

**Expected:** it is *already expanded* on arrival — its child topics are visible without you clicking anything. This proves that the storage key written by the topic page is read identically by the course-home page (i.e. the template tag is consistent across both render contexts).

3. Click the toggle to collapse.
4. Navigate away to any other page (the dashboard, another course) and back to the course home.

**Expected:** the course-part is collapsed on arrival.

## Test 3 — Smoke check, other interactive components

**Goal:** Confirm the template change didn't break the surrounding TOC partial.

1. From the course home, click directly into a child topic of an expanded course-part.

**Expected:** topic loads, title and content render, no console errors.

2. Click the back-link or the parent course-part link from the breadcrumbs/TOC.

**Expected:** navigation works; no broken markup.

## What to do if something fails

- **Key value is `"undefined"` or `"null"`:** the template tag is rendering the wrong value (probably missing argument). Check `course_minimal_toc.html` line 109 wiring.
- **Key prefix is wrong (e.g. `course_part_...` instead of `coursePart_...`):** the constant in `course_storage_keys.py` was renamed. Restore the original `coursePart` prefix — it's load-bearing for any user with existing localStorage state.
- **Component throws a JS error in console:** inspect the rendered HTML; the `data-storage-key` attribute is probably missing entirely. The Alpine component does `this.$el.dataset.storageKey || null` so it shouldn't crash, but a missing attribute will silently disable persistence.
- **Test 1 passes but Test 2 fails:** the two render paths (course-home vs. topic-detail) are using the partial differently. `forloop.counter` may not be the same value in both contexts — verify both templates pass the same iteration index into the partial.

Open a bug ticket and file-and-skip the related test per the spec's standing rule. Do not patch over symptoms.
