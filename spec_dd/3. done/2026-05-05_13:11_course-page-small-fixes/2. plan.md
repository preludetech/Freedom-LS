# Implementation plan: Course page small fixes

Two presentational template tweaks, bundled together because they share the same area of the codebase. No model, view, URL, migration, or behaviour changes.

## Summary of changes

- `freedom_ls/student_interface/templates/student_interface/course_topic.html` — fix Next/Previous icon sides on the topic-page navigation buttons.
- `freedom_ls/student_interface/templates/student_interface/course_finish.html` — remove the secondary "View Course" CTA so "Return to Home" is the only CTA.

Both files are the *only* files in scope.

## Existing functionality being reused

- **`<c-button>`** at `freedom_ls/base/templates/cotton/button.html` already supports `icon_left` and `icon_right` attributes. It renders `mr-2`/`ml-2` spacing automatically. Do not add new component code or new icon nesting — switch to the attribute-based API.
- **Semantic icons** `next` (→ `arrow-right`) and `previous` (→ `arrow-left`) are already registered at `freedom_ls/icons/semantic_names.py` and `freedom_ls/icons/mappings.py`. No new icons are needed.
- The codebase convention (per the cotton component's own docstring and most existing call-sites) is **underscored** attributes: `icon_left="..."` / `icon_right="..."`. The two existing kebab-case usages in `course_topic.html` (`icon-left="..."`) work because cotton normalizes hyphens, but they are inconsistent with the rest of the file (`base/templates/cotton/button.html` examples, `content_engine/templates/cotton/file-download.html`). Normalise to underscored form while we are in there.

## Task 1 — Fix Next / Previous icon position on the topic page

**File:** `freedom_ls/student_interface/templates/student_interface/course_topic.html`

### 1a. Previous button (currently lines 25–30)

Replace the manually-nested `<c-icon>` child with the `icon_left` attribute:

```django
{# before #}
<c-button href="{{ previous_url }}" variant="outline">
    <c-icon name="previous" class="size-5" />
    Previous
</c-button>

{# after #}
<c-button href="{{ previous_url }}" variant="outline" icon_left="previous">
    Previous
</c-button>
```

### 1b. Submit button inside the `mark_complete` form (currently line 36)

Split the icon attribute by branch so the directional case gets a right-side arrow while the terminal "Finish Course" case keeps its left-side check:

```django
{# before #}
<c-button type="submit" name="mark_complete" icon-left="check">
    {% if is_last_item %}
        Finish Course
    {% else %}
        Next
    {% endif %}
</c-button>

{# after #}
{% if is_last_item %}
    <c-button type="submit" name="mark_complete" icon_left="check">
        Finish Course
    </c-button>
{% else %}
    <c-button type="submit" name="mark_complete" icon_right="next">
        Next
    </c-button>
{% endif %}
```

Splitting the buttons rather than building a single button with conditional attributes keeps the template readable and matches how the surrounding `{% elif %}` branches already structure their buttons.

### 1c. Next link button (currently line 45)

Move the icon to the right and switch to underscored form:

```django
{# before #}
<c-button href="{{ next_url }}" icon-left="next">
    Next
</c-button>

{# after #}
<c-button href="{{ next_url }}" icon_right="next">
    Next
</c-button>
```

### 1d. Normalise the remaining `icon-left` attributes in this file

The two terminal-action buttons in the `is_last_item` fallback branch (lines 50–54 "Home" and 56–58 "Finish Course") keep their *icon side* as-is (left, non-directional), but rename `icon-left` → `icon_left` for consistency with the rest of the file. This is the only churn outside the directional-icon fix.

### Out of scope (do not touch)

- `freedom_ls/student_interface/templates/student_interface/course_form_page.html` — form-page Back/Next is a separate UX surface; spec explicitly excludes it.
- `<c-button>` cotton component itself — no changes.
- The icon registry — no new semantic names.

## Task 2 — Remove the "View Course" button on the finish page

**File:** `freedom_ls/student_interface/templates/student_interface/course_finish.html`

Delete the second `<c-button>` block (currently lines 48–50). Keep:

- The wrapping `<div class="mt-8 flex justify-center gap-4">` — `flex justify-center` still centres a single child correctly. The `gap-4` becomes a no-op with one child, which is harmless.
- The "Return to Home" button as the sole CTA.

While in there, switch the remaining "Return to Home" button from the manually-nested `<c-icon>` child to `icon_left="home"` for consistency with Task 1's normalisation:

```django
{# before #}
<div class="mt-8 flex justify-center gap-4">
    <c-button href="{% url 'student_interface:home' %}">
        <c-icon name="home" class="size-5 mr-2" />Return to Home
    </c-button>
    <c-button href="{% url 'student_interface:course_home' course_slug=course.slug %}">
        <c-icon name="topic" class="size-5 mr-2" />View Course
    </c-button>
</div>

{# after #}
<div class="mt-8 flex justify-center gap-4">
    <c-button href="{% url 'student_interface:home' %}" icon_left="home">
        Return to Home
    </c-button>
</div>
```

This minor normalisation is in scope of the spec's "Decisions and rationale": *"Use the existing `icon_left` / `icon_right` attributes on `c-button` rather than nesting `<c-icon>` manually."*
