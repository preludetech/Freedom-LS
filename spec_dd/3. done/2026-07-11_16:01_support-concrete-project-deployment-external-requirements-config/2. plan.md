# Implementation Plan: External requirements config (PostHog / Sentry / R2)

Implements `1. spec.md`. Wires PostHog, Sentry, and Cloudflare R2 configuration into FLS for
concrete deployments via a new `freedom_ls/deployment` app, following TDD (red → green → refactor,
one test at a time — do **not** pre-write the whole suite).

## Existing code to reuse (keep it DRY)

- **AppSettings machinery** — `freedom_ls/base/app_settings.py` provides `AppSettings`, `Setting`,
  `required_settings_errors()`. `deployment/config.py` subclasses `AppSettings`; do **not**
  re-implement resolution. It is already unit-tested in `base`, so `config.py` needs no tests for
  the resolution mechanism itself.
- **`config.py` examples** — `freedom_ls/webhooks/config.py` (optional-with-default),
  `freedom_ls/course_access/config.py` (required). Follow the class-annotation + `declared_settings`
  + module-level `config = …()` shape exactly.
- **AppConfig examples** — `freedom_ls/icons/apps.py` (no `default_auto_field`, has `ready()`),
  `freedom_ls/content_engine/apps.py`. `deployment` has no models, so mirror `icons` (omit
  `default_auto_field`).
- **Context-processor siblings** — `freedom_ls/base/context_processors.py` already holds
  `debug_branch_info` and the current `posthog_config`. Refactor in place; don't relocate.
- **URL convention** — `freedom_ls/educator_interface/urls.py` (has `app_name`), plus `config/urls.py`
  top-level includes (`health/`, `educator/`, …). `django.contrib.admin` is installed, so
  `django.contrib.admin.views.decorators.staff_member_required` is available with no new dependency.
- **Storage backend** — `django-storages[s3]>=1.14.6` (with transitive `boto3`) is already in
  `pyproject.toml`; R2 is S3-API-compatible and already uses `storages.backends.s3.S3Storage`. No new
  storage dependency.
- **Factories** — `freedom_ls/accounts/factories.py::UserFactory` has `staff` / `superuser` traits;
  use them for the `sentry-debug/` gating tests. Never `.objects.create()`.

## Where env vars are surfaced onto Django settings (design refinement)

`AppSettings` resolves `config.NAME` via `getattr(django.conf.settings, NAME, None)`, so each declared
setting must be assigned onto Django settings from its env var somewhere.

- **PostHog + Sentry env vars are read in `config/settings_base.py`** (shared by dev and prod), *not*
  `settings_prod.py`. Their consumers — the context processor (every render) and
  `DeploymentAppConfig.ready()` (every boot) — run in all environments and resolve through `config`.
  Reading them in `settings_base` keeps "unset ⇒ feature disabled" uniform, and lets a developer set
  `POSTHOG_API_KEY` / `SENTRY_DSN` locally to exercise the feature. (The spec's "indicative affected
  files" listed `settings_prod`; the spec's resolution note explicitly allows either — `settings_base`
  is chosen for uniform resolution.)
- **R2 stays entirely in `config/settings_prod.py`** — media object storage is a prod-only concern
  (dev uses `FileSystemStorage`), read at settings-load time, and is *not* routed through `config`.

## Task 3 — Surface PostHog + Sentry env vars; register the context processor

**Modify** `config/settings_base.py`:

1. Read the env vars onto Django settings (near the existing settings, following the file's
   `os.environ.get(...)` style). **Coerce the typed Sentry values here, at the surfacing point** —
   not in `init_sentry()` — so `config.SENTRY_TRACES_SAMPLE_RATE`/`SENTRY_SEND_DEFAULT_PII` actually
   return the `float`/`bool` their `config.py` annotations promise (`AppSettings` returns the settings
   value verbatim; it does not coerce). This matches the existing precedent in `settings_prod.py`
   (HSTS: `... == "True"`) and `accounts` (bool settings surfaced as real bools). String settings stay
   raw; unset numeric ⇒ `None` so `config` falls through to its declared default:

```python
# PostHog / Sentry (resolved through freedom_ls.deployment.config)
POSTHOG_API_KEY = os.environ.get("POSTHOG_API_KEY")
POSTHOG_API_HOST = os.environ.get("POSTHOG_API_HOST")
POSTHOG_UI_HOST = os.environ.get("POSTHOG_UI_HOST")
SENTRY_DSN = os.environ.get("SENTRY_DSN")
SENTRY_ENVIRONMENT = os.environ.get("SENTRY_ENVIRONMENT")
SENTRY_RELEASE = os.environ.get("SENTRY_RELEASE")
_sentry_traces = os.environ.get("SENTRY_TRACES_SAMPLE_RATE")
SENTRY_TRACES_SAMPLE_RATE = float(_sentry_traces) if _sentry_traces else None
SENTRY_SEND_DEFAULT_PII = os.environ.get("SENTRY_SEND_DEFAULT_PII", "").strip().lower() == "true"
```
   - `SENTRY_TRACES_SAMPLE_RATE`: unset ⇒ `None` ⇒ `config` returns the `0.1` float default; set ⇒
     parsed to `float` (a non-numeric value fails fast at boot, which is acceptable).
   - `SENTRY_SEND_DEFAULT_PII`: coerced to a real `bool` with an **allowlist** (`== "true"`,
     case-insensitive) so anything but an explicit truthy token — including `"false"`/`"0"`/typos —
     is `False`. This is the safe default for a PII-sending flag (avoids the `bool("false") is True`
     footgun). (Confirm `import os` is present at the top of `settings_base.py`.)

2. Register the context processor in the `TEMPLATES[0]["OPTIONS"]["context_processors"]` list
   (`config/settings_base.py:177-184`), appending
   `"freedom_ls.base.context_processors.posthog_config"`.

### 5b. `init_sentry()` helper
**File (new):** `freedom_ls/deployment/sentry.py`:

```python
from __future__ import annotations

import sentry_sdk

from freedom_ls.deployment.config import config


def init_sentry() -> None:
    dsn = config.SENTRY_DSN
    if not dsn:
        return  # no DSN ⇒ Sentry disabled (dev / unconfigured deploys)
    sentry_sdk.init(
        dsn=dsn,
        environment=config.SENTRY_ENVIRONMENT,          # sourced from config for discoverability
        release=config.SENTRY_RELEASE,                   # None ⇒ no release tag
        traces_sample_rate=config.SENTRY_TRACES_SAMPLE_RATE,
        send_default_pii=config.SENTRY_SEND_DEFAULT_PII,
    )
```
- **No coercion needed here** — Task 3 surfaces `SENTRY_TRACES_SAMPLE_RATE`/`SENTRY_SEND_DEFAULT_PII`
  onto Django settings already typed (`float`/`bool`), so `config` returns real typed values. Read
  them straight through.
- Note on `environment`: passing `config.SENTRY_ENVIRONMENT` (which is `None` when unset) is
  functionally the same as omitting the kwarg — it does **not** prevent the SDK's silent
  `"production"` fallback. The real mitigation is the operator setting `SENTRY_ENVIRONMENT` in the
  deploy env; Task 7 documents that in `.env.example`. Keep the comment accurate (don't claim this
  line defeats the footgun).
- `DeploymentAppConfig.ready()` (Task 1) calls `init_sentry()`.

## Task 6 — Cloudflare R2 correctness in `settings_prod.py`

The current block (`config/settings_prod.py`, "Media Storage" section) reads env vars and builds
`default_storage` inline, passing `default_acl` and lacking the checksum workaround. Extract the
backend-options construction into a **pure, testable helper** so the R2 branch logic (private vs
public serving, region default) can be unit-tested with explicit inputs rather than pinning live
config.

### 6a. Storage helper
**File (new):** `freedom_ls/deployment/storage.py`:

```python
from __future__ import annotations

from botocore.config import Config


def build_s3_media_storage(
    *,
    bucket_name: str,
    access_key: str | None,
    secret_key: str | None,
    endpoint_url: str | None,
    region_name: str | None,
    custom_domain: str | None,
    querystring_auth: bool,
    querystring_expire: int,
) -> dict[str, object]:
    """Assemble the STORAGES['default'] entry for an R2 (S3-compatible) bucket.

    R2 landmines handled here: no ACLs (R2 has none), the boto3 >=1.35.99 checksum
    headers R2 rejects, and region defaulting to "auto".
    """
    options: dict[str, object] = {
        "bucket_name": bucket_name,
        "access_key": access_key,
        "secret_key": secret_key,
        "endpoint_url": endpoint_url,
        "region_name": region_name or "auto",
        "signature_version": "s3v4",
        "querystring_auth": querystring_auth,
        "querystring_expire": querystring_expire,
        "client_config": Config(
            request_checksum_calculation="when_required",
            response_checksum_validation="when_required",
        ),
    }
    if custom_domain:
        options["custom_domain"] = custom_domain
    return {"BACKEND": "storages.backends.s3.S3Storage", "OPTIONS": options}
```
- **No `default_acl` key at all.** (`botocore.config.Config` import is safe at settings-load time —
  it touches no Django settings.)

### 6b. Rewrite the `settings_prod.py` Media Storage block
- **Remove** the commented `ACL_OPTIONS = [...]` block and the `AWS_DEFAULT_ACL = os.getenv(...)` read.
- Read the R2 env vars (bucket, keys, endpoint, region, custom domain, querystring auth/expire) and
  call the helper. Defaults live in this assembly (region falls back inside the helper;
  querystring-auth defaults **True** = private signed URLs; expire defaults `3600`):

```python
AWS_STORAGE_BUCKET_NAME = os.getenv("AWS_STORAGE_BUCKET_NAME")

if AWS_STORAGE_BUCKET_NAME:
    from freedom_ls.deployment.storage import build_s3_media_storage

    default_storage = build_s3_media_storage(
        bucket_name=AWS_STORAGE_BUCKET_NAME,
        access_key=os.getenv("AWS_S3_ACCESS_KEY_ID"),
        secret_key=os.getenv("AWS_S3_SECRET_ACCESS_KEY"),
        endpoint_url=os.getenv("AWS_S3_ENDPOINT_URL"),
        region_name=os.getenv("AWS_S3_REGION_NAME"),
        custom_domain=os.getenv("AWS_S3_CUSTOM_DOMAIN"),          # unset ⇒ private signed URLs
        # default True (private signed URLs); case-insensitive so `false`/`False` both disable it
        querystring_auth=os.getenv("AWS_QUERYSTRING_AUTH", "True").strip().lower() != "false",
        querystring_expire=int(os.getenv("AWS_QUERYSTRING_EXPIRE", "3600")),
    )
else:
    default_storage = {"BACKEND": "django.core.files.storage.FileSystemStorage"}
```

  intact (the `STORAGES` dict literal continues to reference `default_storage`).
