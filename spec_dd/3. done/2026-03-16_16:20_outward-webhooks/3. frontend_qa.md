# Outward Webhooks — Frontend QA Plan

## Setup

- **Base URL:** `http://127.0.0.1:8000`
- **Login:** Use admin credentials (`demodev@email.com` / `demodev@email.com`) — must be a superuser
- **Admin URL:** `http://127.0.0.1:8000/admin/`

For test ping verification, you can use a request capture service like webhook.site or run a local server. For QA purposes, you can use `http://localhost:9999/webhook` as a dummy URL (it won't receive, but we can verify the delivery records are created).

---

## Test 1: Create a Webhook Endpoint

1. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookendpoint/`
2. Click "Add Webhook Endpoint"
3. Verify the form shows:
   - A text input for **Description**
   - A text input for **URL**
   - **Event types** rendered as checkboxes with labels: "User registered", "Course completed", "Course registered"
   - An **Is active** checkbox (checked by default)
   - The **Secret** field should NOT be visible on the create form (it's auto-generated)
4. Fill in:
   - Description: "Test Endpoint"
   - URL: `http://localhost:9999/webhook` (HTTP allowed in dev)
   - Check "User registered" and "Course completed"
   - Leave "Is active" checked
5. Click Save
6. Verify you're redirected to the detail/change view
7. Verify the **Secret** field is now visible and populated with a long random string (read-only)
8. Verify **Failure count** shows 0
9. Verify **Created at** and **Updated at** are populated

**Expected:** Endpoint is created successfully with auto-generated secret and selected event types stored.

## Test 2: Webhook Endpoint List View

1. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookendpoint/`
2. Verify the list shows columns: Description, URL (truncated), Event types, Is active, Failure count, Created at
3. Verify filters are available for: Is active, Created at
4. Verify the endpoint created in Test 1 appears

**Expected:** List view displays all columns correctly with working filters.

---

## Test 3: Send Test Ping

1. Navigate to the webhook endpoint list
2. Select the endpoint created in Test 1
3. From the actions dropdown, select "Send test ping"
4. Click "Go"
5. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookevent/`
6. Verify a new event appears with event_type `webhook.test`
7. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookdelivery/`
8. Verify a delivery record exists for the test ping event
9. The delivery status should be `failed` (since the dummy URL is not listening) with a connection error

**Expected:** Test ping creates an event and delivery record. Since the endpoint isn't reachable, the delivery shows as failed with appropriate error info.

---

## Test 4: Webhook Event List (Read-Only)

1. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookevent/`
2. Verify events are listed with columns: Event type, ID (short), Created at
3. Verify there is NO "Add" button (events are system-generated)
4. Click on an event to view details
5. Verify all fields are read-only
6. Verify the payload JSON is displayed

**Expected:** Events are read-only with no add/edit capability.

---

## Test 5: Webhook Delivery List and Retry

1. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookdelivery/`
2. Verify deliveries are listed with columns: Event type, Endpoint, Status, Attempt count, Last status code, Last attempt at
3. Verify filters are available for: Status, Endpoint, Event type
4. Click on a delivery to view details
5. Verify full response body and all delivery fields are shown (read-only)
6. Go back to the list, select a failed delivery
7. From the actions dropdown, select "Retry delivery"
8. Click "Go"
9. Verify the attempt count has incremented

**Expected:** Delivery details are visible and retry action works.

---

## Test 6: Enable/Disable Endpoint Actions

1. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookendpoint/`
2. Select an active endpoint
3. From the actions dropdown, select "Disable endpoints"
4. Click "Go"
5. Verify the endpoint now shows as inactive
6. Select it again, choose "Enable endpoints", click "Go"
7. Verify it shows as active again

## Test 9: End-to-End Webhook Flow (User Registration)

1. Log out of the admin
2. Navigate to `http://127.0.0.1:8000/accounts/signup/` (if signup is enabled)
3. Register a new user account
4. Log in as admin (`demodev@email.com`)
5. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookevent/`
6. Verify a `user.registered` event was created with the new user's details in the payload
7. Navigate to `http://127.0.0.1:8000/admin/webhooks/webhookdelivery/`
8. Verify a delivery was created for the endpoint that subscribes to `user.registered`

**Expected:** Real event fires and creates event + delivery records automatically.
