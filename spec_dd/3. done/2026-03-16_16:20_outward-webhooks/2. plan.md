# Outward Webhooks — Implementation Plan

## Prerequisites

This feature requires Django 6.0+ with `django.tasks` available. The Django 6.0 upgrade must be completed before starting implementation. All async delivery uses `django.tasks.enqueue()` directly.

### Task 2.1: Implement webhook signing

Create `freedom_ls/webhooks/signing.py`:

```python
def sign_webhook(
    body: str,
    secret: str,
    webhook_id: str,
    timestamp: int,
) -> str:
    """
    Compute Standard Webhooks signature.
    Returns 'v1,{base64-encoded HMAC-SHA256}'.

    message = f"{webhook_id}.{timestamp}.{body}"
    signature = hmac.new(secret.encode(), message.encode(), hashlib.sha256)
    return f"v1,{base64.b64encode(signature.digest()).decode()}"
    """
```

### Task 2.2: Implement delivery logic

Create `freedom_ls/webhooks/delivery.py`:

```python
import json
import time
import httpx
from django.utils import timezone

RETRY_DELAYS = [60, 300, 1800, 7200, 43200]  # seconds
MAX_ATTEMPTS = 6  # initial + 5 retries

def build_webhook_payload(event: WebhookEvent) -> str:
    """Build the JSON envelope: {id, type, timestamp, data}."""

def build_webhook_headers(
    body: str, endpoint: WebhookEndpoint, event: WebhookEvent
) -> dict[str, str]:
    """Build headers: webhook-id, webhook-timestamp, webhook-signature, Content-Type."""

def attempt_delivery(delivery: WebhookDelivery) -> None:
    """
    Send HTTP POST to the endpoint URL.
    - Build payload and headers
    - Send request with 30s timeout using httpx
    - Classify response (2xx success, 429 retry with Retry-After, 4xx fail, 5xx/timeout/connection error retry)
    - Update delivery record (status, attempt_count, last_status_code, last_response_body, last_attempt_at, last_latency_ms)
    - On success: reset endpoint failure_count to 0, clear disabled_at
    - On retryable failure: increment endpoint failure_count, schedule next retry with jitter, or mark dead_letter if max attempts reached
    - On permanent failure (4xx except 429): mark delivery as failed
    """

def calculate_next_retry(attempt_count: int) -> datetime:
    """Base delay from RETRY_DELAYS + random jitter up to 20%."""

def check_circuit_breaker(endpoint: WebhookEndpoint) -> bool:
    """
    Returns True if delivery should proceed.
    - If endpoint.is_active: proceed
    - If not active and disabled_at is None: skip (manually disabled)
    - If not active and disabled_at + 1 hour < now: proceed (probe)
    - Otherwise: skip
    """

def handle_circuit_breaker_trip(endpoint: WebhookEndpoint) -> None:
    """If failure_count >= 5: set is_active=False, disabled_at=now."""
```

Use `httpx` (add to dependencies via `uv add httpx` if not already present).

**TDD:** Test each function independently:
- `build_webhook_payload` returns correct envelope structure
- `build_webhook_headers` includes all required Standard Webhooks headers
- `attempt_delivery` with mocked httpx — test 2xx, 429, 4xx, 5xx, timeout, connection error scenarios
- `calculate_next_retry` returns values within expected ranges (base + jitter)
- `check_circuit_breaker` returns correct boolean for active, disabled, probing states

## Phase 3: Event Firing

**Skills:** `testing`

### Task 3.1: Implement `fire_webhook_event`

In `freedom_ls/webhooks/events.py`:

```python
from django.db import transaction
from django.tasks import enqueue
from freedom_ls.site_aware_models.models import _thread_locals, get_cached_site

def fire_webhook_event(event_type: str, payload: dict) -> None:
    """
    Fire a webhook event. Must be called within a request context.

    1. Validate event_type against registry
    2. Capture site_id from current request context
    3. Inside transaction.on_commit():
       a. Create WebhookEvent record (with explicit site_id)
       b. enqueue(dispatch_event, event_id, site_id)
    """

def send_test_ping(endpoint: WebhookEndpoint) -> None:
    """
    Send a webhook.test event to a specific endpoint.
    Bypasses registry validation. Creates a WebhookEvent with type='webhook.test'
    and a static payload {'ping': True}, then creates and attempts a single delivery.
    """

def dispatch_event(event_id: str, site_id: int) -> None:
    """
    Background task. Look up all active WebhookEndpoint records for the given
    site_id that subscribe to the event type. Filter explicitly by site_id
    (cannot use SiteAwareManager — no request context in background tasks).

    For each matching endpoint:
    - Check circuit breaker
    - Create WebhookDelivery(status='pending')
    - Call attempt_delivery(delivery)
    """
```

**Important:** `dispatch_event` runs outside request context (no `_thread_locals.request`), so `SiteAwareManager` returns the unfiltered queryset. The function must filter explicitly by `site_id`: `WebhookEndpoint.objects.filter(site_id=site_id)`.

**Circuit breaker behaviour in dispatch:** For each matching endpoint, check the circuit breaker first. Only create a `WebhookDelivery` record and attempt delivery if the circuit breaker allows it (active endpoint, or disabled endpoint eligible for probing). If the circuit breaker says skip, do not create a delivery record — just move on to the next endpoint.

**TDD:**
- `fire_webhook_event` validates event type, creates event record on commit
- `fire_webhook_event` raises `ValueError` for unknown event types
- `dispatch_event` finds matching endpoints and creates deliveries
- `dispatch_event` skips endpoints that don't subscribe to the event type
- `dispatch_event` respects circuit breaker
- `send_test_ping` creates event and delivery for a specific endpoint

### Task 5.2: Fire `course.registered` event

Course registrations can happen in multiple places (student self-registration in `student_interface/views.py:register_for_course`, educator actions via admin, etc.). To cover all paths, override `save()` on `UserCourseRegistration` in `freedom_ls/student_management/models.py` (line 46) to fire the event on creation:

```python
def save(self, *args, **kwargs):
    is_new = self._state.adding
    super().save(*args, **kwargs)
    if is_new:
        from freedom_ls.webhooks.events import fire_webhook_event
        fire_webhook_event("course.registered", {
            "user_id": self.user_id,
            "user_email": self.user.email,
            "course_id": str(self.collection_id),
            "course_title": self.collection.title,
            "registered_at": self.registered_at.isoformat(),
        })
```

Use `self._state.adding` to detect creation vs update, avoiding the need to check in every view. Use `select_related` in calling code where needed to avoid extra queries.

**TDD:** Test that creating a `UserCourseRegistration` fires the event, and saving an existing one does not.
