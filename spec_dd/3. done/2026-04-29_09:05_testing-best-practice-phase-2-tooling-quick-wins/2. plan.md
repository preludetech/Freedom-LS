# Implementation plan — Phase 2: testing tooling quick wins

This plan installs four pytest plugins, wires them into `pyproject.toml` / `pytest-playwright` config, sweeps the suite for failures uncovered by random ordering and socket blocking, and gates branch coverage in CI.

The plan follows TDD where applicable, but most of this work is config wiring. The "test" for each step is the green-build outcome described in the success criteria — no new feature tests are added in this spec.

Open decision is **resolved as option 1 (ratchet)**: branch-coverage threshold is set to whatever current branch coverage is on `main`, rounded down to the nearest integer.

- `.github/workflows/tests.yml` defines the `unit-tests` and `playwright-tests` jobs. The Playwright job already uploads `test-results/` on failure — confirm pytest-playwright writes traces / screenshots there by default (it does — `test-results/<test-id>/trace.zip`).

- `fls:testing` and `fls:playwright-tests` skill files already document the new tooling as "planned for upcoming phase 2"; those forward-looking notes need to be reworded to reflect that the tooling is now live, plus the marker conventions need to be added.

## Open-decision resolution

- **Branch coverage threshold approach: option 1 (ratchet from current).** The user ticked the open-decision checkbox in `todo.md`, confirming the spec's recommended option. Implementation:
  1. Run `uv run pytest --cov --cov-branch --cov-report=term -m "not playwright"` once on the current worktree (with the rest of the new addopts in place but without `--cov-fail-under`) to get the current branch-coverage percentage.
  2. Round that number **down** to the nearest integer (e.g. 67.84 % → 67) and use it as `--cov-fail-under=<N>`. Never round up — a ratchet that bumps the bar above current breaks main.
  3. Record the chosen `<N>` in the plan execution notes when implementing.

## Tasks

The order matters: install + config first, then sweep, then docs / CI.

### 1. Add dev dependencies

- Run `uv add --dev pytest-randomly pytest-socket pytest-xdist time-machine` in one command so the lock file updates atomically.
- Verify `uv.lock` updated and that `[dependency-groups] dev` and (if used) `[project.optional-dependencies] dev` stay in sync. If `uv add --dev` only updates one of the two blocks, mirror the additions into the other block by hand so the existing duplication pattern is preserved.

Note: `django-htmx` is **not** added by this spec. It is a runtime dependency (middleware + `request.htmx` API surface), not a testing dependency. Adding it would be scope creep for a testing-tooling PR. Verified absent from `uv.lock`, `pyproject.toml`, `config/`, and `freedom_ls/`.

### 2. Configure pytest addopts and coverage

In `pyproject.toml` `[tool.pytest.ini_options]`:

- Replace the current `addopts = "--strict-markers"` with the full set:

```toml
addopts = "--strict-markers --disable-socket --allow-hosts=127.0.0.1,::1 --cov --cov-branch --cov-report=term-missing --cov-fail-under=<N>"
```

  Where `<N>` is the integer determined in the open-decision-resolution step above.

- Note on `pytest-xdist`: do **not** add `-n auto` to `addopts` permanently. Keeping `-n auto` always-on slows single-test debug runs and conflicts with some debugger workflows. Document the recommended invocation in the testing skill (`uv run pytest -n auto`) and leave `addopts` serial. CI may opt-in to `-n auto` per job if wall-clock matters; for this spec, leave the CI commands unchanged.

- Add a top-level coverage section to `pyproject.toml`:

```toml
[tool.coverage.run]
branch = true
source = ["freedom_ls"]
omit = [
    "*/migrations/*",
    "*/tests/*",
    "*/conftest.py",
    "manage.py",
]

[tool.coverage.report]
show_missing = true
skip_covered = false
```

  The `source` and `omit` keys keep the report focused on production code so the ratchet number isn't inflated by un-covered fixtures or dragged down by migrations.

### 3. Configure pytest-playwright traces and screenshots

`pytest-playwright` is configured through pytest, not `playwright.config.{ts,js}` — there is no `playwright.config.*` file in the repo and we shouldn't create one.

In `pyproject.toml` `[tool.pytest.ini_options]`, add a second key:

```toml
# applies globally; pytest-playwright reads these from CLI / addopts
addopts = "... --tracing=retain-on-failure --screenshot=only-on-failure"
```

These options are no-ops for non-Playwright tests, so a single `addopts` covers both jobs.

Verify trace output path: `pytest-playwright` writes traces to `test-results/<nodeid>/trace.zip` by default, which matches the existing CI artifact upload (`.github/workflows/tests.yml` line 138, `path: test-results/`).

### 4. Sweep the suite

- Run `uv run pytest -m "not playwright"` locally with all of the above in place but **without** `--cov-fail-under` (or set it to 0) so failures from random ordering / socket blocking surface without noise from the coverage gate.
- For each failure:
  - **Order-dependence** (e.g. test asserts on PK ordering, a global is mutated by an earlier test, a fixture leaks state): fix the test. Do **not** add `@pytest.mark.order` — that defeats the purpose of `pytest-randomly`. Common fixes: replace ordered-PK assumptions with explicit ordering (`Model.objects.order_by("created_at")`), reset module-level caches in fixtures, use factory_boy instead of bare `.objects.create()` so the per-test cleanup is guaranteed.
  - **Unexpected socket** (DNS lookup, outbound HTTP not mocked): add a boundary mock following the `freedom_ls/webhooks/tests/test_admin.py` pattern (`mock.patch("module.httpx.request", return_value=mock_response)`). Only add `@pytest.mark.allow_hosts([...])` if the call is genuinely intended (e.g. a test that needs `live_server`, which already auto-passes via the `127.0.0.1` allow-list, or a test that needs `socket.getaddrinfo` for SSRF-validation testing). **Never use `@pytest.mark.allow_hosts(["*"])` or omit the host list** — that defeats the boundary-mocking guarantee for the affected test. Always allow-list specific hosts only, and prefer mocking over allow-listing wherever feasible. SSRF-validation tests in `freedom_ls/webhooks/` should mock at the `socket.getaddrinfo` boundary if they're verifying validation logic; only allow-list a real host if the test genuinely requires resolving it (which should be rare).
- Re-run until green. Once green with random order and socket blocking, capture the current branch-coverage percentage and update `--cov-fail-under` per the open-decision resolution.

### 5. Update CI workflow

In `.github/workflows/tests.yml`:

- The `unit-tests` job (`uv run pytest -m "not playwright"`) will pick up the new `addopts` automatically — including coverage. Verify the job still passes within its 10-minute timeout; if it doesn't (because of the coverage-collection overhead) bump `timeout-minutes` to 12.
- The `playwright-tests` job (`uv run pytest -m playwright`) will **also** pick up `--cov` from `addopts` by default. Per the spec ("decide explicitly whether the Playwright job should also enforce coverage or run with `--no-cov`"), pass `--no-cov` on the Playwright run:

```yaml
- name: Run Playwright tests
  run: uv run pytest -m playwright --no-cov
```

  Rationale: Playwright-driven coverage double-counts paths that the unit suite already exercises and inflates the ratchet artificially, and a Playwright flake should not interact with the coverage gate.

- The `Upload Playwright traces` step (line 133–139) already uploads `test-results/` on failure with 7-day retention. No changes needed; confirm during the sweep that the new trace artifacts land at that path.

> **Security concern resolved:** Test fixtures use throwaway dev values only — no real secrets, no production credentials, no PII. The existing 7-day artifact retention is acceptable as-is. No changes to retention or access controls required.

Touch only what the spec calls out — no broader rewrites.

### 7. Final verification

- Run `uv run pytest` (full suite, with `--cov-fail-under` enforced) and confirm green.
- Run `uv run pytest -m "not playwright"` and confirm green with the ratchet enforced.
- Run `uv run pytest -m playwright --no-cov` and confirm green.
- Re-run `uv run pytest` once more with a different random seed (`-p randomly --randomly-seed=<N>`) to sanity-check that ordering issues from the sweep are actually fixed, not just hidden by a single seed.

## Risks and mitigations

- **Coverage overhead inflates wall-clock time.** Mitigation: keep xdist as opt-in (don't bake `-n auto` into `addopts`); if CI exceeds its budget after coverage is enabled, switch the unit-tests CI command to `uv run pytest -m "not playwright" -n auto`.
- **The sweep finds many ordering bugs.** This is the *purpose* of the spec but could balloon the PR. Mitigation: each fix is mechanical (re-order a sort, swap to a factory, fix a fixture); if the count exceeds ~20 fixes, capture the long tail in a follow-up Phase 3 cleanup spec rather than expanding scope here. Cite the sweep results in the PR description.
- **Coverage source/omit miscalibration.** If the chosen `source = ["freedom_ls"]` excludes a path that should count (e.g. `concrete_apps` if used), the ratchet will be set lower than the real number. Mitigation: spot-check the coverage report's "files included" section against the `freedom_ls/` tree before fixing the threshold.
