## Phase 5: Adopt new Django 6 features

### Task 5.1: Add CSP middleware (report-only mode)

Edit `config/settings_base.py`:

1. Add to `MIDDLEWARE` list (after `SecurityMiddleware`):
   ```python
   "django.middleware.csp.ContentSecurityPolicyMiddleware",
   ```

2. Add to template `context_processors`:
   ```python
   "django.template.context_processors.csp",
   ```

3. Add CSP configuration at end of file:
   ```python
   from django.utils.csp import CSP

   SECURE_CSP_REPORT_ONLY = {
       "default-src": [CSP.SELF],
       "script-src": [CSP.SELF, CSP.UNSAFE_INLINE],
       "style-src": [CSP.SELF, CSP.UNSAFE_INLINE],
       "img-src": [CSP.SELF, "data:"],
       "connect-src": [CSP.SELF],
       "frame-src": [CSP.NONE],
   }
   ```

4. Run tests to verify CSP headers don't break anything:
   ```
   uv run pytest
   ```

**Skill:** `testing` — write a test in a new file `freedom_ls/base/tests/test_csp.py` that:
- Makes a GET request to any page (e.g., home page)
- Asserts `Content-Security-Policy-Report-Only` header is present in the response
- Asserts the header contains `default-src 'self'`

### Task 5.2: Configure Background Tasks Framework (dev backend)

Edit `config/settings_base.py`:

1. Add to `INSTALLED_APPS` (if required by Django 6):
   ```python
   "django.tasks",
   ```

2. Add task backend configuration:
   ```python
   # Background Tasks Framework
   # Using ImmediateBackend for development. A production backend
   # (e.g., database-backed) is needed before using tasks in production.
   TASKS = {
       "default": {
           "BACKEND": "django.tasks.backends.immediate.ImmediateBackend",
       },
   }
   ```

3. Run `uv run manage.py migrate` if the tasks framework requires migrations.

4. Run tests:
   ```
   uv run pytest
   ```
