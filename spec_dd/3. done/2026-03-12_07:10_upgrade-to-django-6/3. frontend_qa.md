## Test 6: CSP Headers (Report-Only)

1. Navigate to the home page (`http://localhost:8000/`)
2. Use Playwright `browser_network_requests` to inspect response headers on the page load request
3. Look for `Content-Security-Policy-Report-Only` header in the response
4. Use Playwright `browser_console_messages` to check for any CSP violation reports

**What to look for:**
- The `Content-Security-Policy-Report-Only` header is present in the response
- It contains directives: `default-src 'self'`, `script-src 'self' 'unsafe-inline'`, `style-src 'self' 'unsafe-inline'`
- No functionality is blocked (report-only mode should not block anything)
- Console may show CSP violation reports â€” these are informational only and expected
