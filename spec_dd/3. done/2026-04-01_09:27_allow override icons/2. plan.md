## Task 3: Create the Iconify JSON loader

Create `freedom_ls/icons/loader.py` with a function that reads and caches Iconify JSON data.

```python
# pseudocode
_cache: dict[str, dict] = {}

def load_iconify_data(set_name: str) -> dict:
    """Read and cache icons.json for the given icon set from node_modules."""
    if set_name in _cache:
        return _cache[set_name]
    # Map set_name to npm package name (e.g. "phosphor" -> "ph")
    package_map = {"heroicons": "heroicons", "lucide": "lucide", "tabler": "tabler", "phosphor": "ph"}
    pkg = package_map[set_name]
    json_path = Path(settings.BASE_DIR) / "node_modules" / f"@iconify-json/{pkg}" / "icons.json"
    data = json.loads(json_path.read_text())
    _cache[set_name] = data
    return data
```

The Iconify JSON format has: `{"prefix": "...", "icons": {"icon-name": {"body": "<svg path data>"}}, "width": 24, "height": 24}`. Each icon entry has a `body` key with raw SVG inner content, and may have per-icon `width`/`height` overrides.

**TDD:** Test that `load_iconify_data("heroicons")` returns a dict with `"icons"` key. Test that a known icon name exists in the data. Test that an unknown set name raises an appropriate error.

**Files:** `freedom_ls/icons/loader.py`, `freedom_ls/icons/tests/test_loader.py`

## Task 5: Build the SVG rendering function

Add a `render_icon` function to `freedom_ls/icons/renderer.py`:

```python
def render_icon(
    semantic_name: str,
    variant: str = "outline",
    css_class: str = "size-5",
    aria_label: str = "",
) -> str:
    """Resolve a semantic icon name and render it as inline SVG HTML."""
    icon_set_name = getattr(settings, "FREEDOM_LS_ICON_SET", "heroicons")
    overrides = getattr(settings, "FREEDOM_LS_ICON_OVERRIDES", {})

    set_config = ICON_SETS[icon_set_name]
    mapping = {**set_config.mapping, **overrides}
    icon_name = mapping[semantic_name]

    # Resolve variant for lookup in Iconify data
    variant_key = set_config.variants.get(variant)

    data = load_iconify_data(icon_set_name)
    icon_data = data["icons"][icon_name]

    body = icon_data["body"]
    width = icon_data.get("width", data.get("width", 24))
    height = icon_data.get("height", data.get("height", 24))

    # Accessibility: if no explicit label, use aria-hidden for decorative icons
    if aria_label:
        aria_attrs = f'role="img" aria-label="{aria_label}"'
    else:
        # Default: use semantic name as label (matching current behaviour)
        aria_attrs = f'role="img" aria-label="{semantic_name}"'

    return (
        f'<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 {width} {height}" '
        f'class="inline {css_class}" {aria_attrs}>'
        f'{body}</svg>'
    )
```

**Key decisions for implementation:**
- Do NOT hardcode `fill="none" stroke="currentColor"` on the wrapper `<svg>`. Iconify JSON body content includes `fill` and `stroke` attributes on the path elements themselves. The wrapper SVG should only set `xmlns`, `viewBox`, `class`, and ARIA attributes.
- Compare output against current `django-heroicons` rendering for the default Heroicons set to ensure visual parity (success criterion 7).

**TDD:** Test that `render_icon("success")` returns valid SVG with correct viewBox and ARIA. Test that variant changes produce different SVG content. Test that overrides are applied. Test that missing icon raises `KeyError`. Test that default Heroicons output contains the same SVG path data as `django-heroicons` produces (visual regression check).

**Files:** `freedom_ls/icons/renderer.py`, `freedom_ls/icons/tests/test_renderer.py`
