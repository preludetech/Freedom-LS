### Task 1.3: Create the `{% icon %}` template tag

**Skill**: `tdd_implement`

Create `freedom_ls/base/templatetags/icon_tags.py` with an `icon` template tag.

**How it works internally:**

```python
# pseudocode
@register.simple_tag
def icon(name: str, variant: str = "outline", class_: str = "size-5", force: bool = False, aria_label: str | None = None) -> str:
    if not force:
        heroicon_name = ICONS[name]  # KeyError if not found
    else:
        heroicon_name = name

    # Use heroicons package to render SVG
    svg = render_heroicon(heroicon_name, variant)

    # Add CSS classes
    # Add aria attributes (aria-hidden="true" by default, or role="img" + <title> if aria_label)

    return mark_safe(svg)
```

Note: The `heroicons` package provides `heroicons.templatetags.heroicons` with tags like `{% heroicon_outline "arrow-right" %}`. Study how these work internally and either:
- Call the package's rendering functions directly, or
- Build the SVG path lookup using the package's icon data

The `class` parameter must be passed as a keyword arg. Since `class` is a Python reserved word, accept it as `class_` in the Python function but in templates it will be used as: `{% icon "name" class="size-5" %}`. Django template tags handle this — use `**kwargs` and extract `kwargs.get("class", "size-5")`.

**TDD**: Write tests first in `freedom_ls/base/tests/test_icon_tag.py`:
1. `{% icon "next" %}` renders an `<svg>` element
2. `{% icon "next" %}` and `{% icon "arrow-right" force=True %}` render the same heroicon
3. `variant="solid"` renders the solid variant
4. Default output includes `aria-hidden="true"`
5. `aria_label="Close"` renders `role="img"` and `<title>Close</title>`
6. Unknown name raises `KeyError`
7. `force=True` bypasses registry
8. Default class is `"size-5"`
9. Custom `class="size-6 text-red-500"` is applied


## Phase 5: Create Claude skill

### Task 5.1: Create `icon-usage` Claude skill

Create `.claude/skills/icon-usage/SKILL.md` with instructions for future icon work:

- Always use `{% icon "semantic_name" %}` template tag
- Add new semantic names to `ICONS` dict in `freedom_ls/base/icons.py` when new concepts need icons
- Never use raw FA classes, hand-coded inline SVGs for standard icons, or Unicode icon characters
- Use appropriate Tailwind sizing: `size-4` compact, `size-5` standard, `size-6` emphasis
- For Alpine.js dynamic toggling, use `x-show` on wrapper spans around two `{% icon %}` calls (or `rotate-180` class for directional flips)
- For accessibility: decorative icons get `aria-hidden="true"` (default), standalone informative icons use `aria_label` parameter
