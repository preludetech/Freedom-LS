# Frontend QA: Icon Migration

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- Tailwind built: `npm run tailwind_build`
- A student user exists and is enrolled in at least one course
- An educator user exists with access to at least one cohort

---

## Test 1: Student Course Navigation Icons

**Login as**: Student user
**URL**: `/courses/` (or the student dashboard)

1. Navigate to any course home page
2. **Verify**: Course status icons render as SVG icons (not text/squares/missing):
   - Completed courses show a check-circle icon
   - In-progress courses show a spinner/loading icon
3. Click into a course to view the Table of Contents sidebar

### TOC Sidebar
4. **Verify** the TOC panel:
   - The "open TOC" button shows a chevron-right SVG icon
   - The "collapse TOC" button shows a chevron-left SVG icon
   - Both buttons have `cursor: pointer` and consistent styling
5. **Verify** status icons in the TOC:
   - Locked items show a lock icon (greyed out)
   - Ready/in-progress items show a play icon (blue/primary)
   - Completed items show a check icon (green/success)
   - Failed items show a retry/repeat icon (red/danger)
6. **Verify** content type icons:
   - Topics show a book icon
   - Forms/quizzes show a pencil-square icon
   - Courses show an academic-cap icon
   - Course parts/sections show a folder icon
7. **Verify** deadline badges show a clock icon next to dates
8. **Verify** expandable course parts:
   - Collapsed shows a chevron-right icon
   - Click to expand — shows chevron-down icon
   - Toggling works smoothly (no layout jump or flash)

## Test 13: No Font Awesome Loaded

**How to check**: Open browser DevTools → Network tab

1. Reload any page
2. **Verify**: No request to `cdnjs.cloudflare.com/ajax/libs/font-awesome/` appears
3. Open DevTools → Elements tab, search for `font-awesome` — should find nothing
4. Search for `class="fa` in Elements — should find nothing

---

## Test 14: Accessibility Check

1. On any page with icons, use browser DevTools to inspect SVG icons
2. **Verify** decorative icons (next to text) have `aria-hidden="true"`
3. **Verify** standalone informative icons have `role="img"` and a `<title>` element
4. Tab through interactive elements — verify screen reader announces button labels, not icon names
