# Idea: Replace Font Awesome with Heroicons

## Problem

Currently we use Font Awesome loaded via CDN for all icons. This has several problems:

- Third-party CDN dependency (privacy concern, single point of failure)
- Loads ~100KB CSS for only 28 icons actually used
- Icons are implemented inconsistently: Font Awesome `<i>` tags, inline SVGs, and Unicode characters are all used for the same semantic concepts
- No central registry — changing an icon's meaning requires hunting through templates

## Solution

Replace Font Awesome with **Heroicons** (by Tailwind Labs) using the **cotton-icons** package for Django Cotton integration.

### Icon library: Heroicons

- Made by Tailwind Labs — guaranteed visual harmony with our TailwindCSS stack
- ~316 unique icons across 4 styles: outline, solid, mini, micro
- MIT license
- Excellent Django support via `cotton-icons` package

### Integration: cotton-icons + semantic `<c-icon>` component

- Use the `cotton-icons` package which provides `<c-heroicon.arrow-right />` style components
- Build a semantic `<c-icon name="next" />` wrapper component that maps semantic names to actual Heroicons
- The semantic mapping lives in a central Python registry (`icons.py`) exposed via context processor
- Editing the registry updates all icons application-wide

### Central icon registry

Create a Python dict mapping semantic purpose to icon name. For example:

- `next` → arrow-right
- `previous` → arrow-left
- `deadline` → clock
- `menu_open` → chevron-right (open TOC sidebar)
- `menu_close` → chevron-left (close TOC sidebar)
- `success` → check-circle
- `error` → x-circle
- `warning` → exclamation-triangle
- `close` → x-mark
- etc.

### What to migrate

Replace all three current icon systems:

1. **38 Font Awesome icon instances** (28 unique icons) across templates
2. **11 inline SVG icons** (6 unique shapes) in templates
3. **4 Unicode characters** (✓, ▶, —, ⏲) in educator progress panel

Key abstraction points that make migration easier:

- `<c-button>` component has `icon_left`/`icon_right` params — changing this propagates to all button usages
- `<c-chip>` component has `icon` param
- `<c-callout>` component maps callout types to icon classes

### Accessibility

- All icons default to `aria-hidden="true"` (decorative)
- Provide an `aria_label` parameter for informative/standalone icons
- Default size: `size-5` (20px), using Tailwind's `size-*` utility

### Remove Font Awesome

- Remove the CDN link from `_base.html`
- Write a CI test that greps for leftover FA patterns (`fa-`, `fa `, `font-awesome`) to prevent regression

### Claude Skill

Create a Claude skill for icon usage that ensures all future work:

- Uses the `<c-icon>` component with semantic names
- Adds new semantic names to the registry when needed
- Never uses raw FA classes, inline SVGs for standard icons, or Unicode icon characters
