## Phase 1: Shared Sidebar Component (Issues #1, #2)

These two issues are tightly coupled and require creating a reusable cotton component.

### Task 1.2: Study existing sidebar implementations

Read and understand the two current sidebar patterns:
- **Student course sidebar:** `freedom_ls/student_interface/templates/student_interface/_course_base.html` - has matchMedia + localStorage pattern
- **Educator sidebar:** `freedom_ls/educator_interface/templates/educator_interface/interface.html` - hardcoded `sidebarOpen: true`, `w-64`, no mobile detection

### Task 1.3: Create shared sidebar cotton component

Create `freedom_ls/base/templates/cotton/sidebar.html` that:
- Accepts slot content for sidebar items
- Uses `window.matchMedia('(min-width: 1024px)').matches` for default state
- Persists state via localStorage with a configurable key (e.g., `sidebar-open-educator`, `sidebar-open-course-toc`)
- On mobile (< 1024px) when expanded: overlay pattern with backdrop (fixed positioning over content, semi-transparent backdrop)
- On desktop (>= 1024px) when expanded: standard side-by-side layout (sidebar pushes content)
- Toggle button always visible
- Tapping backdrop closes sidebar on mobile
- Accepts a `storage-key` attribute to scope localStorage per interface

Pseudocode for Alpine.js data:
```javascript
{
  sidebarOpen: JSON.parse(localStorage.getItem(storageKey)) ?? window.matchMedia('(min-width: 1024px)').matches,
  isMobile: !window.matchMedia('(min-width: 1024px)').matches,
  init() {
    const mq = window.matchMedia('(min-width: 1024px)');
    mq.addEventListener('change', (e) => {
      this.isMobile = !e.matches;
      // Optionally auto-close on resize to mobile
    });
    this.$watch('sidebarOpen', (val) => {
      localStorage.setItem(storageKey, JSON.stringify(val));
    });
  },
  toggle() { this.sidebarOpen = !this.sidebarOpen; }
}
```

Mobile overlay styles:
- Sidebar: `fixed inset-y-0 left-0 z-40 w-64 bg-surface`
- Backdrop: `fixed inset-0 z-30 bg-black/50` with `@click="sidebarOpen = false"`

Desktop styles:
- Sidebar: `relative w-64 flex-shrink-0`
- No backdrop needed

## Phase 2: Mobile Table/Grid Strategy (Issues #3, #5)

These share the same pattern and should be implemented together.

### Task 2.1: Verify table issues exist

Use **use-playwright** skill to:
- Navigate to educator progress grid at 375px and confirm first column is sticky and rest is hidden
- Navigate to any data table (cohort list, student list) at 375px and confirm similar issues

### Task 2.2: Implement mobile scroll indicator for `c-data-table`

Edit `freedom_ls/base/templates/cotton/data-table.html`:

Below `md:` breakpoint:
- Remove sticky/frozen first column behavior
- Add JavaScript (Alpine.js) to detect horizontal scroll position
- When scrolled past the first column, show the first column value as a floating label above each visible row

Pseudocode for the scroll behavior:
```javascript
{
  scrolledPastFirst: false,
  init() {
    const container = this.$refs.tableContainer;
    container.addEventListener('scroll', () => {
      // Check if first column is scrolled out of view
      this.scrolledPastFirst = container.scrollLeft > firstColWidth;
    });
  }
}
```

For the floating row identifier:
- Each `<tr>` gets a `data-row-label` attribute with the first column value
- When `scrolledPastFirst` is true, use CSS `::before` pseudo-element or an Alpine-driven element to show the label above each row
- Style: small text, background matching surface, positioned above the row

Since this is the `c-data-table` cotton component, the fix applies to all data tables automatically (cohort lists, student lists, etc.).

### Task 2.3: Apply same pattern to progress grid

Edit `freedom_ls/educator_interface/templates/educator_interface/partials/course_progress_panel.html`:

The progress grid is a custom table (not using `c-data-table`), so apply the same pattern:
- Below `md:` breakpoint, remove `sticky left-0` from the student name column
- Add the same scroll detection and floating row identifier behavior
- Keep the `overflow-auto` container

Consider extracting the scroll-detection JavaScript into a shared Alpine component or inline script that both the data-table and progress grid can use.

### Task 2.4: Verify table fixes

Use **use-playwright** skill at 375px:
- Progress grid scrolls freely, student name appears above row when scrolled
- Data tables scroll freely, first column value appears above row when scrolled
- All data remains accessible, nothing hidden

### Task 3.3: Issue #7 - Dropdown menu viewport detection

Edit `freedom_ls/base/templates/cotton/dropdown-menu.html`:

Add viewport boundary detection to the Alpine.js positioning logic:

```javascript
// After calculating position:
const menuWidth = 160; // w-40
const menuHeight = menuEl.offsetHeight;
const viewportWidth = window.innerWidth;
const viewportHeight = window.innerHeight;

// Right edge detection
if (x + menuWidth > viewportWidth) {
  x = viewportWidth - menuWidth - 8; // 8px margin
}
// Left edge detection
if (x < 8) {
  x = 8;
}
// Bottom edge detection - open upward
if (y + menuHeight > viewportHeight) {
  y = rect.top - menuHeight;
}
```

Verify with **use-playwright** at 360px by triggering dropdowns near screen edges.
