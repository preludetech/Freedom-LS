## Batch 6: Claude Code Security Hooks & Commands (Spec §6)

### Task 6.1: Create security guard PreToolUse hook

**File:** `.claude/hooks/security-guard.sh`

Create a bash script that reads JSON from stdin, checks `tool_name` and `tool_input`, and exits with code 2 to block dangerous patterns being introduced. The script should:

- For `Edit`/`Write` tools: check `new_string`/`content` for blocked patterns (`raw(`, `extra(`, `RawSQL`, `mark_safe(`, `@csrf_exempt`, `eval(`, `exec(`, `pickle.loads(`, `yaml.load(`, `**request.POST.dict()`, `**request.GET.dict()`)
- For `Bash` tool: check `command` for `rm -rf`, access to `.env`/`id_rsa`/`*.pem`/`*.key` files
- For `Write`/`Edit` tools: block modifications to `.env*` files (allow `.env.example`), block modifications to existing migration files

Exit 0 (allow) by default, exit 2 (block) with a message when a pattern is detected.

Use `jq` to parse the JSON input. Use `shellcheck`-clean bash.

### Task 6.2: Create post-edit Bandit scan hook

**File:** `.claude/hooks/post-edit-bandit.sh`

```bash
#!/bin/bash
set -euo pipefail
INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | jq -r '.tool_input.file_path // .tool_input.path // ""')
if [[ "$FILE_PATH" =~ \.py$ ]]; then
    RESULTS=$(uv run bandit -q -ll "$FILE_PATH" 2>&1) || true
    if [[ -n "$RESULTS" ]]; then
        echo "Bandit found issues in $FILE_PATH:"
        echo "$RESULTS"
    fi
fi
exit 0
```
