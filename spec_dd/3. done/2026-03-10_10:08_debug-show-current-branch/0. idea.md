When DEBUG=True then all pages should display an unobtrusive panel that displays what git branch we are on.

## Core behaviour

- Fixed position, bottom-left corner (least contested screen area, avoids conflict with chat widgets etc.)
- Small pill/badge in collapsed state (~40px), expands on click to show full branch name
- Collapsed state shows a colored dot; expanded state shows the full branch name
- Click to toggle expand/collapse (not hover — unreliable on touch devices)
- Persist expand/collapse state in `localStorage` so it survives page loads
- Only visible when `DEBUG=True` — two layers of safety: context processor returns empty dict when not DEBUG, template uses `{% if %}` guard
- By default it should be expanded. We should easily see what branch we are on

## Color from branch name

- Use hash-based HSL color generation (deterministic: same branch name always gives the same color)
- Hash the branch name with MD5, map to hue (0-360), constrain saturation (0.45-0.75) and lightness (0.40-0.60) for readability
- Use generated color as the badge background; pick black or white text based on luminance (WCAG contrast)
- No third-party dependencies needed — Python's `hashlib` and `colorsys` are sufficient

## Git branch detection

- Read `.git/HEAD` directly (fastest, no subprocess, no dependencies)
- Must handle git worktrees (`.git` can be a file containing `gitdir: /path/...` rather than a directory) — this project uses worktrees
- Handle detached HEAD gracefully (show short SHA)
- Return None if no git repo detected
- Cache the branch name at first read (lazy singleton) — branch doesn't change while server is running
- Zero overhead in production: context processor is only registered in dev settings, so it is never called when `DEBUG=False`

## Django integration

- Context processor in `freedom_ls/base/context_processors.py`, but only registered in `config/settings_dev.py` (not `settings_base.py`) — so it is never called in production at all. Follow the same pattern used for `debug_toolbar` and `django_browser_reload` in `settings_dev.py`: append to the `TEMPLATES[0]["OPTIONS"]["context_processors"]` list.
- Template badge in `_base.html` just before `</body>` so it appears on all pages. The template block should be guarded with `{% if debug_branch_name %}` so it's a no-op when the context processor isn't registered.
- Scoped CSS/styles to avoid conflicts with the rest of the app
- `position: fixed` with reasonable z-index (z-50 in Tailwind)

## Style notes

- Use `font-mono` for the branch name
- Small offset from the corner (8-12px) so it doesn't crowd the edge
- Smooth CSS transition for expand/collapse
- On very small screens (< 480px), consider hiding or making extra compact
