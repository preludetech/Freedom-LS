## Task 1: Branch Detection Logic

**Skill:** `testing` (TDD)

**File to edit:** `freedom_ls/base/context_processors.py`
**Test file:** `freedom_ls/base/tests/test_context_processors.py`

Implement a function `get_current_branch()` that:

1. Finds the `.git` path relative to `settings.BASE_DIR`
2. If `.git` is a file (worktree), reads the `gitdir:` path from it and follows that path to find `HEAD`
3. If `.git` is a directory, reads `.git/HEAD` directly
4. Parses `HEAD`:
   - If it starts with `ref: refs/heads/`, extract the branch name
   - Otherwise, it's a detached HEAD — return the first 7 characters (short SHA)
5. Returns `None` if anything fails (no git repo, file not found, etc.)
6. Caches the result in a module-level variable (lazy singleton pattern) so it's only read once per server lifetime

```python
# Pseudocode
_cached_branch: str | None = ...  # sentinel to distinguish "not yet read" from "no branch"
_UNSET = object()

def get_current_branch() -> str | None:
    global _cached_branch
    if _cached_branch is not _UNSET:
        return _cached_branch

    git_path = settings.BASE_DIR / ".git"
    if not git_path.exists():
        _cached_branch = None
        return None

    # Handle worktree: .git is a file with "gitdir: /path/..."
    if git_path.is_file():
        content = git_path.read_text().strip()
        if content.startswith("gitdir:"):
            git_dir = Path(content.split("gitdir:", 1)[1].strip())
            if not git_dir.is_absolute():
                git_dir = git_path.parent / git_dir
            head_path = git_dir / "HEAD"
        else:
            _cached_branch = None
            return None
    else:
        head_path = git_path / "HEAD"

    head_content = head_path.read_text().strip()
    if head_content.startswith("ref: refs/heads/"):
        _cached_branch = head_content[len("ref: refs/heads/"):]
    else:
        _cached_branch = head_content[:7]  # detached HEAD, short SHA
    return _cached_branch
```

**Test cases:**
- Normal `.git/HEAD` with `ref: refs/heads/main` -> returns `"main"`
- Worktree `.git` file with `gitdir:` pointing to a dir containing HEAD -> returns branch name
- Detached HEAD (raw SHA in HEAD) -> returns first 7 chars
- No `.git` path -> returns `None`
- Malformed `.git` file -> returns `None`
- Caching: second call returns cached value without re-reading files
- Relative `gitdir:` path resolution

Use `tmp_path` and `unittest.mock.patch` to mock `settings.BASE_DIR` for isolation.

## Task 2: Color Generation from Branch Name

**Skill:** `testing` (TDD)

**File to edit:** `freedom_ls/base/context_processors.py`
**Test file:** `freedom_ls/base/tests/test_context_processors.py`

Implement two functions:

### `branch_name_to_color(name: str) -> str`
- Hash with `hashlib.md5(name.encode()).hexdigest()`
- Extract hue, saturation, lightness from hash bytes:
  - Hue: `int(hash_hex[:8], 16) % 360` (0-360)
  - Saturation: `0.45 + (int(hash_hex[8:16], 16) % 100) / 100 * 0.30` (0.45-0.75)
  - Lightness: `0.40 + (int(hash_hex[16:24], 16) % 100) / 100 * 0.20` (0.40-0.60)
- Convert HSL to RGB using `colorsys.hls_to_rgb` (note: colorsys uses HLS order)
- Return hex string like `#a3b2c1`

### `get_text_color(bg_hex: str) -> str`
- Parse hex to RGB
- Calculate relative luminance per WCAG: `L = 0.2126*R + 0.7152*G + 0.0722*B` (where R/G/B are linearized sRGB values)
- Return `"#000000"` if luminance > 0.179, else `"#ffffff"`

```python
# Pseudocode for text color
def get_text_color(bg_hex: str) -> str:
    r, g, b = int(bg_hex[1:3], 16)/255, int(bg_hex[3:5], 16)/255, int(bg_hex[5:7], 16)/255
    # Linearize sRGB
    def linearize(c):
        return c / 12.92 if c <= 0.04045 else ((c + 0.055) / 1.055) ** 2.4
    luminance = 0.2126 * linearize(r) + 0.7152 * linearize(g) + 0.0722 * linearize(b)
    return "#000000" if luminance > 0.179 else "#ffffff"
```

**Test cases:**
- Same branch name always produces the same color (deterministic)
- Different branch names produce different colors (spot check a few)
- Generated colors have saturation in 0.45-0.75 range and lightness in 0.40-0.60 range
- Text color is black for light backgrounds, white for dark backgrounds
- Known input/output pairs (compute expected values once and hardcode in tests)

```python

    "debug_branch_color": branch_name_to_color(branch_name),
    "debug_branch_text_color": get_text_color(branch_name_to_color(branch_name)),
}
```

**Test cases:**
- When branch is detected: returns all three keys with correct types
- When no branch: returns empty dict

## Task 5: Badge Template Markup in `_base.html`

**Skills:** `template`, `frontend-styling`

**File to edit:** `freedom_ls/base/templates/_base.html`

Add just before `</body>`. Use Alpine.js (already loaded in `_base.html`) for all state management, toggling, and localStorage persistence — no manual DOM manipulation or separate `<script>` blocks needed.

```html
{% if debug_branch_name %}
<div x-data="{ expanded: localStorage.getItem('debug-branch-expanded') !== 'false' }"
     x-effect="localStorage.setItem('debug-branch-expanded', expanded)"
     @click="expanded = !expanded"
     :style="expanded
         ? 'padding: 4px 10px; border-radius: 9999px'
         : 'width: 16px; height: 16px; border-radius: 50%'"
     style="position: fixed; bottom: 10px; left: 10px; z-index: 50;
            font-family: ui-monospace, monospace; font-size: 13px;
            cursor: pointer; user-select: none; transition: all 0.2s ease;
            background-color: {{ debug_branch_color }};
            color: {{ debug_branch_text_color }};">
    <span x-show="expanded" x-transition.opacity>{{ debug_branch_name }}</span>
</div>
<style>
@media (max-width: 479px) {
    #debug-branch-badge { font-size: 10px !important; bottom: 4px !important; left: 4px !important; }
}
</style>
{% endif %}
```

**Why Alpine:** It's already loaded globally. `x-data` initializes state from localStorage, `x-effect` persists it reactively, `@click` toggles, `:style` swaps between pill and dot, and `x-show` with `x-transition.opacity` handles the text fade. This replaces ~30 lines of vanilla JS with a single declarative element.

Use inline styles (not Tailwind utility classes) to keep this fully scoped and avoid needing Tailwind to be aware of debug-only markup. The `{% if debug_branch_name %}` guard ensures nothing renders when the context variable is absent.

**Key requirements:**
- Inline/scoped styles only (no Tailwind dependency for this element)
- Alpine.js for all interactivity (click toggle, localStorage, style binding)
- Responsive compact on small screens via `<style>` block
- `{% if debug_branch_name %}` guard
