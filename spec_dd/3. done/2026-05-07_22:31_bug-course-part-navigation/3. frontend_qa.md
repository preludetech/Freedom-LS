# Frontend QA ‚Äî Course part navigation re-index

## What we're verifying

The two reported bugs and the latent Next round-trip are gone, and the existing happy path still works:

1. **Bug 1** ‚Äî On the very first viewable item of a course that begins with a `CoursePart`, the Previous button is **not rendered**.
2. **Bug 2** ‚Äî On the first viewable item of any non-first part, Previous links **directly** to the last viewable item of the previous part (no redirect bounce, no landing-on-the-same-page).
3. **Latent Next** ‚Äî Last viewable item of a part links **directly** to the first viewable item of the next part (single hop, not a redirect).
4. **TOC** ‚Äî Clicking a `CoursePart` row in the table of contents lands directly on its first viewable child (no redirect hop).
5. **No regression** ‚Äî Middle-of-part navigation, last item of the course, and direct topic/quiz/survey rendering still work.

We use the **DemoDev** site and the `functionality_demo_course_parts` course as the reproduction case.

---

## Pre-flight

From the worktree root:

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
```

Take note of `$PORT`. The base URL throughout this plan is `http://127.0.0.1:$PORT`.

Make sure the demo content is loaded on DemoDev:

```bash
uv run python manage.py content_save ./demo_content DemoDev
```

This loads the full `demo_content` tree (including `functionality_demo_course_parts`) onto the `DemoDev` site.

Start the server:

```bash
uv run python manage.py runserver $PORT
```

Open the Playwright MCP browser and log in:

- Navigate to `http://127.0.0.1:$PORT/`.
- Log in as the DemoDev user from `.claude/fls/config.md` (admin email `demodev@email.com`).
- Make sure the logged-in user is registered for "Functionality Demo - Course Parts". If not, register from the course home page.

## Test 1 ‚Äî Bug 1: no Previous button on the first viewable item

**Steps**:

1. Navigate to the course home for "Functionality Demo - Course Parts".
2. Click the first item ("Welcome") in the TOC. This should land on `index=1`.

**Expected**:

- The page renders the **Welcome** topic.
- The Previous button is **absent**. (Use `browser_snapshot` to confirm there is no link/button labelled "Previous" in the navigation row.)
- A Next control is present (because Welcome is not the last item).
- The URL ends with `/1/` (not `/2/`).

---

## Test 2 ‚Äî Bug 2: Previous on first item of a non-first part links to last item of previous part

**Steps**:

1. Navigate directly to `index=3` (Key Ideas ‚Äî first viewable item of "Core Concepts"):
   `http://127.0.0.1:$PORT/student/courses/functionality-demo-course-parts/3/`
2. Confirm the page shows **Key Ideas**.
3. In the snapshot, locate the Previous button. Note its `href`.
4. Click the Previous button.

**Expected**:

- Previous's `href` ends with `/2/` (no redirect intermediary).
- After clicking, the network log (`browser_network_requests`) shows a single `GET .../2/` returning 200 ‚Äî no 302 hop through `/...part-header/...`.
- The page now renders **What to Expect** (last viewable item of "Getting Started").
- The browser URL ends with `/2/`.

## Test 3 ‚Äî Latent Next: last item of a part links directly to first item of next part

**Steps**:

1. Navigate to `index=2` (What to Expect ‚Äî last viewable item of "Getting Started").
2. In the snapshot, locate the Next control. Note its `href` (or, on a topic that requires marking complete, the form action).
3. If the topic is already marked complete, click the Next button. Otherwise, mark complete (which submits a POST and then redirects to the next item).

**Expected**:

- For an already-completed topic: the Next button's `href` ends with `/3/`. Clicking it produces a single `GET .../3/` returning 200 ‚Äî no extra 302 through a part-header URL.
- After clicking, the page renders **Key Ideas** (first viewable item of "Core Concepts").

Repeat the spot-check at the boundary between parts 2 and 3:

4. Navigate to `index=5` (Knowledge Check). After completing/passing the quiz, the Continue/Next path should land directly at `index=6` (Summary).

## Test 7 ‚Äî Out-of-range index still 500s the same way (sanity, not a fix)

The spec explicitly leaves out-of-range index handling out of scope. After the re-index, the largest valid index is 7 (down from 10 today). Visiting `/8/` should fail in the same way it fails today (500 from `IndexError`). This is a sanity confirmation, not a defect: do **not** file this as a bug ‚Äî it's pre-existing.

**Steps**:

1. Visit `http://127.0.0.1:$PORT/student/courses/functionality-demo-course-parts/8/`.

**Expected**:

- Server returns an error page (500). No new behaviour, no special handling. Move on.

## Test 8 ‚Äî Deep links to former CoursePart slots no longer resolve

Today, `index=1` of `functionality-demo-course-parts` was a CoursePart slot that redirected forward. After the fix, `index=1` is the first viewable item directly. Confirm there is no remaining "redirect through part header" path:

**Steps**:

1. Watch the network log while clicking through items 1 ‚Üí 7 in order using the in-page Next buttons.

**Expected**:

- Every Next click is a single `GET` (or POST + 302 to a single GET for "mark complete" topics) returning 200 on the next item.
- No request chain shows two GETs back-to-back where the first is a 302 to a part-header URL.
