# Implementation Plan: Bottom-anchored toast notifications

This plan implements the spec at `1. spec.md`. It is intentionally TDD-friendly:
write the failing test first for each task that has a clear behavioural seam,
then implement.

The work splits into five layers:

1. Server-side template + base layout (live regions, viewport-fit, partial
   rewrite).
2. Alpine component rewrite (per-toast component with severity timing,
   pause-on-hover/focus/blur, soft cap eviction).
3. Django middleware that injects an OOB messages fragment into HTMX
   responses.
4. HTMX client-side hook so the OOB fragment surfaces on non-2xx responses.
5. Tests (pytest unit/integration + Playwright visual screenshots).

All tasks reference specific files. Reuse the existing
`partials/messages.html` template across both delivery paths — no duplicate
rendering logic.

  Continue to use icons `success`, `error`, `warning`, `info`, `close`. No

- **Brand colour tokens** `border-success`, `border-warning`, `border-danger`,

- **Existing `htmx:beforeSwap` handler** in

- **Django `MessageMiddleware`** is already registered in
  `config/settings_base.py`. The new middleware reads from the same message

- **Existing message tag mapping** (`debug`, `info`, `success`, `warning`,

## Task 2 — Rewrite `partials/messages.html`

File: `freedom_ls/base/templates/partials/messages.html`

Single template, used by:

- The base layout (renders the `#toast-container` wrapper plus all queued
  messages on full-page loads).
- The new HTMX middleware (renders ONLY OOB toast fragments, no container).

Use a template parameter (e.g. `{% if oob %}`) to switch between "render
container + queued messages" and "render OOB-only fragments". Keep a single
loop over `messages` so severity routing is defined in one place.

Severity → live region mapping:

| tag      | region                       |
|----------|------------------------------|
| debug    | `#toast-region-polite`       |
| info     | `#toast-region-polite`       |
| success  | `#toast-region-polite`       |
| warning  | `#toast-region-polite`       |
| error    | `#toast-region-assertive`    |

For each toast, render:

```html
<div id="toast-{{ unique_hex_id }}"
     x-data="toast"
     data-severity="{{ severity }}"
     {% if oob %}hx-swap-oob="beforeend:#toast-region-polite"{% endif %}
     aria-atomic="true"
     class="pointer-events-auto bg-surface ... border-l-4 border-{severity-token} ...">
    {# icon, message text, close button — same structure as today,
       but x-on:click="dismiss" / x-show="show" only #}
</div>
```

Notes:

- The OOB attribute differs per severity:
  `hx-swap-oob="beforeend:#toast-region-polite"` for non-error,
  `hx-swap-oob="beforeend:#toast-region-assertive"` for error. Branch in the
  template (or pass the target id in via context).
- The unique id comes from `uuid.uuid4().hex` rendered server-side. Compute
  it in a small template tag or in the view context — pick template tag so
  the partial stays self-contained.
- `aria-atomic="true"` is set on each toast itself, never on the live region
  containers (Bootstrap pitfall, spec §"Severity model and ARIA").
- The close button keeps `aria-label="Dismiss notification"` (spec changed
  from "Dismiss" — update from current).
- The icon stays `aria-hidden`. Continue using `<c-icon name="...">`.
- Retain the existing border-token mapping for severity colours.

## Task 4 — Rewrite the Alpine `message` component as `toast`

File: `freedom_ls/base/static/base/js/alpine-components.js`

Remove the existing `Alpine.data("message", ...)` registration. Replace with a
new `Alpine.data("toast", ...)` registration that supports:

- Per-severity duration: read `data-severity` from `this.$el.dataset.severity`
  and pick the duration:
  - `success` / `info` / `debug` → 5000 ms.
  - `warning` → 7000 ms.
  - `error` → no auto-dismiss (skip timer).
- Pause on hover (`@mouseenter` / `@mouseleave`).
- Pause on focus (`@focusin` / `@focusout`).
- Pause on window blur (`window.addEventListener("blur" / "focus")`,
  attached in `init`, detached in `destroy`).
- "Pause" semantics: store `remainingMs` and `startedAt`; on pause, compute
  `remainingMs -= now - startedAt` and `clearTimeout`; on resume, `setTimeout`
  with `remainingMs`. Do not restart from zero.
- `dismiss()` clears the timer and removes the element from the DOM (or sets
  `show = false` and lets `x-transition` run, then `this.$el.remove()` in a
  `setTimeout` matched to the leave duration).
- Esc handling: in `init`, add a keydown listener on `this.$el` (or use Alpine
  `@keydown.escape.window` via `x-on:keydown.escape="onEscape"` directive in
  template — note the CSP build allows method references in `x-on`). The
  handler dismisses the toast only when keyboard focus is currently inside
  `this.$el`.

Pseudocode:

```js
Alpine.data("toast", () => ({
    show: true,
    _timeoutId: null,
    _remainingMs: 0,
    _startedAt: 0,
    _paused: false,
    _windowBlurHandler: null,
    _windowFocusHandler: null,

    init() {
        const severity = this.$el.dataset.severity;
        const duration = this._durationFor(severity);
        if (duration === null) return;        // errors are persistent
        this._remainingMs = duration;
        this._start();

        this._windowBlurHandler = () => this._pause();
        this._windowFocusHandler = () => this._resume();
        window.addEventListener("blur", this._windowBlurHandler);
        window.addEventListener("focus", this._windowFocusHandler);

        // Soft-cap eviction: enforce ≤5 visible non-error toasts after insert.
        this._enforceCap();
    },

    destroy() {
        clearTimeout(this._timeoutId);
        window.removeEventListener("blur", this._windowBlurHandler);
        window.removeEventListener("focus", this._windowFocusHandler);
    },

    onMouseEnter() { this._pause(); },
    onMouseLeave() { this._resume(); },
    onFocusIn()    { this._pause(); },
    onFocusOut()   {
        // resume only when focus actually left the toast
        if (!this.$el.contains(document.activeElement)) this._resume();
    },
    onKeydown(event) {
        if (event.key === "Escape" && this.$el.contains(document.activeElement)) {
            this.dismiss();
        }
    },

    dismiss() {
        clearTimeout(this._timeoutId);
        this.show = false;
        // wait for x-transition leave (~150ms) then remove from DOM so
        // ARIA regions stay tidy and the cap accounting stays accurate
        setTimeout(() => this.$el.remove(), 200);
    },

    _durationFor(severity) {
        if (severity === "warning") return 7000;
        if (severity === "error")   return null;
        return 5000;                     // success, info, debug, fallback
    },

    _start() {
        this._startedAt = Date.now();
        this._timeoutId = setTimeout(() => this.dismiss(), this._remainingMs);
    },

    _pause() {
        if (this._paused || this._timeoutId === null) return;
        this._paused = true;
        clearTimeout(this._timeoutId);
        this._remainingMs -= Date.now() - this._startedAt;
        if (this._remainingMs < 0) this._remainingMs = 0;
    },

    _resume() {
        if (!this._paused) return;
        this._paused = false;
        this._start();
    },

    _enforceCap() {
        // see Task 5
    },
}));
```

Wire the handlers into the template via:

```django
x-on:mouseenter="onMouseEnter"
x-on:mouseleave="onMouseLeave"
x-on:focusin="onFocusIn"
x-on:focusout="onFocusOut"
x-on:keydown.escape="onKeydown"
```

(All method references — CSP build compatible.)

## Task 5 — Soft cap with error-priority eviction

In Alpine `_enforceCap()`:

- Hard cap of 5 for non-error toasts. After this toast inserts, count visible
  toasts in `#toast-region-polite` *and* `#toast-region-assertive`. If total
  visible non-error toasts > 5 OR (total visible toasts > 5 AND this toast is
  non-error), find the oldest non-error toast in either region (by DOM
  position, oldest first) and call its `dismiss()`. Loop until the count of
  non-error toasts is at most `5 - errorCount` and total non-error ≤ 5.
- Errors are never auto-evicted.
- If 5 errors are already visible and a new non-error arrives, drop the new
  non-error silently (remove from DOM in `init` before showing). The spec
  calls this rule "hard cap of 5 for non-error".

Implementation note: walk the two live region children and dispatch a
custom Alpine event (`Alpine.$dispatch`) or call the sibling toast's
`dismiss()` directly via `Alpine.$data(otherEl).dismiss()`.

## Task 7 — Django middleware for OOB toast injection

New file: `freedom_ls/base/middleware.py`

Class: `HtmxMessagesMiddleware`.

Behaviour:

- Only acts when `request.headers.get("HX-Request") == "true"`.
- After the view runs, read queued messages via
  `django.contrib.messages.get_messages(request)` (this iterates and clears
  the storage so messages aren't re-rendered on the next request).
- If there are no messages, return the response unchanged.
- Skip if response is a redirect (3xx) — the redirected request renders them
  via the full-page path. (HTMX may convert `HX-Redirect`, but the response
  body for a redirect is usually empty.)
- Skip if `Content-Type` is not text-like (e.g. JSON endpoints don't carry
  toast HTML). Inspect `response.get("Content-Type", "")` for `text/html`.
- For 4xx/5xx responses, still inject. The frontend handler (Task 8)
  permits the OOB swap regardless of status.
- Render the partial in OOB mode using
  `render_to_string("partials/messages.html", {"messages": messages, "oob": True}, request=request)`.
- Concatenate the rendered fragment to the existing response body. For
  `HttpResponse` and `TemplateResponse`, mutate `response.content`. For
  `StreamingHttpResponse`, skip (toasts can wait for the next non-streamed
  response).

Pseudocode:

```python
class HtmxMessagesMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request: HttpRequest) -> HttpResponse:
        response = self.get_response(request)
        if request.headers.get("HX-Request") != "true":
            return response
        if isinstance(response, StreamingHttpResponse):
            return response
        if 300 <= response.status_code < 400:
            return response
        content_type = response.get("Content-Type", "")
        if "text/html" not in content_type:
            return response

        storage = messages.get_messages(request)
        queued = list(storage)
        if not queued:
            return response

        fragment = render_to_string(
            "partials/messages.html",
            {"messages": queued, "oob": True},
            request=request,
        )
        response.content = response.content + fragment.encode("utf-8")
        # update Content-Length if set
        if response.has_header("Content-Length"):
            response["Content-Length"] = str(len(response.content))
        return response
```

Type hints required on every method per project conventions. No `Any`.

## Task 9 — Extend `htmx:beforeSwap` to honour OOB messages on non-2xx

File: `freedom_ls/base/static/base/js/alpine-components.js`

Extend the existing handler. Current logic treats 422 as a normal swap. The
new behaviour is:

- If `event.detail.xhr.status` is anything other than 2xx/422 AND the response
  body contains a substring matching
  `hx-swap-oob="beforeend:#toast-region-` (cheap detection, no DOM parse),
  set `event.detail.shouldSwap = true` so HTMX processes the OOB swap. Keep
  `event.detail.isError` truthful (don't lie about errors) so any other
  consumers see the real status.
- The 422 branch stays as-is.

Pseudocode:

```js
document.addEventListener("htmx:beforeSwap", (event) => {
    const status = event.detail.xhr.status;
    if (status === 422) {
        event.detail.shouldSwap = true;
        event.detail.isError = false;
        return;
    }
    if (status >= 400 && status < 600) {
        const body = event.detail.xhr.responseText || "";
        if (body.includes('hx-swap-oob="beforeend:#toast-region-')) {
            event.detail.shouldSwap = true;
            // leave isError = true so other listeners still see the error
        }
    }
});
```

Caveat to test: for a 4xx/5xx with `shouldSwap = true`, HTMX will attempt
the *primary* swap as well. Where the primary target is the form (typical
422 flow) this is fine. For a 500 error with no expected primary content,
the OOB fragment still processes because HTMX always processes OOB siblings.
Verify that primary-target swapping with garbage content does not break
anything — in practice HTMX will swap whatever is in the response. If this
is a real risk, additionally inspect the response body for the primary
target id and skip primary swapping; defer that refinement until a real
failing case appears.

## Notes

- Keep middleware logic narrow and free of side effects beyond the response
  body and `Content-Length` header.
- Do NOT introduce a `MESSAGE_TAGS` override. Default tags map cleanly.
- Do NOT add an `HX-Trigger`-based path. The spec mandates OOB only.
- Type hints on every Python function; no `Any`. Modern syntax (`X | None`).
- No silent failures: catch only the specific exceptions you expect.
- No new abstract base classes.
- No logging unless a test forces the need.
- Tests assert on behaviour (which region, which fragment, whether the timer
  pauses, whether storage clears), never on Tailwind class names.
