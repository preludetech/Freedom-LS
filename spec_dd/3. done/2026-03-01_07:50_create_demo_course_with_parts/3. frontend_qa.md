# Frontend QA: Demo Course with Course Parts

## Prerequisites

1. Start the dev server: `uv run python manage.py runserver`
2. Load the demo content: `uv run python manage.py content_save demo_content/functionality_demo_course_parts/ example.com`
3. Ensure a test student user exists and is registered for the course

## Test 1: Course appears in course listing

**Steps**:
1. Log in as a student user
2. Navigate to the course listing page

**Expected**:
- "Functionality Demo - Course Parts" appears in the list
- Subtitle "Demonstrating course organisation with parts" is visible
- Description text is shown

## Test 2: Course table of contents shows parts as section headers

**Steps**:
1. Click into "Functionality Demo - Course Parts"
2. View the course table of contents / sidebar

**Expected**:
- 3 parts appear as section headers: "Getting Started", "Core Concepts", "Wrapping Up"
- Under "Getting Started": 2 topics listed (Welcome, What to Expect)
- Under "Core Concepts": 2 topics + 1 quiz listed (Key Ideas, Going Deeper, Knowledge Check)
- Under "Wrapping Up": 1 topic + 1 survey listed (Summary, Course Feedback)
- Items are nested under their respective part headers

## Test 3: Navigate through topics in Part 1

**Steps**:
1. Click the first topic under "Getting Started" (Welcome)
2. Read the content
3. Mark it as complete / navigate to the next topic
4. Continue to "What to Expect"

**Expected**:
- Each topic loads and displays its content correctly
- Navigation flows through topics in order within the part
- Progress indicators update after completing topics

## Test 4: Quiz in Part 2 works correctly

**Steps**:
1. Navigate to the "Knowledge Check" quiz in Part 2
2. Answer some questions incorrectly, submit
3. Review results
4. Retake the quiz with all correct answers, submit

**Expected**:
- Quiz loads with multiple-choice questions
- After submission, incorrect answers are highlighted (quiz_show_incorrect: true)
- Pass/fail is based on 80% threshold
- After answering all correctly, the quiz shows as passed

## Test 5: Survey in Part 3 works correctly

**Steps**:
1. Navigate to the "Course Feedback" survey in Part 3
2. Select answers for each question
3. Submit the survey

**Expected**:
- Survey loads with multiple-choice questions
- All questions can be answered and submitted
- Survey submission succeeds without errors

## Test 6: Progress tracking with parts

**Steps**:
1. Complete all items in Part 1
2. Check the course table of contents

**Expected**:
- Part 1 ("Getting Started") shows as complete or shows a progress indicator reflecting all children complete
- Overall course progress reflects partial completion
- Parts 2 and 3 still show as incomplete

## Test 7: Full course completion

**Steps**:
1. Complete all remaining items across all parts
2. Return to the course overview

**Expected**:
- All 3 parts show as complete
- Overall course progress shows 100% / complete
- Course appears as completed in the student's course listing

## Tools

Use the `use-playwright` skill and Playwright MCP to execute these tests interactively.
