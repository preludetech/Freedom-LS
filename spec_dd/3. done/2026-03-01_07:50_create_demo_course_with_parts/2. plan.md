# Plan: Create Demo Course with Course Parts

## Overview

Create demo content files in `demo_content/functionality_demo_course_parts/` demonstrating the CoursePart feature. This is a **content-only task** — no Python code, models, or migrations are needed. All files follow existing conventions from the other demo courses and the RPC Course reference.

## Prerequisites

None. The content engine already supports CourseParts, auto-discovery, quizzes, and surveys.

## Tasks

### Task 1: Create the course file

Create `demo_content/functionality_demo_course_parts/course.md` with frontmatter:

```yaml
---
content_type: COURSE
title: Functionality Demo - Course Parts
subtitle: Demonstrating course organisation with parts
description: This course demonstrates how content can be organised into parts (chapters/modules).
---
```

No body content. No `children:` list — rely on auto-discovery. No UUID (auto-generated by `content_save`).

Reference existing course files for format: `demo_content/functionality_demo_end_with_quiz/course.md`

### Task 2: Create Part 1 — "Getting Started" (topics only)

Create directory `demo_content/functionality_demo_course_parts/01. Getting Started/` with:

**`part.yaml`**:
```yaml
---
content_type: COURSE_PART
title: Getting Started
---
```

**`01. welcome.md`** — Topic with minimal self-documenting content (heading + 1-2 paragraphs explaining this is inside Part 1).

**`02. what-to-expect.md`** — Topic with minimal content describing what the course covers.

Reference format: RPC Course part.yaml files at `/home/sheena/workspace/lms/first-class-courses/RPC Course/01. Airlaw/part.yaml`

### Task 3: Create Part 2 — "Core Concepts" (topics + quiz)

Create directory `demo_content/functionality_demo_course_parts/02. Core Concepts/` with:

**`part.yaml`**:
```yaml
---
content_type: COURSE_PART

---
```

**`01. key-ideas.md`** — Topic with minimal content.

**`02. going-deeper.md`** — Topic with minimal content.


Verify:

- UUIDs are auto-generated and written back to files
