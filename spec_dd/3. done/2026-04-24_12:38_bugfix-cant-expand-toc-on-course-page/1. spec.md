# Bugfix: TOC expand/collapse broken on course detail pages

## Problem

On course detail pages (e.g. `/courses/functionality-demo-course-parts/2/`), clicking to expand sections in the right-hand sidebar Table of Contents does nothing. The browser console shows:

```
Alpine Expression Error: Undefined variable: toggleExpanded
Expression: "toggleExpanded"
```

The TOC works correctly on the main course home page (e.g. `/courses/functionality-demo-course-parts/`).

## Root Cause

The `toggleExpanded` method is defined in the `coursePart` Alpine.js component in `freedom_ls/student_interface/static/student_interface/js/alpine-components.js`. This script is loaded via the `extra_alpine_components` template block.

- `course_home.html` (the working page) extends `_base.html` and **overrides** `extra_alpine_components` to load the script.
- `_course_base.html` (used by all detail pages: `course_topic.html`, `course_form.html`, `course_form_page.html`) extends `_base_interface.html` -> `_base.html` but **does not** override `extra_alpine_components`, so the Alpine component is never registered.

Both templates render the same TOC partial (`course_minimal_toc.html`) which uses `x-data="coursePart"` and calls `toggleExpanded`, but only the home page loads the JS that defines it.

## Fix

Add the `extra_alpine_components` block override to `_course_base.html` so that all course detail pages load the Alpine components script. This is the correct place because:

1. All course detail templates extend `_course_base.html`
2. `_course_base.html` already loads the TOC via HTMX in its `sidebar_content` block
3. Putting it here avoids duplicating the script tag across each individual detail template

### Files to change

- **`freedom_ls/student_interface/templates/student_interface/_course_base.html`**: Add `extra_alpine_components` block with the script tag for `student_interface/js/alpine-components.js`

## Success Criteria

- Expanding/collapsing TOC sections works on all course detail pages (topics, forms, form pages)
- No `Alpine Expression Error` in the browser console on course detail pages
- TOC expand/collapse continues to work on the course home page
- Expand/collapse state persists in localStorage as before
