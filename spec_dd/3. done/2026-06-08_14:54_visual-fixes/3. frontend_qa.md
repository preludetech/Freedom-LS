# Frontend QA: Course-player visual fixes & header branding

Manual browser QA for the eight visual fixes that shipped on the `visual-fixes` branch
(PR #118): settings-driven header branding (logo / favicon / title), header + player
layout alignment, the HTMX-boosted player navigation (no full-page flash, OOB outline
update, redirect fallback), the new compact `c-button size="small"`, the course-outline
redesign (status-by-icon-colour, row-end chevron, whole-row toggle), and the `first_class`
polish (mono zero-padded counters, prominent course-card grid lines).

---

## 0. Start the dev server

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

If anything looks unstyled, build Tailwind first: `npm run tailwind_build`. The `.btn-sm`
button size and the `theme.css` changes (grid lines, mono counters) only appear after a
fresh Tailwind build.

## 2. Header branding (spec §1)

On any logged-in page (e.g. the dashboard at `http://127.0.0.1:$PORT/`):

- **Logo** — the FirstClass logo image renders in the header bar (left of / in place of the
  title text), vertically centred, ~32px tall.
- **Favicon** — the browser tab shows the FirstClass favicon, not the default Django/globe
  icon.
- **Title** — the header title reads **"FirstClass"** and is rendered **italic** (the inline
  `font-style: italic;` style).
- **Responsive** — narrow the window to ~375px and reload: because a logo is present, the
  `<h1>` title is **hidden** on the small viewport (logo only), and the header doesn't crowd
  or overflow. Widen back to desktop → title returns beside the logo.

> Regression check: nothing about the logo/favicon should bleed into the content area — only
> the header chrome changes.

## 3. Header & player alignment (spec §2)

Open a course in the player (`/courses/<slug>/…` → an item page):

- The sticky **header's** left and right edges line up with the **content column** gutter at
  desktop, tablet (~768px) and mobile (~375px) widths — no off-by-a-gutter step between the
  header padding and the content padding.
- The **breadcrumb header** (inside the content area) sits in the same centred `max-w-7xl`
  well as the content/markdown below it — left edges aligned, not shifted.

## 4. Boosted player navigation — the headline fix (spec §3)

Open a multi-item course and step through it with the **Previous / Next / Finish Course**
buttons:

- **No full-page white flash** on each navigation — only the content column repaints; the
  header, sidebar chrome, and background stay put (it should feel like a single-page app).
- The **URL updates** to the new item on each click. Press the browser **Back** button → you
  return to the previous item (history works). **Forward** works too. The browser tab
  **title updates** to the new item.
- The **sidebar outline updates without a reload**: the **current-item highlight** moves to
  the item you navigated to, and the **progress icon/colour** of the item you just completed
  updates (out-of-band swap). The outline panel's open/scroll state is **preserved** across
  the navigation (it does not snap closed or jump to top).

**Forms:** open a course containing a form.

- Click **"Start Form"** (or **"Continue Form"** if resuming) → the fill-form page swaps in
  and the URL becomes `…/fill_form/…` without a full reload.
- Step through the form's Previous/Next/submit controls (the `c-player-nav` wrapper on
  `course_form.html`) → same no-flash boosted behaviour.

**Fallback (redirect out of the player):** if you can reach a state where a boosted click
redirects **out** of the player — e.g. a **deadline-locked** item (redirects to course
detail) or an **expired session** (clear the session cookie, then click Next → redirects to
login) — confirm the app performs a **full navigation to the real page** (course detail /
login), **not** a blank/empty content swap. This exercises `interface-swap-fallback.js`.

## 5. Compact buttons (spec §4)

- The player nav buttons (Previous / Next / Finish Course) are visibly **small** (tighter
  padding, smaller text) compared to a standard primary button.
- On a topic that embeds an image, the **"Open image"** lightbox trigger is the same compact
  size. **Tab** to it and press **Enter** (keyboard) → the lightbox opens; clicking it opens
  the lightbox too. It still shows its fullscreen icon and label.

---

## 6. Course-outline redesign (spec §5)

In the course player's sidebar outline (open the outline panel if collapsed):

- Each item's **content type-icon is colour-coded by status**: locked items muted/grey,
  completed green, failed red, ready/in-progress in the primary colour. There is **no separate
  trailing status icon** on the right of the row anymore.
- **Screen-reader check** (optional but ideal): with a screen reader / accessibility
  inspector, each item still announces a status word ("Locked", "Not started", "In progress",
  "Completed", "Needs retry") via the `sr-only` text.
- For a course **part** (an expandable row): clicking **anywhere on the row** toggles
  expand/collapse, and the **chevron sits at the right end** of the row (where the old status
  icon was). Expanding reveals the nested children; the nested list no longer has the left
  border rule.

## 7. first_class theme polish (spec §6)

- **Outline counters:** top-level outline items read **`01`, `02`, `03`…** — zero-padded, in
  the mono font. Expand a course part: its child items read **`3.1`, `3.2`…** (the normal
  `parent.child` form), not zero-padded.
- **Course-card grid lines:** on a surface showing course cards (dashboard / all-courses at
  `/courses/`), the accent grid lines on the cards are **clearly visible / prominent** (more
  so than the previous faint 0.07-opacity lines).

---

## 8. Default-theme regression (only if a non-`first_class` site is reachable)

If you can reach a site/configuration using the **default** theme:

- Top-level outline counters stay **`1.` / `1.1`** (not zero-padded, not mono).
- Outline type-icons render normally without the first_class-only treatment leaking in.
- No branding (logo/favicon/title override) appears unless that site sets the header settings.

## Pass criteria

- [ ] Header shows the FirstClass logo, the tab shows the FirstClass favicon, and the title is
      "FirstClass" in italic; the title hides on a ~375px viewport when a logo is present.
- [ ] Header gutter and the player breadcrumb header line up with the content column at all
      widths.
- [ ] Previous/Next/Finish navigation swaps only the content with **no full-page flash**,
      updates the URL (back/forward + title work), and updates the outline highlight/progress
      via OOB swap with the panel state preserved.
- [ ] "Start Form" / "Continue Form" swap in the fill-form page and push a `…/fill_form/…` URL.
- [ ] A boosted click that redirects out of the player (deadline-lock / expired session) falls
      back to a full navigation, landing on the real page — never a blank swap.
- [ ] Player nav buttons and the "Open image" trigger are compact and keyboard-accessible; the
      lightbox opens.
- [ ] Outline conveys status by type-icon colour + `sr-only` word with no trailing status icon;
      a course-part row toggles on whole-row click with the chevron at the right end.
- [ ] first_class top-level counters read `01, 02, 03…` (mono) while child counters stay `3.1`;
      course-card grid lines are clearly prominent.
- [ ] (If checked) the default theme keeps `1.`/`1.1` counters and shows none of the
      first_class-only treatment.
