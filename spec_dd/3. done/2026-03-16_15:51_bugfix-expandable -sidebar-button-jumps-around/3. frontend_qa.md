# Frontend QA: Fix Expandable Sidebar — Button Jumps Around

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- A student user exists and is enrolled in at least one course
- An educator user exists with access to the educator interface
- Test at **mobile** (< 1024px) and **desktop** (>= 1024px) viewport sizes

## Login Credentials

Use the test accounts available in the dev environment. Log in at `/accounts/login/`.

---

## Test 1: Toggle Button Position Stability (Mobile)

**Setup:** Resize browser to < 1024px width (e.g., 375px). Navigate to a course page as a student.

1. Observe the header bar below the site header — it should show:
   - A hamburger/menu icon on the left
   - The course name centered in the bar
2. Note the **exact screen position** of the toggle button
3. Tap the toggle button to open the sidebar
4. The sidebar slides in from the left, overlaying content
5. The close button (X) should be at the **top of the sidebar panel**, NOT in the header bar
6. Tap the close button
7. The toggle button reappears in the **exact same position** as step 2

**Pass criteria:** The toggle button does NOT jump or shift position between open/close states.

---

## Test 2: Toggle Button Position Stability (Desktop)

**Setup:** Resize browser to >= 1024px. Navigate to a course page as a student.

1. If sidebar is open, close it first (clear localStorage or click close)
2. Observe the header bar — toggle button on the left, course name centered
3. Note the toggle button position
4. Click the toggle button to open the sidebar
5. The header bar should **split**: left section (w-64) shows "Table of Contents" + close button, right section shows course name centered
6. The sidebar pushes content to the right (no overlay)
7. Click the close button in the header bar
8. The toggle button returns to the **same position** as step 3

**Pass criteria:** Toggle button position is stable. Header bar splits cleanly on desktop.

---

## Test 3: Mobile Sidebar Overlay Behavior

**Setup:** Mobile viewport (< 1024px). Navigate to a course page.

1. Open the sidebar via the toggle button
2. Verify the sidebar:
   - Slides in from the left with a smooth transition
   - Shows "Table of Contents" heading at the top with a close (X) button
   - Has a width of w-64 (256px)
   - Overlays both the header bar and main content
   - Does NOT cover the site header (top navigation bar with site title)
3. Verify a semi-transparent **backdrop** covers the area to the right of the sidebar
4. Tap the **backdrop** — sidebar should close
5. Reopen the sidebar, then tap the **close button** — sidebar should close
6. Reopen the sidebar, then tap a **navigation link** inside the sidebar — sidebar should auto-close

**Pass criteria:** All three close methods work. Sidebar overlays below site header only.

---

## Test 4: Desktop Sidebar Push Behavior

**Setup:** Desktop viewport (>= 1024px). Navigate to a course page.

1. Open the sidebar
2. Verify:
   - Sidebar appears on the left as a w-64 column
   - Main content is **pushed** to the right (not overlaid)
   - No backdrop is visible
   - Header bar left section shows "Table of Contents" + close button
   - Header bar right section shows course name, centered
3. Close the sidebar
4. Verify main content returns to full width

**Pass criteria:** Content pushes, no overlay/backdrop on desktop.

---

## Test 5: Header Bar Contextual Text — Student Course Pages

**Setup:** Navigate to any course topic page as a student.

1. Verify the header bar shows the **course title** as the centered text
2. Open the sidebar — verify the sidebar heading says "Table of Contents"

**Pass criteria:** Course name appears in header bar, "Table of Contents" in sidebar heading.

---

## Test 6: Header Bar Contextual Text — Educator Interface

**Setup:** Log in as an educator. Navigate to the educator interface.

1. Verify the header bar shows **"Educator Interface"** as the centered text
2. Open the sidebar — verify the sidebar heading says "Navigation"

**Pass criteria:** "Educator Interface" in header bar, "Navigation" in sidebar heading.

---

## Test 7: Sidebar State Persistence

**Setup:** Desktop viewport.

1. Navigate to a course page, open the sidebar
2. Refresh the page — sidebar should still be open
3. Close the sidebar
4. Refresh the page — sidebar should still be closed
5. Repeat on the educator interface (uses a different storage key)

**Pass criteria:** Sidebar state persists across page refreshes via localStorage.

---

## Test 8: Smooth Transitions

1. On **mobile**: open and close the sidebar. Verify the sidebar slides in/out smoothly (not instant).
2. On **desktop**: open and close the sidebar. Verify the sidebar appears/disappears with a smooth transition.
3. On **mobile**: verify the backdrop fades in/out smoothly.

**Pass criteria:** No jarring or instant state changes. Transitions are visually smooth.

---

## Test 9: Responsive Breakpoint Transition

1. Open the sidebar on **desktop** (>= 1024px)
2. Slowly resize the browser window below 1024px
3. The sidebar should transition to mobile overlay behavior
4. Resize back above 1024px — sidebar should return to desktop push behavior

**Pass criteria:** Breakpoint transitions are handled without visual glitches.
