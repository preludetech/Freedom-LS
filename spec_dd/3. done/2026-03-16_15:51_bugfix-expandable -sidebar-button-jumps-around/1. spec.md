# Spec: Fix Expandable Sidebar — Button Jumps Around on Mobile

## Problem

After implementing the expandable sidebar (see `spec_dd/2. in progress/bugfix-expandable-sidebar/`), the open and close buttons appear in different screen locations on mobile. When a user taps the toggle button, the button "teleports" to a new position, creating a jarring experience.

**Root cause:** The toggle button currently lives inside the sidebar column. When the sidebar opens, the column changes width and layout, causing the button to shift position. On mobile this is especially noticeable because the sidebar overlays content as a fixed panel.

## Solution

Add a **dedicated header bar** within `_base_interface.html` that sits between the `_base.html` site header and the main content area. This header bar provides a fixed, predictable location for the sidebar toggle button, eliminating the position jump.

## Header Bar Behavior

### Content

The header bar displays:
- **Toggle button** on the left (hamburger icon when closed, close icon when open)
- **Contextual text** centered in the available space

The contextual text is provided by child templates via a new `{% block header_bar_text %}` block:
- **Student course pages:** The course name
- **Educator interface:** "Educator Interface"

### Sidebar heading

When the sidebar is open, a **sidebar heading** appears at the top of the sidebar panel area. This is provided by child templates via a new `{% block sidebar_heading %}` block:
- **Student course pages:** "Table of Contents"
- **Educator interface:** "Navigation"

## Mobile Behavior (< 1024px)

### Sidebar closed

```
+------------------------------------------+
|  Site Title              [User Menu]     |  <-- _base.html header (always visible)
+------------------------------------------+
| [☰]  Course Name / Educator Interface   |  <-- New header bar (toggle left, text centered)
+------------------------------------------+
|                                          |
|            Main Content                  |
+------------------------------------------+
```

- The header bar shows the toggle button on the left and centered contextual text.
- Main content fills the area below.

### Sidebar open

```
+---------------------------------------------+
|  Site Title              [User Menu]        |  <-- _base.html header (always visible)
+---------------------------------------------+
| Sidebar heading [✕]|/////BACKDROP///////////|
|-------------------|/////////////////////////|
|                   |/////////////////////////|
|   Sidebar         |/////////////////////////|
|   (w-64)          |/////////////////////////|
|   overlays        |/////////////////////////|
|   content         |/////////////////////////|
+---------------------------------------------+
```

- The sidebar slides in from the **left** and overlays **everything below the `_base.html` header** — this includes both the new header bar and the main content area.
- The sidebar heading and close button appear at the top of the sidebar panel.
- A semi-transparent backdrop covers the remaining area to the right.
- Tapping the backdrop or the close button closes the sidebar.
- The centered header bar text (course name etc.) is **not visible** while the sidebar is open on mobile — it is covered by the sidebar and backdrop.

## Desktop Behavior (≥ 1024px)

### Sidebar closed

```
+----------------------------------------------------------+
|  Site Title                              [User Menu]     |
+----------------------------------------------------------+
| [☰]  Course Name / Educator Interface                   |  <-- Header bar
+----------------------------------------------------------+
|                                                          |
|                     Main Content                         |
|                   (full width)                           |
+----------------------------------------------------------+
```

- Same as mobile: toggle button on the left, centered text.

### Sidebar open

```
+----------------------------------------------------------+
|  Site Title                              [User Menu]     |
+----------------------------------------------------------+
| Sidebar heading [✕] | Course Name / Educator Interface   |  <-- Header bar splits
+----------------------+------------------------------------+
|                      |                                    |
|   Sidebar            |         Main Content               |
|   (w-64)             |       (remaining width)            |
|                      |                                    |
|   pushes content --> |                                    |
+----------------------+------------------------------------+
```

- The header bar **splits** into two sections:
  - **Left (w-64):** Sidebar heading text + close button (right-aligned within this section)
  - **Right (remaining width):** Centered contextual text (course name etc.)
- The sidebar pushes the main content to the right (no overlay, no backdrop).
- The sidebar heading and close button sit in the header bar row, visually aligned with the sidebar column below.

## Template Blocks

Child templates need to provide:

| Block | Purpose | Example |
|---|---|---|
| `sidebar_storage_key` | localStorage key (existing) | `sidebar-course-toc` |
| `header_bar_text` | Centered text in header bar (new) | `{{ course.title }}` or `Educator Interface` |
| `sidebar_heading` | Text at top of sidebar panel (new) | `Table of Contents` or `Navigation` |
| `sidebar_content` | Sidebar panel content (existing) | Navigation links, TOC |
| `content` | Main content area (existing) | Page content |

The `sidebar_class` block can be removed or repurposed since the layout is being restructured with the new header bar.

## Alpine.js Component Changes

The `sidebarComponent` in `alpine-components.js` may need minor adjustments:
- The toggle button moves from inside the sidebar column to inside the header bar, but it still calls `toggle()`.
- The `isMobile` / `sidebarOpen` state and localStorage persistence remain unchanged.
- The mobile sidebar panel positioning changes: it must overlay from below the `_base.html` header (not from `inset-y-0`). Use a top offset matching the `_base.html` header height.

## Success Criteria

1. **Toggle button stays in the same screen position** when opening/closing the sidebar on both mobile and desktop — no jumping or teleporting.
2. **Mobile: sidebar overlays content** below the `_base.html` header (covers the new header bar and main content). Backdrop covers the right side.
3. **Mobile: sidebar slides in from the left** with a smooth transition.
4. **Desktop: sidebar pushes content** to the right. Header bar splits to show sidebar heading on the left and contextual text on the right.
5. **Header bar text is contextual:** shows course name on course pages, "Educator Interface" on educator pages.
6. **Sidebar heading is contextual:** "Table of Contents" for courses, "Navigation" for educator.
7. **Existing child templates** (`interface.html`, `_course_base.html`) are updated to provide the new blocks.
8. **Sidebar state persistence** via localStorage continues to work.
9. **Clicking backdrop or close button** closes the sidebar on mobile.
10. **Auto-close on mobile link click** continues to work.
11. **Smooth transitions** on open/close for both mobile and desktop.
