# Implementation Plan: Course-player image spotlight modal

This plan implements the spec in `1. spec.md`. It is a **redesign of existing
functionality** plus a **cross-cutting backdrop refactor** and a **hard rename**
of the `c-picture` `caption` attribute to `title` (+ new `description`).

We follow **TDD**: where a task has a testable surface (the markdown shorthand
rewriter, the sanitiser allow-list, rendered component output), write the failing
test first, then implement. The pure-presentation pieces (CSS utility, dialog
layout, Alpine controller) are verified through the QA process, not unit tests —
per project convention we do **not** assert on CSS classes or styling in tests.

## Existing code this builds on (keep DRY — reuse, don't reinvent)

- **`sidePanel` native-dialog precedent** —
  `freedom_ls/base/static/base/js/alpine-components.js`. Its `init()` wires a
  `dialog.addEventListener("close", …)` to restore focus to the trigger, and a
  `dialog.addEventListener("click", …)` that closes when
  `event.target === this.dialog` (lines 365–371). The new lightbox controller
  mirrors this **click-target guard** and nothing more — `showModal()` gives us
  focus-trap / Escape / focus-restore for free, so we do **not** re-implement
  them.
- **Side-panel `<dialog>` transition pattern** — `tailwind.base_interface.css`
  lines 40–86 (`@starting-style`, `transition … overlay/display allow-discrete`).
  The spotlight's enter/leave CSS copies this shape, but lives in
  `tailwind.components.css` (reusable home), because `tailwind.base_interface.css`
  is documented as single-use for the one side panel.
- **`c-modal` close-button styling** —
  `freedom_ls/base/templates/cotton/modal.html` lines 60–65
  (`text-muted hover:text-on-surface focus:outline-none focus:ring-2
  focus:ring-primary rounded-md`, `size-6` `c-icon name="close"`, `sr-only`
  label). The spotlight close button reuses this exact treatment instead of the
  current `size-12` opacity hack.
- **`c-media-card`** — `freedom_ls/base/templates/cotton/media-card.html`. The
  inline thumbnail card stays as-is; only its `footer` caption text source
  changes from `caption` to `title`.
- **`get_file_by_path` content tag** — already used at `picture.html` line 18 to
  resolve `src`. Unchanged.
- **`fullscreen` icon** — confirmed valid in the spec (registered in
  `freedom_ls/icons/semantic_names.py` / `mappings.py`). The trigger icon does
  **not** change. The close button uses `c-icon name="close"`.
- **`markdown_translate`** rewriter —
  `freedom_ls/content_engine/management/commands/content_save.py` lines 305–326.
  Two regex substitutions emit `<c-picture …>`; the captioned branch (line 316)

  visible title) and `description` (longer text, spotlight only), and that `alt`

### 3b. Thumbnail card (footer) — swap `caption` → `title`
The footer markup (lines 28–40) keeps its current behaviour; only the variable

```django
{% if number %}<span class="font-semibold">Figure {{ number }}</span>{% if title %}: {% endif %}{% endif %}{{ title }}
```

Trigger button, `x-ref="trigger"`, `@click`, and the `fullscreen` icon are

Task 4 — `open` instead of `show`).

### 3c. Replace the Alpine `x-show` overlay with a native modal `<dialog>`
Delete the `<div x-show="open" …>` overlay (lines 44–77) and replace with a
`<dialog>` opened by the new controller. Structure follows `image.png`:

```django
<dialog x-ref="dialog"
        class="modal-backdrop-host spotlight-dialog ..."  {# bg-transparent, fills viewport #}
        @click="onBackdropClick"
        {% if title %}aria-labelledby="{{ heading_id }}"{% else %}aria-label="{{ alt }}"{% endif %}>

  {# top-left: figure number + title heading. Chrome text sits on the scrim, so #}
  {# use a fixed light colour via plain utilities (e.g. text-white / text-white/90) #}
  {# — NOT a made-up class and NOT a theme token like text-on-surface. #}
  {% if title %}
    <h2 id="{{ heading_id }}" class="text-white ...">
      {% if number %}Figure {{ number }} — {% endif %}{{ title }}
    </h2>
  {% endif %}

  {# top-right: close button — reuse c-modal close treatment #}
  <button type="button" autofocus aria-label="Close image"
          @click="close"
          class="text-muted hover:text-on-surface focus:outline-none focus:ring-2 focus:ring-primary rounded-md ...">
    <c-icon name="close" class="size-6" />
  </button>

  {# centre: the spotlight card holding the image #}
  <div class="spotlight-card bg-surface rounded-lg shadow-xl ..."> {# does NOT close on click #}
    <img src="{{ file_obj.file.url }}"
         alt="{{ alt }}"
         {% if description %}aria-describedby="{{ desc_id }}"{% endif %}
         class="w-full max-h-[...] object-contain" />
  </div>

  {# bottom-left: title + description (no number prefix), fixed light text on scrim #}
  {% if title or description %}
    <div class="text-white ...">
      {% if title %}<p class="font-semibold">{{ title }}</p>{% endif %}
      {% if description %}<p id="{{ desc_id }}" class="text-sm text-white/80">{{ description }}</p>{% endif %}
    </div>
  {% endif %}
</dialog>
```

Key requirements (from spec §6.3, §8, §9):
- **Two heading IDs** are needed (`aria-labelledby` for the top heading, and the
  `aria-describedby` target on the description). Generate stable per-instance ids
  the same way `c-modal` does — `{% unique_id "spotlight-title" as heading_id %}`
  / `{% unique_id "spotlight-desc" as desc_id %}` (the `unique_id` tag `c-modal`
  already uses; reuse it, don't hand-roll ids).
- **Title appears in both** top-left (with "Figure N" prefix) and bottom (no
  prefix) — intentional, per spec.
- **Empty-state degradation:** when `title` is empty, omit the heading box and use
  `aria-label="{{ alt }}"`; when `description` is empty, omit the bottom
  description line.
- **Chrome text** sits on the scrim → use a fixed light colour (theme-neutral),
  **not** `text-on-surface` (spec §8). The card interior uses semantic tokens
  (`bg-surface`, `text-on-surface`, `text-muted`, `border-border`).
- **Scrim** is carried by `::backdrop` via the `modal-backdrop-host` class
  (Task 5). The dialog box itself is `bg-transparent` and fills the viewport so
  backdrop clicks land on the dialog element (enables the click-target guard).
- The error branch (lines 80–86, "Image not found") is unchanged.

## Task 4 — Replace `contentLightbox` with a minimal dialog controller

**File:**
`freedom_ls/content_engine/static/content_engine/js/alpine-components.js`
(CSP build — register via `Alpine.data()`, no inline expressions).

**Delete** the entire current `contentLightbox` (lines 52–77) with its `open`
state, `_trigger` capture/restore, `onEscape`, and `onTab` focus trap — all now
provided natively by `showModal()`. The `equation` component is unrelated and
stays.

**Replace with** (mirrors `sidePanel`'s click-target guard, nothing more):
```js
// Content lightbox — native <dialog> spotlight (cotton/picture.html).
// showModal() provides focus-trap, Escape, inert background and focus-restore
// to the trigger; we only add open() and a backdrop-click-to-close guard.
Alpine.data("contentLightbox", () => ({
    open() {
        this.$refs.dialog.showModal();
    },
    close() {
        this.$refs.dialog.close();
    },
    onBackdropClick(event) {
        // The full-viewport dialog reports event.target === the dialog only when
        // the surrounding scrim is clicked; the card/image are descendants and
        // are correctly ignored. Mirrors the sidePanel mobile handler.
        if (event.target === this.$refs.dialog) this.close();
    },
}));
```

Update the file's top doc-comment (lines 8–11) which currently says a later task
"adds `contentLightbox`" — describe the native-dialog controller instead.

> The component method is named `open()`; the thumbnail trigger's `@click` becomes
> `open` (Task 3b). Focus-restore-to-trigger is native — no `_trigger` ref. If the
> design wants focus on the close button at open, the `autofocus` attribute on the
> close button (Task 3c) handles it without JS.

## Task 5 — Shared `.modal-backdrop` utility + spotlight dialog CSS

**File:** `tailwind.components.css` (`@layer components`)

### 5a. Shared backdrop utility (single source of truth, spec §7.2)
```css
/* Modal scrim — one look for every modal surface. Two hooks because the scrim is
   rendered two ways: a real overlay <div> (Alpine modals) vs the ::backdrop pseudo
   of a native <dialog>, whose own box stays bg-transparent. Carries only the scrim
   look; positioning / z-index / transitions stay inline on each element. */
.modal-backdrop,                 /* overlay <div>: Alpine modals + lightbox */
.modal-backdrop-host::backdrop { /* native <dialog>: styles its ::backdrop  */
    @apply bg-black/50 backdrop-blur-sm;
}
```

### 5b. Spotlight dialog transition CSS (copy the side-panel pattern, spec §9.1)
Add `.spotlight-dialog` rules in the same file: full-viewport flex layout for the
chrome, `@starting-style` + `transition … overlay/display allow-discrete` for
enter/leave, scale+fade on open. Under `@media (prefers-reduced-motion: reduce)`
drop the scale (fade only); do not animate the backdrop blur. Card uses
`max-height: ~90dvh` with internal scroll for long descriptions (spec §9.2).

> No `z-index` — a modal `<dialog>` renders in the top layer. Blur is standardised
> at `backdrop-blur-sm`; if the spotlight wants more it layers an extra blur inline
> rather than forking the utility.
