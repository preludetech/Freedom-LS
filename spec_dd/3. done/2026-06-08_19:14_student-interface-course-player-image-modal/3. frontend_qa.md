# Frontend QA: Course-player image spotlight modal

Manual browser QA for the redesigned image lightbox (now a native modal
`<dialog>` "spotlight"), the `caption` → `title` rename plus new `description`,
and the shared `.modal-backdrop` scrim applied across all three modal surfaces.

---

## 0. Start the dev server

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
npm run tailwind_build          # required — this feature adds new CSS utilities
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

If anything looks unstyled, re-run `npm run tailwind_build`.

## 2. Open the spotlight (success criteria 1, 2)

Navigate into the Content Widgets course → **Media** topic (course player,
`…/courses/<slug>/items/<item>/…`). Scroll to the **Pictures** section.

- Each image shows as an inline **thumbnail card** with a figcaption and an
  **"Open image"** button (unchanged behaviour). The figcaption text is the
  `title` (e.g. "Figure 1: A captioned, numbered figure…").
- Click **"Open image"** on the numbered figure (Figure 1).

**Expect the spotlight:**
- A **dark, blurred scrim** over the whole page — the page does **not** bleed
  through (this is the headline change; the old version only blurred).
- A centred **image card** (rounded, soft shadow, image scaled with
  `object-contain`), with visible scrim margin around it.
- **Top-left:** "Figure 1 — &lt;title&gt;" heading.
- **Top-right:** a clean **close (X)** button (padded, always visible — not a
  faint oversized corner icon).
- **Bottom-left:** the title again (no "Figure N" prefix) and, if present, the
  **description** beneath it.
- In `first_class` theme it should match `image.png`; in `default` it should look
  clean and sensible.

## 3. Empty-state degradation (success criterion 2)

Some Media-topic figures have a `title` but **no** `description`; the grid/gallery
images have short titles. To check the no-title path you can temporarily set a
`c-picture` to `alt="…"` only (no `title`) and reload content.

- **No description:** the bottom block shows the title only — **no** empty
  description line, no empty box.
- **No title:** **no** heading box top-left and **no** bottom title block; the
  dialog still opens and is still announced (its accessible name falls back to the
  `alt` text — verify with a screen reader or by inspecting `aria-label`).

## 4. Three close mechanisms + card/image must NOT close (success criterion 4)

With the spotlight open, verify **all three**:

1. **Close button** (top-right) → closes.
2. **Escape** key → closes.
3. **Click-outside** — click the dark scrim area (outside the white card) →
   closes.

Then the negative cases — these must **NOT** close the dialog:

- Click **on the image** → stays open.
- Click **on the white card** (e.g. its padding) → stays open.

## 5. Keyboard & screen-reader (success criterion 5)

- Open the spotlight with the keyboard: `Tab` to the "Open image" button, press
  `Enter`. The spotlight opens and focus moves **into** the dialog (the close
  button has `autofocus`).
- Press `Tab` repeatedly: focus stays **trapped** inside the dialog (it cycles
  through the dialog's focusables and never reaches the page behind). `Shift+Tab`
  wraps the other way.
- Press `Escape`: closes, and focus returns to the **"Open image"** trigger you
  came from.
- The close button has an accessible name (**"Close image"**) and a comfortable
  hit target (~44×44px) — check it is easy to click, with a visible focus ring on
  `Tab`.
- With a screen reader (VoiceOver/NVDA): on open it announces a dialog labelled by
  the figure heading (or the `alt` text when there is no title). If the image has
  a `description`, the image is described by it (`aria-describedby`).

## 6. Reduced motion (success criterion 6)

Enable "Reduce motion" in the OS (or DevTools → Rendering → emulate
`prefers-reduced-motion: reduce`). Open the spotlight.

- **Expect:** it **fades** in only — **no** scale/zoom animation. The backdrop
  blur does not animate either. Closing is likewise a plain fade.
