Key changes from current template:
- Remove the absolute-positioned open button (lines 10-18)
- Remove the close button inside the sidebar panel (lines 46-53)
- Add a persistent sidebar column (`shrink-0`) that always renders
- Single toggle button inside the column with icon toggling via `x-show` on wrapper spans
- On desktop: sidebar column gets `w-64` when open, auto-shrinks when closed вҖ” main content compresses/expands via flex
- On mobile: sidebar panel is fixed overlay (existing behavior), toggle button stays in normal flow
- Add `x-on:click="handleSidebarClick"` on sidebar panel for auto-close on mobile link clicks
- Mobile panel gets `pt-14` to avoid overlapping the toggle button

### Step 2: Add `handleSidebarClick` method to Alpine component

**File:** `freedom_ls/base/static/base/js/alpine-components.js`

Add a new method to the `sidebarComponent` definition:

```javascript
handleSidebarClick(event) {
    if (this.isMobile && event.target.closest("a")) {
        this.closeSidebar();
    }
},
```

This uses event delegation вҖ” any click on an `<a>` element inside the sidebar panel (including HTMX-loaded content) will close the sidebar on mobile. On desktop, it does nothing.
