# Frontend QA: Phase 3 — First Class Tier-2 component-class overrides

## What we're verifying

1. Under `FLS_THEME=first_class`, the three overridden components render with the Tier-2 tweaks: `btn-primary` rounder + heavier shadow, `chip` flatter, `btn-outline` thicker border.
2. Under `FLS_THEME=default`, those same components are unchanged from the Phase-1 baseline — the overrides are scoped to the active theme.
3. The default theme's `theme.css` remains tokens-only (sparse principle).
4. Phases 1 + 2 deliverables remain green.

We use the **DemoDev** site for all test data.

## Test 2 — Tier-2 overrides under first_class

(Continuing with `FLS_THEME=first_class` from above.)

Find pages that render each overridden class. Suggestions:

- `btn-primary`: login button, course-detail CTAs, "Mark complete" buttons.
- `chip`: progress dashboards, cohort detail, content-topic chip annotations.
- `btn-outline`: secondary CTAs (cancel, back) — typically alongside a primary button.

For each, take a screenshot via Playwright MCP and confirm:

- **`btn-primary`** — visibly rounder corners than under default (compare to the Test 1.1 screenshot). Shadow is heavier (more prominent drop shadow). Colour and label unchanged.
- **`chip`** — no shadow visible; chip looks flatter against the background. Pill shape and colour band unchanged.
- **`btn-outline`** — border visibly thicker than under default. Border colour and shape otherwise unchanged.

**Things that must NOT have changed:**

- Other component classes (e.g. `chip-success`, `btn-success`, `surface`) — Tier-2 overrides were scoped to the three demo classes.
- Cotton template markup — no Tier-3 overrides land until Phase 4.
- Any layout / structure — we re-shaped three classes, we did not restructure.

## Test 3 — Default theme: tweaks must NOT bleed across

Stop the server. Then:

```bash
FLS_THEME=default npm run tailwind_build
FLS_THEME=default uv run python manage.py runserver $PORT
```

Re-visit the same pages. Confirm:

- `btn-primary` is back to default radius / shadow.
- `chip` has its default elevation back.
- `btn-outline` has its default border thickness back.

If anything looks like the first_class shape under default, the cascade order in `tailwind.input.css` is wrong — investigate Task 1.

## Test 4 — Sparse-default invariant

Confirm the default theme's `theme.css` did not gain a component layer:

```bash
grep -n "@layer components" "freedom_ls/themes/default/static/themes/default/theme.css"
```

Expected: no matches. If anything matched, the sparse principle has been broken — the default theme should be tokens-only.

Also confirm `tailwind.components.css` was not edited structurally:

```bash
git diff main -- tailwind.components.css
```

Expected: at most no diff, or only the Phase-1 token-rename + `@theme` removal already on this branch. No new component classes added to `tailwind.components.css` for the Tier-2 demo.
