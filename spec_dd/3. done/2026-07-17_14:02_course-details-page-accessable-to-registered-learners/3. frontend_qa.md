# Frontend QA: "Details" entry points to the course details page

Manual browser QA for the "Details" link on dashboard cards, all-courses rows, and the
repointed player breadcrumb. Everything here is verified in a browser — no code.

## 0. Setup

Seed QA data on the **DemoDev** site (login convention: **password == email address**):

```bash
uv run python manage.py qa_create_rich_dashboard_student
uv run python manage.py qa_create_course_visibility
```

- `qa_create_rich_dashboard_student` → student **`demodev_s1@email.com`** with an
  In-progress course, a Completed course, and a Recommended (not-registered) course on
  the dashboard.
- `qa_create_course_visibility` → adds a **coming-soon** course (and a published-free
  course) visible on the all-courses page, plus a login-ready student on DemoDev.

Start the dev server on a random port:

```bash
PORT=$(.claude/fls/scripts/find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL below is `http://127.0.0.1:$PORT` (substitute the actual port printed).

Log in:

1. Visit `http://127.0.0.1:$PORT/accounts/login/`.
2. Email `demodev_s1@email.com`, password `demodev_s1@email.com`. Submit.
3. You land on the dashboard (`/`).

## Test 1 — Dashboard course cards (all states)

**Where:** `http://127.0.0.1:$PORT/` (the dashboard).

You should see cards across the **In progress**, **Completed**, and **Recommended**
(not-registered) sections.

For **each** card:

1. Confirm a low-emphasis **"Details"** text link appears in the card's content area,
   **bottom-right on its own line**, with a small right-pointing chevron after the word.
2. Confirm it sits **below the hero image** and **clear of the progress bar / "Completed"
   badge** — it does not overlap them.
3. **Click the "Details" link** → you land on the course **details / overview page**
   (URL ends in `/courses/<slug>/detail/`), showing the description, "what you'll learn",
   and the content table of contents.
4. Go **Back**. Now click **elsewhere on the same card** (the title or hero, not the
   Details link) → this still takes you into the **course player / resume point** (or, for
   a not-registered card, also the details page). The two click targets are independent.

Expected: every card has exactly one "Details" link, always in the same corner, and it is
independently clickable from the whole-card link.

**Keyboard check:** put focus in the page and press **Tab** repeatedly. The "Details" link
must receive a visible focus ring and activate with **Enter**, independently of the
card's main link.

## Test 2 — All-courses rows (all states)

**Where:** `http://127.0.0.1:$PORT/courses/`.

This flat list shows row states: **Registered / In progress**, **Completed**,
**Not registered**, and **Coming soon** (from the visibility QA command).

For **each** row:

1. Confirm a **"Details"** link with trailing chevron appears **inline, right-aligned
   beside the course title** (not on a separate footer line — rows are wider than cards).
2. On the **Coming soon** row, confirm the "Details" link coexists cleanly with the
   existing "Coming soon" chip on the right (they don't overlap or wrap awkwardly).
3. **Click "Details"** → lands on that course's `/courses/<slug>/detail/` page.
4. Go Back, click the **row title / body** (not Details) → still goes to the row's normal
   destination (player for registered, finish page for completed, details page for
   not-registered/coming-soon).

**Anonymous + unregistered check:**

1. Log out (`/accounts/logout/`) — or open a private window.
2. Visit `http://127.0.0.1:$PORT/courses/` while **anonymous**. Confirm the "Details" link
   still renders on every visible row and navigates to the details page.
3. (Optional) Log in as a different student who is **not registered** for these courses
   and confirm the same.

## Test 3 — Player first breadcrumb repoint

**Where:** open any course you are registered for and enter the **player**.

1. From the dashboard, click a **registered / in-progress** card to enter the course
   player (a lesson page, URL like `/courses/<slug>/<index>/`).
2. Look at the breadcrumb trail at the top: `Course title  >  [part]  >  current item`.
3. **Click the first breadcrumb** (the **course title**).
4. Expected: you land on the course **details / overview page**
   (`/courses/<slug>/detail/`) — **not** on lesson 1 / "start over".
5. Confirm the browser address bar now shows the details-page URL. This URL is the
   shareable link — copying the address bar is the intended "share" path.

---

## What "pass" looks like

- Every dashboard card (all states) and every all-courses row (all states) has one
  consistently-placed, keyboard-accessible "Details" link that navigates to
  `/courses/<slug>/detail/`.
- The Details link is clickable independently of the whole-card / whole-row link.
- The player's first breadcrumb now lands on the details page, not lesson one.
- Nothing on the details page itself changed.
