## Bug 1 ‚Äî Course player content renders in the wrong column, then jumps

### Root cause (confirmed)
`freedom_ls/base/templates/_base_interface.html:30` ships the `panelGrid` div with
`data-panel-open="true"`, so the browser applies the two-column grid template on
first paint. The sidebar is a native `<dialog>` (`display:none` until Alpine's
`sidePanel.init()` calls `dialog.show()`), so on cold paint the only participating
grid child ‚Äî `#interface-main` ‚Äî is placed in **track 1 (the narrow sidebar
column)** and visibly jumps to track 2 when the dialog opens.

`sidePanel._syncGrid()` (`freedom_ls/base/static/base/js/alpine-components.js:462`)
already writes the correct `data-panel-open` value during `init()`, so flipping
the static default is safe ‚Äî Alpine remains the source of truth after init.

### Change
In `freedom_ls/base/templates/_base_interface.html` (the `panelGrid` div, ~line 30):

```diff
 <div x-ref="panelGrid"
-     data-panel-open="true"
+     data-panel-open="false"
      class="side-panel-grid grid lg:gap-12 relative {% block sidebar_class %}{% endblock sidebar_class %}">
```

With a single-column template on first paint, the content renders full-width in
its correct horizontal position. When Alpine opens the dialog and `_syncGrid()`
runs, the second column appears with the sidebar already in it ‚Äî no content
reflow into the sidebar track.

### Out of scope (do not touch)
The HTMX-boosted Previous/Next nav (`c-player-nav`, `hx-select-oob="#course-toc-region"`)
already swaps content + outline together and is not the cause.

### Tests
None. Presentational ‚Äî verified visually (see QA, run separately). A template
default-value snapshot is **not** required and adds little value here.

## Bug 2 ‚Äî Excess whitespace under the course-player header

### Root cause (confirmed)
Double-spacing below the header's bottom border:
1. The shared header in `_base_interface.html` (~line 80,
   `max-w-7xl mx-auto pt-4 pb-4 lg:border-b lg:border-border`) owns `pb-4` (16px).
2. The content wrapper `<c-page>` (`freedom_ls/base/templates/cotton/page.html:10`)
   unconditionally adds `py-8 lg:py-12` (32px mobile / 48px desktop) of **top**
   padding.

`cotton/page.html` renders `{{ class }}` on the content-well div, and `flush`
only drops horizontal padding (not vertical) ‚Äî so a pass-through `class` is the
supported, scoped mechanism. Precedent: `course_detail.html:36` already trims the
same default with `<c-page class="pt-2 lg:pt-2">`.

### Change (scope: course player only)
Add a top-padding override to the `<c-page flush="true">` tag in **each**
course-player template that renders under `_base_interface.html`. Confirmed full
set:

| Finish | `freedom_ls/student_interface/templates/student_interface/course_finish.html` | `<c-page width="narrow" flush="true">` (line 6) |

Add a `class` attribute reducing the top padding, e.g.:

```html

```

(`course_form_complete.html` / `course_finish.html` keep `width="narrow"`:
`<c-page width="narrow" flush="true" class="pt-2 lg:pt-4">`.)

## Bug 3 ‚Äî "Save and add another" does not refresh the cohorts table

### Root cause (confirmed)
"Save and add another" (`CreateInstanceAction.form_valid`,
`freedom_ls/panel_framework/actions.py:162`) saves, re-renders the empty modal
form, and sets `HX-Trigger: <created-event>` (cohorts ‚Üí `cohortCreated`, from
`get_created_event_name()`). **The event fires on `document.body` but nothing
listens**, so the table never re-fetches. (Plain "Save" returns 204 +
`HX-Redirect` and navigates away, so it never needed a refresh.)

Two missing pieces:
1. **No client-side listener** for the created-event.
2. **The list-view render path returns more than the table.**
   `_render_list_view_content` (`freedom_ls/panel_framework/views.py:371`) always
   prepends `actions_html` (create button + modal) above the table and has **no
   `HX-Target` short-circuit** ‚Äî unlike `DataTablePanel.render`
   (`panels.py:69-75`), which returns only the table content when
   `HX-Target == table_id`. So a targeted refresh against `base_url` would swap in
   `actions_html + table_html` and duplicate the create button/modal.

### Fix (scope: generic panel framework ‚Äî fixes cohorts and every other list view)
Two pieces, mirroring the existing `panelChanged`/`tabContainer` convention.

The list-view `table_id` is the `DataTable.render` default `"data-table-container"`
(`tables.py:67`; `ListViewConfig.render` at `views.py:188` does not pass a
`table_id`). `base_url` is `request.path` (`views.py:536`).

#### Piece 1 ‚Äî Server-side short-circuit (`views.py`)
Make `_render_list_view_content` return **only** the table HTML when the request
targets the table, mirroring `DataTablePanel.render`:

```python
# freedom_ls/panel_framework/views.py
def _render_list_view_content(
    request: HttpRequest, list_config: type[ListViewConfig], base_url: str
) -> str:
    """Render a list view with its actions."""
    table_id = "data-table-container"  # ListViewConfig.render uses DataTable's default
    # Targeted table refresh (e.g. event-driven re-fetch): return only the table,
    # mirroring DataTablePanel.render ‚Äî short-circuit BEFORE any actions work,
    # avoids duplicating the create button/modal.
    if request.headers.get("HX-Target") == table_id:
        return list_config.render(request, base_url=base_url)

    list_actions = [
        action
        for action in list_config.get_actions(request)
        if action.has_permission(request)
    ]
    if not list_actions:
        return list_config.render(request, base_url=base_url)

    rendered_actions = [
        action.render(request, None, base_url) for action in list_actions
    ]
    created_events = [
        action.get_created_event_name()
        for action in list_actions
        if isinstance(action, CreateInstanceAction)
    ]
    actions_html = render_to_string(
        "panel_framework/partials/instance_actions.html",
        {"actions": rendered_actions},
        request=request,
    )
    table_html = list_config.render(request, base_url=base_url)
    if created_events:
        actions_html = render_to_string(
            "panel_framework/partials/list_refresh.html",
            {
                "actions_html": actions_html,
                # Space-separated event names ‚Äî simple identifiers, no JSON
                # escaping needed in the data-* attribute.
                "refresh_events": " ".join(created_events),
                "refresh_url": base_url,
                "refresh_target": table_id,
            },
            request=request,
        )
    return actions_html + table_html
```

- Extend the existing actions import to
  `from freedom_ls.panel_framework.actions import CreateInstanceAction, PanelAction`
  in `views.py` (currently imports only `PanelAction`, line 10). No `json` import
  is needed ‚Äî event names are passed as a space-separated string, not JSON.
- The short-circuit returns before any action-list work, and `list_config.render()`
  is called exactly once per code path (mirrors `DataTablePanel.render`).
- **Double-listener safety:** on full panel navigation, the entire `main-content`
  region is replaced, so the `listRefresh` wrapper is not accumulated ‚Äî a fresh
  GET renders exactly one wrapper (asserted in test 2 below). When Alpine removes
  the old wrapper element, its `destroy()` removes the old body listeners.
- **Do not** add the listener wiring to `instance_actions.html` ‚Äî that partial is
  shared with instance-view actions (`views.py:84`, used at 98 and 129). Wrap it
  with a dedicated `list_refresh.html` partial (below) only on the list path. The
  wrapper lives **outside** the `#data-table-container` div, so it survives the
  `outerHTML` table swap and the listener is not torn down/re-registered on each
  refresh.

#### Piece 2 ‚Äî Client-side listener

New partial `freedom_ls/panel_framework/templates/panel_framework/partials/list_refresh.html`:

```html
{% comment %}
Registers body listeners that re-fetch the list table when a create action
fires its created-event (HX-Trigger). Lives outside #data-table-container so
it survives the table's outerHTML swap.
{% endcomment %}
<div x-data="listRefresh"
     data-refresh-events="{{ refresh_events }}"
     data-refresh-url="{{ refresh_url }}"
     data-refresh-target="{{ refresh_target }}">
    {{ actions_html|safe }}
</div>
```

(`actions_html` is a pre-rendered HTML string from `render_to_string`, so `|safe`
is intentional and correct. `refresh_events` is a plain space-separated string of
identifiers ‚Äî no `|safe` and no JSON, so auto-escaping is a non-issue.)

New Alpine component in
`freedom_ls/panel_framework/static/panel_framework/js/alpine-components.js`
(register inside the existing `alpine:init` listener, mirroring `tabContainer` ‚Äî
body listener + `htmx.ajax` + `destroy()` cleanup; CSP-compatible, no inline
expressions):

```javascript
// List-view table auto-refresh (panel_framework/partials/list_refresh.html).
// Re-fetches the list table when a create action's HX-Trigger event fires.
Alpine.data("listRefresh", () => ({
    _handlers: [],
    init() {
        const url = this.$el.dataset.refreshUrl;
        const target = this.$el.dataset.refreshTarget;
        const events = (this.$el.dataset.refreshEvents || "")
            .split(/\s+/)
            .filter(Boolean);
        events.forEach((eventName) => {
            const handler = () => {
                htmx.ajax("GET", url, {
                    target: "#" + target,
                    swap: "outerHTML",
                });
            };
            document.body.addEventListener(eventName, handler);
            this._handlers.push([eventName, handler]);
        });
    },
    destroy() {
        this._handlers.forEach(([eventName, handler]) => {
            document.body.removeEventListener(eventName, handler);
        });
    },
}));
```

#### Flow after the fix
1. Educator clicks "Save and add another" ‚Üí server saves, returns empty form +
   `HX-Trigger: cohortCreated`.
2. htmx fires `cohortCreated` on `document.body`.
3. `listRefresh` handler calls
   `htmx.ajax("GET", base_url, {target: "#data-table-container", swap: "outerHTML"})`.
   htmx sets `HX-Target: data-table-container`.
4. Piece 1's short-circuit returns **only** the new table HTML ‚Üí swapped into
   `#data-table-container`. The create button/modal (outside that div) is
   untouched and not duplicated.
