touches `pyproject.toml`, FLS's own test modules across several apps, a new

decision is made and recorded upstream of the guards. Do Task 0 (record verdicts +
create the debt spec), commit it on its own, *then* proceed to
markers/collection-safety/de-brand.

Follow **TDD** throughout (Red → Green → Refactor). Where a task is a pure
de-brand of an existing assertion, the "test" is the rewritten assertion itself —

## Task 2 — Layer 1b: collection-safety guards for optional-app importers

Six modules import `freedom_ls.course_applications` factories/models at module
scope with no guard (spec §1). When a downstream omits the optional app, the
model-registry `RuntimeError` fires at **collection** time and aborts the whole
session. Guard each so an absent app degrades to a skip.

### 2a. Colocated `collect_ignore_glob` conftest (belt)

Create `freedom_ls/course_applications/tests/conftest.py` exactly as spec §4b:

```python
from django.conf import settings

collect_ignore_glob: list[str] = []
if "freedom_ls.course_applications" not in settings.INSTALLED_APPS:
    collect_ignore_glob = ["test_*.py"]
```

`collect_ignore*` is scoped to the conftest's own directory, so this only guards
the four `course_applications/tests/test_*.py` files. The two cross-app importers
that live in other apps' test dirs are handled by their per-module guards (2b).

### 2b. Per-module `INSTALLED_APPS` skip guard (braces)

Insert the guard **above the offending import**, in the same module, in each of:

- `freedom_ls/course_applications/tests/test_backends.py` (import at line 12)
- `freedom_ls/course_applications/tests/test_models.py` (line 11)
- `freedom_ls/course_applications/tests/test_queries.py` (line 11)
- `freedom_ls/course_applications/tests/test_views.py` (lines 12–13, guards both the
  factory and model import)
- `freedom_ls/student_interface/tests/test_course_access_integration.py` (line 27)
- `freedom_ls/course_access/tests/test_visibility_enforcing_backend.py` (line 19)
  — **only if Task 0 kept the dependency** rather than removing it (see Task 3).

Guard form (spec §4b), placed after `from __future__ import annotations` and
`import pytest`, before the first `course_applications` import:

```python
from django.conf import settings

if "freedom_ls.course_applications" not in settings.INSTALLED_APPS:
    pytest.skip("course_applications not installed", allow_module_level=True)

from freedom_ls.course_applications.factories import CourseApplicationFactory  # now safe
```

Notes:
- Keep the guard **inline and verbatim** in each file rather than extracting a
  `skip_unless_installed(...)` helper. The spec §4b form is the exact pattern Task 5
  documents in the testing skill; a helper would make the documented convention and
  the code diverge, and the 3-line block must sit immediately above the import it
  guards. This is a deliberate, spec-sanctioned exception to "avoid repeating code".
- The four `course_applications/tests/*` files get **both** the conftest ignore
  and the module guard (defense in depth — spec §4b). The two cross-app files
  rely on the module guard alone; their sibling tests must still collect, so a
  directory-level ignore is wrong there.
- Do not use `pytest.importorskip` (the package is importable; the failure is the
  model-registry `RuntimeError`) and do not use `@pytest.mark.skipif` (the
  module-scope import raises before the decorator is reached) — spec §4b.

### 2c. Automated collection-safety verification (drives criterion 2, TDD-first)

Success criterion 2 must be proven automatically, not by eyeballing. Django's app
registry is populated once per process, so proving collection survives a different
`INSTALLED_APPS` genuinely requires a **fresh subprocess**. Add one test that runs
pytest **collect-only** in a child process against a settings module whose
`INSTALLED_APPS` omits `course_applications`, and asserts collection does not
abort. Write it **first** — it fails today (returncode 2, "errors during
collection") and passes once 2a/2b land.

Suggested location: `freedom_ls/course_applications/tests/test_collection_safety.py`.
This file itself must NOT import `course_applications` at module scope. Run it from
FLS's own suite where the app IS present.

**Preferred mechanism: pytest's built-in `pytester` fixture** (enable via
`pytest_plugins = ["pytester"]` in the module or the app's conftest), using
`pytester.runpytest_subprocess(...)`. It runs a fresh, isolated child process and
**auto-isolates coverage, cache, and cwd** — which removes the repo-pollution risk
below by construction, and reads far more simply than raw `subprocess`.

**MANDATORY for the child run, whichever mechanism is used:** disable the inherited
`--cov*` and pytest-playwright `--tracing/--screenshot` addopts so the child does
not (a) write/overwrite the repo `.coverage` file while the parent's own `--cov`
session is live — corrupting the parent's `--cov-fail-under=73` accounting — or
(b) create an ungitignored `test-results/` trace dir in the repo root. Pass
`-p no:cov -p no:playwright -p no:cacheprovider`, **or** neutralise everything with
`-o addopts=""` and re-specify only `--strict-markers`. These are **not optional**.

Pseudocode (pytester form):

```python
pytest_plugins = ["pytester"]

def test_collection_survives_without_course_applications(pytester):
    # Throwaway settings written into pytester's isolated dir — never the repo tree.
    pytester.makepyfile(settings_no_ca=(
        "from config.settings_dev import *\n"
        "INSTALLED_APPS = [a for a in INSTALLED_APPS "
        "if a != 'freedom_ls.course_applications']\n"
    ))
    result = pytester.runpytest_subprocess(
        "--co", "-q",
        "-p", "no:cov", "-p", "no:playwright", "-p", "no:cacheprovider",
        "-o", "addopts=",              # drop inherited -m/--cov/--tracing/... entirely
        f"--rootdir={REPO_ROOT}",
        str(REPO_ROOT / "freedom_ls" / "course_applications"),
        str(REPO_ROOT / "freedom_ls" / "student_interface"),
        str(REPO_ROOT / "freedom_ls" / "course_access"),
        # DJANGO_SETTINGS_MODULE=settings_no_ca via monkeypatch/env for the child
    )
    # exit 2 == "errors during collection" (the bug). 0 (collected) or 5 (none) == no abort.
    assert result.ret != 2
```

Pin down at implementation time only the plumbing that makes the child resolve the
`freedom_ls` package + `pytest-django` and pick up `settings_no_ca` as
`DJANGO_SETTINGS_MODULE` (env for the child, or a `conftest`/`-o` in the isolated
dir). If `pytester` proves awkward for the Django-settings hand-off, fall back to
raw `subprocess.run(..., cwd=REPO_ROOT)` with a `tmp_path` settings module on
`PYTHONPATH` — but keep the mandatory `-p no:cov -p no:playwright -p no:cacheprovider`
flags either way.

**Verify:** the test is Red before Task 2a/2b, Green after; the repo tree is
unchanged after the run (no stray `.coverage`, `test-results/`, `.pytest_cache`).
