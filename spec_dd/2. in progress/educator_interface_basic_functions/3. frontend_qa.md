# Frontend QA: Educator Interface Basic Functions

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- Tailwind built: `npm run tailwind_build`
- Logged in as an educator user with full cohort permissions at `http://localhost:8000/educator/`
- At least one cohort with students, course registrations, and deadlines exists
- A second cohort exists (for duplicate name testing)

---

## Test 1: Existing functionality still works after refactor

1. Navigate to `http://localhost:8000/educator/`
2. Click "Cohorts" in the sidebar
3. **Expected:** Cohorts list loads with data table showing cohort names, student counts, and registered courses
4. Click on a cohort name to view its detail page
5. **Expected:** Cohort detail page shows all panels (details, course progress, courses, students)
6. Click "Users" in the sidebar
7. **Expected:** Users list loads with searchable data table
8. Click "Courses" in the sidebar
9. **Expected:** Courses list loads with data table

---

## Test 2: Create Cohort — "Save" button

1. Navigate to `http://localhost:8000/educator/cohorts/`
2. **Expected:** A "Create Cohort" button is visible near the cohorts data table
3. Click "Create Cohort"

## Test 3: Create Cohort — "Save and add another"

## Test 16: Delete Cohort — With related records

1. Navigate to a cohort's detail page that has students, course registrations, and deadlines
2. **Expected:** A "Delete" button with danger styling is visible on the page
3. Click "Delete"
4. **Expected:** A confirmation modal appears with:
   - A message: "Are you sure you want to delete [cohort name]?"
   - A summary of related records that will be deleted (e.g., "3 cohort memberships, 1 cohort course registrations")
5. Verify the counts match what you see in the cohort's panels (e.g., student count matches membership count)
6. Click "Cancel"
7. **Expected:** The modal closes, nothing happens
8. Click "Delete" again, then click "Delete" in the confirmation modal
9. **Expected:** The cohort is deleted and you are redirected to the cohorts list
10. **Expected:** The deleted cohort no longer appears in the list

---

## Test 17: Delete Cohort — With no related records

1. Create a new empty cohort (no students, no course registrations)
2. Navigate to that cohort's detail page
3. Click "Delete"
4. **Expected:** The confirmation modal shows the warning message but either shows no cascade summary or a message indicating nothing else will be deleted
5. Click "Delete" in the confirmation modal
6. **Expected:** The cohort is deleted and you are redirected to the cohorts list

## Test 19: Loading indicators on form submissions

1. Open any form modal (Create Cohort, Edit Cohort, Delete confirmation)
2. Click the submit button
3. **Expected:** The button text is replaced with a spinning icon and loading text ("Saving..." or "Deleting...")
4. **Expected:** The button is disabled during the request (cannot double-click)
5. **Expected:** After the response, the button returns to normal (or the modal closes)

---

## Test 20: Double-click prevention

1. Open browser DevTools and switch to the Network tab
2. Open any form modal (Create, Edit, or Delete)
3. Fill in valid data
4. Rapidly double-click the submit button
5. **Expected:** Only one POST/DELETE request appears in the Network tab
