# Frontend QA: Loading Indicators on Form Submissions

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- Tailwind built: `npm run tailwind_build`
- Logged in as an educator user at `http://localhost:8000/educator/`
- At least one cohort exists (or create one during testing)

## Test 1: Create Cohort - "Save" button loading state

1. Navigate to `http://localhost:8000/educator/` and click on "Cohorts" in the sidebar
2. Click the "Create Cohort" button to open the modal
3. Enter a cohort name (e.g., "Test Loading Cohort")
4. Click the "Save" button
5. **Expected:** The "Save" button text should briefly be replaced with a spinning icon and "Saving..." text. The button should appear disabled (reduced opacity, no pointer cursor) until the request completes.
6. **Expected:** After the request completes, you are redirected to the new cohort's detail page

## Test 2: Create Cohort - "Save and add another" loading state

1. Navigate to the Cohorts list view
2. Click "Create Cohort" to open the modal
3. Enter a cohort name (e.g., "Another Test Cohort")
4. Click the "Save and add another" button
5. **Expected:** The button shows the spinner and "Saving..." text during submission
6. **Expected:** After success, the form clears, the modal stays open, and the datatable refreshes behind it
7. **Expected:** The button returns to its normal state after the request completes

## Test 3: Edit Cohort - loading state

1. Navigate to a cohort's detail page (e.g., click on a cohort from the list)
2. Click the "Edit" button on the cohort details panel
3. Modify the cohort name
4. Click "Save"
5. **Expected:** The "Save" button shows the spinner and "Saving..." text during submission
6. **Expected:** After success, the modal closes and the details panel updates with the new name

## Test 4: Delete Cohort - loading state

1. Navigate to a cohort's detail page
2. Click the "Delete" button
3. A confirmation modal should appear showing what will be deleted
4. Click "Delete" in the confirmation modal
5. **Expected:** The "Delete" button shows a spinner and "Deleting..." text during submission
6. **Expected:** The button is disabled during the request (cannot double-click)
7. **Expected:** After success, you are redirected to the cohorts list

## Test 5: Double-click prevention

1. Open browser DevTools and switch to the **Network** tab
2. Open any form modal (Create, Edit, or Delete)
3. Fill in the form
4. Rapidly double-click the submit button
5. **Expected:** Only one POST request appears in the Network tab. The button becomes disabled after the first click, preventing duplicate submissions.

## Test 6: Loading state clears on validation error ("Save" button)

1. Open the "Create Cohort" modal
2. Enter a name that already exists (duplicate within the site)
3. Click "Save"
4. **Expected:** The loading state appears briefly during submission
5. **Expected:** When the 422 response returns with validation errors, the button returns to its normal (non-loading) state
6. **Expected:** The error message is displayed in the form and the user can correct and resubmit

## Test 7: Loading state clears on validation error ("Save and add another" button)

1. Open the "Create Cohort" modal
2. Enter a name that already exists (duplicate within the site)
3. Click "Save and add another"
4. **Expected:** The loading state appears briefly during submission
5. **Expected:** When the 422 response returns, the button returns to its normal state
6. **Expected:** The error message is displayed and the user can correct and resubmit

## Test 8: Normal buttons are unaffected

1. Navigate around the educator interface - click sidebar links, datatable pagination, sorting headers
2. **Expected:** Regular buttons and links that don't have the loading prop behave exactly as before - no spinners, no disabled states
