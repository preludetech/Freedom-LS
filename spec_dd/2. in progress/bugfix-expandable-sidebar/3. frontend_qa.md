# Frontend QA: Expandable Sidebar

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- Tailwind built: `npm run tailwind_build`
- A user account to log in with

## Test 1: Desktop — Toggle Button and Compress Behavior

**URL:** Any page using the sidebar (e.g., a course page or educator interface)

1. Open the browser at >= 1024px wide
2. Verify the sidebar is **open by default** (or matches localStorage state)
3. Verify there is a **single toggle button** at the top of the sidebar area showing a close icon (chevron-left)
4. Click the toggle button
5. Verify the sidebar content collapses and the main content **expands to fill the space** (no overlay, no backdrop)
6. Verify the toggle button is still visible in the same vertical position, now showing an open icon (chevron-right)
7. Click the toggle button again
8. Verify the sidebar reopens and main content compresses back

**Expected:** Main content compresses/expands smoothly. No overlay or backdrop appears. Toggle button stays in place.

## Test 2: Desktop — Links Keep Sidebar Open

1. With the sidebar open on desktop (>= 1024px)
2. Click a link inside the sidebar
3. Verify the page navigates to the link target
4. Verify the sidebar **remains open** after navigation

**Expected:** Sidebar stays open when clicking links on desktop.

## Test 3: Desktop — State Persistence

1. Open sidebar on desktop, note its state
2. Refresh the page
3. Verify the sidebar state is preserved (open stays open, closed stays closed)
4. Close the sidebar, refresh again
5. Verify it stays closed

**Expected:** Sidebar state persists across page loads via localStorage.

## Test 4: Mobile — Hidden by Default

1. Resize the browser to < 1024px wide (or use device emulation)
2. Clear localStorage for the sidebar key, then refresh
3. Verify the sidebar is **hidden by default**
4. Verify the toggle button is visible (showing open/menu icon)

**Expected:** Sidebar hidden, only toggle button visible.

## Test 5: Mobile — Overlay and Backdrop

1. On mobile (< 1024px), click the toggle button
2. Verify the sidebar slides out as a **fixed overlay** on top of the main content
3. Verify a **semi-transparent dark backdrop** appears behind the sidebar, obscuring the main content
4. Click the backdrop
5. Verify the sidebar closes and the backdrop disappears

**Expected:** Sidebar overlays content with backdrop. Clicking backdrop closes sidebar.

## Test 6: Mobile — Auto-Close on Link Click

1. On mobile (< 1024px), open the sidebar
2. Click a link inside the sidebar
3. Verify the page navigates AND the sidebar **closes automatically**

**Expected:** Link navigation works and sidebar auto-closes on mobile.

## Test 7: Smooth Transitions

1. On both mobile and desktop, toggle the sidebar open and closed multiple times
2. Verify transitions are **smooth** (fade + slide) and not jarring or instant

**Expected:** Animations are smooth for both expand and collapse.

## Test 8: Child Template Compatibility

Test that both child templates still work correctly:

### Student course page
1. Navigate to a course page (student interface)
2. Verify the sidebar loads course TOC content via HTMX
3. Verify toggle, collapse, expand all work correctly

### Educator interface
1. Navigate to the educator interface
2. Verify the sidebar shows the navigation menu
3. Verify toggle, collapse, expand all work correctly

**Expected:** Both child templates work without any changes needed.

## Test 9: Toggle Button Vertical Alignment

1. On desktop, note the vertical position of the toggle button when sidebar is open
2. Close the sidebar
3. Verify the toggle button is in the **same vertical position** as when it was open

**Expected:** Button does not jump or shift position between states.
