# Spec: Fix Expandable Sidebar Behavior

## Problem

The current sidebar in `_base_interface.html` has two separate toggle buttons (open and close) that are positioned in different locations, causing visual misalignment. The responsive behavior also needs refinement — mobile link clicks should auto-close the sidebar, and the desktop sidebar should compress content rather than overlay it.

## Scope

This bugfix affects:

- `freedom_ls/base/templates/_base_interface.html` (template structure)
- `freedom_ls/base/static/base/js/alpine-components.js` (Alpine.js `sidebarComponent`)

Child templates (`_course_base.html`, `interface.html`) should not need changes since they only override content blocks.

## Decisions

### Single breakpoint at 1024px

The idea distinguishes "phones" from "tablets and large screens." We use the existing 1024px media query breakpoint (`lg:` in Tailwind) to separate these two modes. There is no meaningful behavioral difference between tablet and desktop, so a third breakpoint is unnecessary.

### Single toggle button inside the sidebar area

Instead of two separate buttons in different positions, there will be one toggle button that lives in the sidebar column. The icon changes based on state (open/close). This ensures perfect vertical alignment because the button never moves — it sits at the top of the sidebar area regardless of whether the sidebar content is visible or collapsed.

## Behavior

### All devices

- A single toggle button at the top of the sidebar area
- The button icon reflects the current state: one icon when collapsed, another when expanded
- Sidebar state persists to localStorage (existing behavior, unchanged)
- Smooth transition animations for expand/collapse

### Mobile (< 1024px)

- Sidebar is **hidden by default**
- When expanded, the sidebar slides out as a **fixed overlay** on top of the main content
- A **semi-transparent backdrop** obscures the main content behind the sidebar
- Clicking the backdrop closes the sidebar
- **Clicking any link inside the sidebar navigates AND closes the sidebar**
- The toggle button remains accessible when the sidebar is closed

### Desktop (>= 1024px)

- Sidebar is **shown by default**
- The sidebar sits beside the main content in a flex layout; toggling it **compresses/expands the main content** (no overlay, no backdrop)
- **Clicking links in the sidebar navigates and keeps the sidebar open**
- The toggle button is visible at the top of the sidebar area

## Implementation Details

### Toggle button positioning

The toggle button lives in a narrow column that is always rendered. When the sidebar is collapsed, this column contains only the toggle button (minimal width). When expanded, the toggle button appears at the top of the full-width sidebar panel. This ensures the button is always in the same vertical position.

### Auto-close on mobile link clicks

Use event delegation on the sidebar panel: listen for clicks on `<a>` elements. When `isMobile` is true and the clicked element is a link, call `closeSidebar()`. This works generically for any content placed in the `sidebar_content` block by child templates, including HTMX-loaded content.

### Desktop compress behavior

The existing flex layout (`flex` container with `shrink-0` sidebar and `flex-1` main content) already handles this correctly. When the sidebar is hidden on desktop, the main content expands to fill the space. No overlay or backdrop is needed on desktop.

## Success Criteria

1. There is a single toggle button that stays in a consistent vertical position across open/closed states
2. The toggle button icon changes to reflect sidebar state
3. On mobile (< 1024px): sidebar is hidden by default, overlays content with backdrop when open, and auto-closes when a link is clicked
4. On desktop (>= 1024px): sidebar is shown by default, compresses main content when toggled, and stays open when links are clicked
5. Existing child templates (`_course_base.html`, `interface.html`) continue to work without changes
6. Sidebar state persistence via localStorage continues to work
7. Transitions are smooth and not visually jarring
