# Mobile Responsiveness - Frontend QA Test Plan

## Setup

- Dev server: `uv run python manage.py runserver` (runs at http://127.0.0.1:8000)
- Login credentials: `demodev@email.com` / `demodev@email.com`
- Use `qa-data-helper` agent to ensure test data exists: at least one cohort with students, courses with topics and forms, and some student progress
- Test at **360px** and **375px** widths (mobile) and **1280px** (desktop regression)

---

## Test 1: Educator Sidebar - Mobile Behavior

**URL:** http://127.0.0.1:8000/educator/ (or wherever educator interface loads)

### Steps (360px viewport):
1. Navigate to educator interface
2. **Verify:** Sidebar is collapsed by default (only toggle button visible, main content fills screen)
3. Click the sidebar toggle button
4. **Verify:** Sidebar appears as an overlay (fixed position, over content, with semi-transparent backdrop)
5. **Verify:** Main content is still visible behind the backdrop
6. Click the backdrop (area outside sidebar)
7. **Verify:** Sidebar closes
8. Open sidebar again, navigate to a different educator page (e.g., click a cohort)
9. **Verify:** Sidebar state persists (still open or closed based on last action)
10. Refresh the page
11. **Verify:** Sidebar state persists after refresh

### Steps (1280px viewport):
1. Navigate to educator interface
2. **Verify:** Sidebar is expanded by default, content is beside it (not overlaid)
3. **Verify:** No backdrop visible
4. Toggle sidebar closed, navigate to another page
5. **Verify:** Sidebar remains closed

## Test 3: Progress Grid - Mobile Scroll

**URL:** http://127.0.0.1:8000/educator/ -> select a cohort -> view course progress

### Prerequisites:
- Need a cohort with students enrolled in a course that has multiple topics/forms (enough columns to require horizontal scrolling)

### Steps (375px viewport):
1. Navigate to the course progress grid
2. **Verify:** The grid is visible and scrollable horizontally
3. **Verify:** The first column (student names) is NOT frozen/sticky
4. Scroll the table to the right until student names are no longer visible
5. **Verify:** Each visible row shows the student name as a floating label above the row
6. Scroll back to the left
7. **Verify:** Floating labels disappear when first column is visible again
8. **Verify:** All data cells are readable and accessible

### Steps (1280px viewport):
1. View the same progress grid
2. **Verify:** Table displays normally with sticky first column (if it was sticky before)
