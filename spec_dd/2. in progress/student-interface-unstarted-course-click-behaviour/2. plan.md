### Task A1 вЂ” Add `DifficultyLevel` choices + three fields to `Course`

File: `freedom_ls/content_engine/models.py` (`Course`, around `models.py:149`).

1. Add a `models.TextChoices` enum near the top of the module, beside `FormStrategy`
   (`models.py:25`), using `gettext_lazy` for labels to match the file's convention:

   ```python
   class DifficultyLevel(models.TextChoices):
       BEGINNER = "beginner", _("Beginner")
       INTERMEDIATE = "intermediate", _("Intermediate")
       ADVANCED = "advanced", _("Advanced")
       ALL_LEVELS = "all_levels", _("All levels")
   ```

2. Add three optional fields to `Course` (import `ArrayField` from
   `django.contrib.postgres.fields`):

   ```python
   learning_outcomes = ArrayField(
       models.CharField(max_length=255),
       blank=True,
       default=list,
       help_text=_("Ordered list of 'what you'll learn' outcomes. Empty hides the section."),
   )
   difficulty = models.CharField(
       max_length=20, blank=True, default="", choices=DifficultyLevel.choices,
   )
   estimated_duration = models.DurationField(null=True, blank=True)
   ```

3. Add a `display_estimated_duration` method on `Course` вЂ” small, dependency-free, returns `""` when
   the field is null so the template can omit the stat. Coarse, honest formatting:

   ```python
   def display_estimated_duration(self) -> str:
       """Human, coarse duration like '~2 hours' / '~45 min' / '~1 hour 30 min'. '' when unset."""
       # Treat both None and a zero timedelta as "unset" (a zero timedelta is
       # falsy, but be explicit so a stored 0 doesn't render a bare "~").
       if not self.estimated_duration or not self.estimated_duration.total_seconds():
           return ""
       total_minutes = round(self.estimated_duration.total_seconds() / 60)
       hours, minutes = divmod(total_minutes, 60)
       parts: list[str] = []
       if hours:
           parts.append(f"{hours} hour{'s' if hours != 1 else ''}")
       if minutes:
           parts.append(f"{minutes} min")
       return "~" + " ".join(parts) if parts else ""
   ```

   `difficulty` needs no helper вЂ” Django generates `get_difficulty_display()` from the choices, which
   returns `""` when blank.

Tests (write first): a field-presence/round-trip test and `display_estimated_duration` cases
(`None в†’ ""`, `timedelta(0) в†’ ""`, exactly 2h, 45 min, 1h30) plus `get_difficulty_display()` blank в†’ `""`. Put model tests in
`freedom_ls/content_engine/tests/` following `fls:testing`.

### Task A4 вЂ” Content schema (Pydantic)

File: `freedom_ls/content_engine/schema.py` (`Course`, `schema.py:96`). The model is `extra="forbid"`, so
fields absent here are silently dropped by `content_save`. Add:

```python

    default_factory=list, description="Ordered 'what you'll learn' outcomes"
)
```

Add a `DifficultyLevel(StrEnum)` to `schema.py` mirroring the model values (`beginner`, `intermediate`,
`advanced`, `all_levels`) вЂ” keep the schema self-contained like the existing `FormStrategy`/`QuestionType`

**Loader compatibility check (important):** `save_with_uuid` (`content_save.py:253`) does
`item.model_dump(exclude=..., exclude_none=True)` then validates every key against the Django model's
field names. Confirm during implementation that:
- `learning_outcomes` dumps as a plain `list[str]` (default `[]`) в†’ accepted by `ArrayField`.
- `difficulty` dumps as the enum's string value (Pydantic dumps `StrEnum` to its `str` value) в†’ accepted
  by the `CharField` choices. If `model_dump` yields an enum object rather than the bare string, coerce in
  the schema (`use_enum_values`) вЂ” verify with a `content_save` round-trip test.
- `estimated_duration` dumps as a `datetime.timedelta` (python mode) в†’ accepted by `DurationField`.
- All three are excluded when unset (`None` for difficulty/duration; empty list is harmless for the
  ArrayField default) so existing demo courses without them keep saving cleanly.

Notes:
- **Breadcrumbs caveat (do not reuse the panel_framework partial).** Spec В§6.1 suggested reusing
  `panel_framework/partials/breadcrumbs.html`, but that partial hard-codes
  `hx-target="#main-content"` on every crumb link. `#main-content` exists only in the panel_framework
  layout; `_base.html` (which this page extends) has no such element, so each breadcrumb click would
  fire an HTMX swap against a missing target and **silently fail to navigate**. Render breadcrumbs
  **inline** in `course_detail.html` (or a new `student_interface/partials/breadcrumbs.html`) with plain
  `{% url %}` anchors and **no** HTMX attributes вЂ” these are full-page navigations. Style/markup may
  mirror the panel_framework partial (incl. an `aria-current="page"` final crumb), just without the
  `hx-*` attributes. Use `{% comment %}` for any real comments.
- The pseudocode above uses `{# вЂ¦ #}` for brevity only. In real templates use Django
  `{% comment %}вЂ¦{% endcomment %}` blocks (project convention) вЂ” never `{# вЂ¦ #}`.
- The hero gradient uses the **existing** `course-accent-<key>` classes (already in
  `tailwind.components.css`, palette-derived per theme) вЂ” this satisfies the brand "no ad-hoc gradients"
  rule because no new gradient is invented. Do not hand-roll `from-вЂ¦/to-вЂ¦` gradients; reuse the accent
  system and keep the icon opacity low so the hero stays subtle.
- `rendered_content()` shifts content H1 в†’ H2 (mdx_headdown). Pick page heading levels so the rendered
  "About" markdown sits correctly beneath the section headings.
- For the "what you'll learn" tick, use a real semantic `<c-icon>` name (the ToC partial uses
  `name="complete"`); verify the name resolves in the active icon set (`fls:icon-usage`). Never raw SVG or
  unicode ticks.
- After authoring the template, run `npm run tailwind_build` so any newly-used utility classes are
  compiled.
- The `course-children` partialdef requires `children`, `course`, `is_registered` in context вЂ” all are
  supplied by the view. Render it via the `#course-children` partialdef include (as `partial_course_toc`
  already does) so only the list renders, not the partial's own `<h1>`/Contents wrapper. Add your own
  "Course content" heading.
- Responsive order (spec В§6.5): on mobile the single column should put hero в†’ sign-up panel (or its CTA)
  в†’ main content. Use grid ordering (`order-*` utilities) so the panel sits near the top on mobile while
  staying in the right column on `lg`.
- Keep all real-data-only rules: no ratings, wishlist, certificate, fabricated tags, or `вЂ”`/`N/A`
  placeholders. Leave a `# TODO`/`@claude` comment where a certificate item would live (spec В§6.4, В§3).
- Mobile sticky-bottom CTA bar is optional/nice-to-have вЂ” skip unless trivial.
