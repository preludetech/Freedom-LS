# Spec: Create Demo Course with Course Parts

## Summary

Create a new demo course in `demo_content/` that demonstrates the CoursePart functionality. This is the only feature of the content system not currently covered by existing demo courses.

## Why This Matters

The three existing demo courses (`functionality_demo_end_with_topic`, `functionality_demo_end_with_quiz`, `functionality_demo_standard_markdown`) all use a flat structure — topics and forms directly under the course. None demonstrate CourseParts, which allow courses to be organized into chapters/sections. Without a demo course using parts, developers and content creators have no in-project reference for how to structure courses with parts.

## What is a CoursePart?

A CoursePart is a grouping mechanism within a course, similar to a chapter or module. Key characteristics:

- Has a title, optional subtitle, description, and category
- Has **no markdown body content** (unlike Topics)
- Contains child items (Topics and/or Forms) that are ordered within the part
- Is defined by a `part.yaml` file placed inside a numbered subdirectory
- Children are auto-discovered from sibling files in the same directory (no explicit `children:` list needed)
- In the student interface, CourseParts appear as section headers in the course table of contents, with their children nested underneath
- CoursePart status is derived from the statuses of its children

## Course Structure

### Course metadata

- **Directory name**: `functionality_demo_course_parts`
- **Title**: "Functionality Demo - Course Parts"
- **Subtitle**: "Demonstrating course organisation with parts"
- **Description**: "This course demonstrates how content can be organised into parts (chapters/modules)."
- **content_type**: COURSE
- **UUID**: Omit from all frontmatter. UUIDs are auto-generated by the `content_save` management command and written back to the files. Do not manually add or edit UUIDs.

### Parts

The course should have **3 parts** with a mix of content types to demonstrate flexibility:

#### Part 1: "Getting Started" (topics only)

- Contains 2 short topics with minimal placeholder content
- Demonstrates the simplest use case: a part that groups related topics

#### Part 2: "Core Concepts" (topics + quiz)

- Contains 2 short topics followed by a quiz form
- Demonstrates that parts can contain both topics and forms
- The quiz should have 2-3 simple multiple-choice questions
- Quiz should use `QUIZ` strategy with a pass percentage

#### Part 3: "Wrapping Up" (topic + survey)

- Contains 1 topic and 1 survey form
- Demonstrates a different form type (survey vs quiz) within a part
- The survey should use `CATEGORY_VALUE_SUM` strategy with 1-2 questions

### Decision: Why 3 parts?

Three parts is the minimum needed to demonstrate:
1. A part with only topics (simplest case)
2. A part with topics and a quiz (mixed content, assessment)
3. A part with a topic and a survey (different form type)

This keeps the demo lean while showing the range of what parts can contain.

### Decision: Why minimal placeholder content?

The existing demo courses already thoroughly demonstrate cotton components (callouts, pictures, YouTube embeds, code blocks, tables, etc.). Duplicating that content here would add bloat without value. The focus of this demo is the **parts structure itself**, not the content within topics.

Topic content should be short (a heading and 1-2 paragraphs) and can reference the parts structure to make the demo self-documenting (e.g., "This topic is inside Part 1").

## File Structure

```
demo_content/functionality_demo_course_parts/
  course.md                          # Course definition
  01. Getting Started/
    part.yaml                        # CoursePart: "Getting Started"
    01. welcome.md                   # Topic
    02. what-to-expect.md            # Topic
  02. Core Concepts/
    part.yaml                        # CoursePart: "Core Concepts"
    01. key-ideas.md                 # Topic
    02. going-deeper.md              # Topic
    03. knowledge-check/
      form.md                        # Quiz form
      1. page.yaml                   # Quiz questions
  03. Wrapping Up/
    part.yaml                        # CoursePart: "Wrapping Up"
    01. summary.md                   # Topic
    02. feedback/
      form.md                        # Survey form
      1. page.yaml                   # Survey questions
```

### Decision: File naming convention

Following the same conventions as the RPC Course (`/home/sheena/workspace/lms/first-class-courses/RPC Course`):
- Zero-padded numbered directory prefixes (`01.`, `02.`, `03.`) control ordering
- `part.yaml` identifies each subdirectory as a CoursePart (detected by `content_type: COURSE_PART`)
- Numbered `.md` files within each part directory control topic ordering
- Forms live in their own numbered subdirectories within parts

## Content Details

### course.md frontmatter

```yaml
---
content_type: COURSE
title: Functionality Demo - Course Parts
subtitle: Demonstrating course organisation with parts
description: This course demonstrates how content can be organised into parts (chapters/modules).
---
```

No `children:` list — rely on auto-discovery.

### part.yaml files

Each part.yaml should contain:
```yaml
---
content_type: COURSE_PART
title: <Part Title>
---
```

### Topic .md files

Each topic should have standard frontmatter:
```yaml
---
content_type: TOPIC
title: <Topic Title>
---
```

Body content should be minimal — a heading and 1-2 short paragraphs. Content can be self-referential (describing the demo structure) to make the demo self-documenting.

### Quiz form (Part 2)

- Strategy: `QUIZ`
- Pass percentage: 80%
- Show incorrect answers: true
- 2-3 simple multiple-choice questions with obvious correct answers

### Survey form (Part 3)

- Strategy: `CATEGORY_VALUE_SUM`
- 1-2 multiple-choice questions, each with a `category` field (required for category-based scoring)

## Success Criteria

1. The course loads successfully via the content save management command without errors
2. The course appears in the course listing with its title and description
3. The course table of contents shows 3 parts as section headers with their children nested underneath
4. Navigating through the course works correctly — topics and forms within parts are accessible and ordered correctly
5. Progress tracking works — completing items within parts updates part status and overall course progress
6. The quiz in Part 2 functions correctly (can be submitted, scored, shows results)
7. The survey in Part 3 functions correctly (can be submitted)
8. The file structure follows the same conventions as the RPC Course
