# Implementation plan: FLS test-suite portability — Part 2 (positive signal)

Delivers the four layers of the spec: an importable **conformance suite**
(Layer 3), boot-time **system checks** (Layer 4), the **upgrade-notes** guidance
tie-in (Layer 5), and the Part-2 **plugin-doc** touches (Layer 6). Part 1
(marker taxonomy, `INSTALLED_APPS`-keyed collection safety, de-branded
assertions) is assumed present and is reused, not re-built.

`config.py` settings convention** — each app declares its FLS-specific settings
(defaults, project-override resolution, and which are *required*) in one place,
app code reads from that module instead of `django.conf.settings` ad hoc, and a
required-empty setting is surfaced by `manage.py check`. Layer 4's
`COURSE_ACCESS_BACKEND` check is re-expressed on top of it.

We follow TDD throughout: for each probe and each check, write the failing
negative meta-test first (proving the probe/check has teeth), then implement.

## Layer 0 — per-app `config.py` settings convention (folded in during plan review)

A codebase-wide convention: each FLS app that reads FLS-specific settings owns a
`freedom_ls/<app>/config.py` declaring its defaults, resolving project overrides,
and marking which settings are **required**. This generalises the sole existing
instance (`freedom_ls/student_management/config.py`) and gives Layer 4 a
single-sourced required-settings declaration.

**Enforcement decision (matches spec D8 / "checks must never raise"):** a
required-empty setting is reported by a system check that **returns an `Error`**
(exits `manage.py check` non-zero, blocks `runserver`/`migrate`/`test`) — *not*
by raising at import/`ready()` time, which would crash every command including
`check` itself before it can report. As a **runtime backstop**, the accessor
raises `ImproperlyConfigured` **lazily, only when a required-empty setting is
actually read** — never at import.

### L0.1 — shared base `freedom_ls/base/app_settings.py` (new)

Imports **only** `django.conf.settings`, `django.core.exceptions`,
`django.core.checks` + stdlib (no models), so `from freedom_ls.<app>.config import
config` is safe at any module top-level (no import-cycle risk).

```python
from __future__ import annotations

from typing import NamedTuple

from django.conf import settings
from django.core.checks import Error
from django.core.exceptions import ImproperlyConfigured


class Setting(NamedTuple):
    """One declared setting: its fallback value and whether the project must set it."""

    default: object = None
    required: bool = False


class AppSettings:
    """Per-app settings: a project's Django settings override declared fallbacks.

    Each app subclass sets ``declared_settings`` to a ``{name: Setting(...)}`` map.
    ``config.NAME`` returns the project's value if set, else the declared default;
    a ``required`` setting the project has not supplied raises ImproperlyConfigured
    on read (lazily, never at import) as a runtime backstop. A caller that must not
    raise (e.g. a system check) wraps the read in ``try/except ImproperlyConfigured``.
    """

    declared_settings: dict[str, Setting] = {}

    def __getattr__(self, name: str) -> object:
        try:
            setting = self.declared_settings[name]
        except KeyError:
            raise AttributeError(name) from None
        value = getattr(settings, name, None)
        if isinstance(value, str):
            value = value.strip()
        if value not in (None, ""):
            return value
        if setting.required:
            raise ImproperlyConfigured(
                f"{name} is required but is not set. "
                f"Set {name} in your Django settings."
            )
        return setting.default

    def missing_required(self) -> list[str]:
        """Names of required settings the project has not supplied. Never raises."""
        missing: list[str] = []
        for name, setting in self.declared_settings.items():
            if not setting.required:
                continue
            try:
                getattr(self, name)
            except ImproperlyConfigured:
                missing.append(name)
        return missing


def required_settings_errors(config: AppSettings, app_label: str) -> list[Error]:
    """Build ``<app_label>.E001`` Errors for each missing required setting.

    The single reusable body every required-settings check calls, so no
    per-setting boilerplate is hand-written.
    """
    return [
        Error(
            f"{name} is required but is not set.",
            hint=f"Set {name} in your Django settings.",
            id=f"{app_label}.E001",
        )
        for name in config.missing_required()
    ]
```

**Typing (no `Any`, per CLAUDE.md):** each per-app subclass declares its settings
as **class-level annotations without assignment**, so consumers get real static
types while `__getattr__` supplies values at runtime (the django-stubs idiom).
Fall back to typed `@property` accessors only if the lint/type step rejects the
idiom — decide by running the step, not up front.

### L0.4 — RISK-1 (must fix): loader-through-config vs `course_access` checks

Rerouting `course_access/loader.py:37` to read `config.COURSE_ACCESS_BACKEND`
makes `get_course_access_backend()` raise `ImproperlyConfigured` when the backend

Layer-4 `student_interface` E001 already reports the unset case cleanly).

### T3.1 — Package scaffolding + override-friendliness core

Create:

- `freedom_ls/contrib/__init__.py` — empty.
- `freedom_ls/contrib/conformance/_registry.py` — the drop() mechanism and
  install-gating helpers (the one non-`test_*` module; its name is `_`-free but
  it defines **no `test_*` callables**, so `import *` from it is inert, and it is
  itself `test_*`-unmatched so pytest ignores it).

`_registry.py` pseudocode:

```python
from __future__ import annotations
from django.conf import settings

# Downstream-mutable set of internal-tier probe ids to skip when a project keeps
# an app but customises one of its routes.
_DROPPED: set[str] = set()

def drop(*probe_ids: str) -> None:
    """Prune internal-tier namespace probes a downstream has customised.

    Call from the downstream's conftest/test module before the run. Contract-tier
    ids cannot be pruned; the probe body enforces that.
    """
    _DROPPED.update(probe_ids)

def _is_dropped(probe_id: str) -> bool:
    return probe_id in _DROPPED

def _app_installed(app_label_dotted: str) -> bool:
    # e.g. "freedom_ls.student_interface"
    return app_label_dotted in settings.INSTALLED_APPS
```

**Note for the real code (both here and every probe/check module below):** the
pseudocode omits some type hints and elides docstrings for brevity — the shipped
code MUST carry full type hints on every function (no `Any`, `X | None` syntax)
per CLAUDE.md, and MUST NOT contain any spec-citation comments (no `D9`, `§`,
"criterion #7"). The pseudocode's decision-letter references are for the plan
reader only.

- Runtime membership check (not collection-time): probes stay **collected**
  (node IDs appear in `--collect-only`, criterion #1) but **skip** at run when
  their app is absent or an internal id is dropped — satisfying criterion #7
  ("skipped, not failed") regardless of import ordering between the downstream's
  `drop()` call and parametrize evaluation.

### T3.2 — Namespace + reference URL probes (`test_urls.py`)

`freedom_ls/contrib/conformance/test_urls.py`. Data tables + two probes.

Pseudocode:

```python
from __future__ import annotations
import pytest
from dataclasses import dataclass, field
from django.conf import settings
from django.urls import reverse, NoReverseMatch
from ._registry import _app_installed, _is_dropped

@dataclass(frozen=True)
class _Probe:
    app: str          # dotted INSTALLED_APPS label, e.g. "freedom_ls.student_interface"
    viewname: str     # e.g. "student_interface:dashboard"
    contract: bool    # True = hard-fail while app installed; False = prunable
    kwargs: dict[str, str] = field(default_factory=dict)   # placeholder reverse kwargs

# viewname ids MUST match the acceptance node IDs.
FLS_NAMESPACE_PROBES: list[_Probe] = [
    _Probe("freedom_ls.student_interface", "student_interface:dashboard", True),
    _Probe("freedom_ls.student_interface", "student_interface:course_detail", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:course_home", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:initiate_course_access", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:courses", False),
    _Probe("freedom_ls.course_applications", "course_applications:apply", True, {"course_slug": "x"}),
    _Probe("freedom_ls.course_applications", "course_applications:status", True, {"pk": "00000000-0000-0000-0000-000000000000"}),
    _Probe("freedom_ls.course_interest", "course_interest:express_interest", True, {"course_slug": "x"}),
    _Probe("freedom_ls.educator_interface", "educator_interface:interface", True, {"path_string": ""}),
    _Probe("freedom_ls.accounts", "accounts:account_profile", True),
    _Probe("freedom_ls.accounts", "accounts:legal_doc", True, {"doc_type": "terms"}),
]

REFERENCE_URL_NAMES: list[str] = ["sitemap", "robots_txt"]

@pytest.mark.parametrize(
    "probe", FLS_NAMESPACE_PROBES,
    ids=[p.viewname for p in FLS_NAMESPACE_PROBES],   # -> "student_interface:dashboard" etc.
)
def test_fls_namespace_reverses(probe: _Probe) -> None:
    if not _app_installed(probe.app):
        pytest.skip(f"{probe.app} not installed")
    if not probe.contract and _is_dropped(probe.viewname):
        pytest.skip(f"{probe.viewname} dropped by downstream")
    reverse(probe.viewname, kwargs=probe.kwargs or None)   # NoReverseMatch => fail

@pytest.mark.parametrize("name", REFERENCE_URL_NAMES, ids=REFERENCE_URL_NAMES)
def test_reference_url_reverses(name: str) -> None:
    reverse(name)   # required (D1); NoReverseMatch => hard fail
```

Notes:
- Parametrize `ids` produce exactly the acceptance node IDs
  (`test_fls_namespace_reverses[student_interface:dashboard]`, …).
- `educator_interface:interface` reverses with `path_string=""` against the
  catch-all `re_path`.
- The probe asserts *resolvability*, never that a target object exists — no DB.

### T3.5 — Migration-state probe (`test_migrations.py`)

`freedom_ls/contrib/conformance/test_migrations.py`.

```python
def test_migration_state_consistent() -> None:
    from django.core.management import call_command
    # --check --dry-run diffs models against migration files and writes NOTHING
    # to disk. Raises SystemExit if a migration is missing.
    call_command("makemigrations", check=True, dry_run=True, verbosity=0)
```

- `call_command(..., check=True)` raises `SystemExit` on model/migration drift;
  pytest reports that as a failure. `--dry-run` guarantees **no migration file is
  written**. Needs **no `django_db` marker and no test-DB records**.
- **Accuracy note (corrects an over-claim):** `makemigrations` internally calls
  `MigrationLoader.check_consistent_history()`, which *may open* a DB connection
  to read `django_migrations`; Django guards that call and swallows the
  `OperationalError` if the DB is unreachable, so the probe stays green with or
  without a live DB. Under FLS's own run there is a test DB and `--allow-hosts`
  already permits `127.0.0.1`, so this is a non-issue in practice — but the probe
  does **not** strictly "never touch a socket", it tolerates the DB being absent.
  T3.8 adds a meta-test asserting the probe still passes when the DB is
  unreachable, so this tolerance is proven, not assumed.

### T4.1 — `student_interface` checks module

New `freedom_ls/student_interface/checks.py`, matching
`freedom_ls/base/checks.py` style (settings/apps only, no DB). App-label IDs
`freedom_ls_student_interface.*` (D3). All three gated on the app being
installed. **E001 is now single-sourced from Layer 0's `course_access.config`
`REQUIRED` declaration** via `required_settings_errors(...)` — no hand-rolled
"is the setting set" body. Importing `course_access.config` from
`student_interface` is a **downward** edge (student_interface already consumes
course_access); it inverts nothing, but flag it for `/plan_structure_review`.

```python
from __future__ import annotations
from collections.abc import Sequence
from django.apps import AppConfig, apps
from django.conf import settings
from django.core.checks import CheckMessage, Error, Warning, register
from django.core.exceptions import ImproperlyConfigured

from freedom_ls.base.app_settings import required_settings_errors
from freedom_ls.course_access.config import config as course_access_config

APP_LABEL = "freedom_ls_student_interface"
APP_DOTTED = "freedom_ls.student_interface"

def _app_relevant(app_configs: Sequence[AppConfig] | None) -> bool:
    if APP_DOTTED not in settings.INSTALLED_APPS:
        return False
    # Respect scoped `manage.py check <app>` runs.
    if app_configs is not None and not any(c.label == APP_LABEL for c in app_configs):
        return False
    return True

@register()
def check_course_access_backend_set(app_configs=None, **kwargs) -> list[CheckMessage]:
    if not _app_relevant(app_configs):
        return []
    # required_settings_errors emits freedom_ls_student_interface.E001 for each
    # missing REQUIRED setting course_access.config declares (COURSE_ACCESS_BACKEND).
    return required_settings_errors(course_access_config, APP_LABEL)

@register()
def check_course_access_backend_app_installed(app_configs=None, **kwargs) -> list[CheckMessage]:
    if not _app_relevant(app_configs):
        return []
    # Checks must never raise: a missing required setting makes the read raise,
    # which E001 already reports, so treat that as "nothing to check here".
    try:
        dotted = course_access_config.COURSE_ACCESS_BACKEND
    except ImproperlyConfigured:
        return []
    # Resolve the backend's containing app by dotted-path prefix match WITHOUT
    # importing the backend class. get_containing_app_config only ever returns an
    # INSTALLED app config (or None) and does not import the target, so it cannot
    # raise here and the "checks must never raise" rule holds by construction.
    module_path = dotted.rsplit(".", 1)[0]              # strip the class name
    app_config = apps.get_containing_app_config(module_path)
    if app_config is None:
        return [Error(f"COURSE_ACCESS_BACKEND {dotted!r} is not provided by any installed app",
                      hint="Add the backend's app to INSTALLED_APPS, or fix the dotted path.",
                      id=f"{APP_LABEL}.E002")]
    return []

@register()
def check_sitemaps_app_present(app_configs=None, **kwargs) -> list[CheckMessage]:
    if not _app_relevant(app_configs):
        return []
    from django.urls import NoReverseMatch, reverse
    try:
        reverse("sitemap")                             # is a sitemap actually wired?
    except NoReverseMatch:
        return []                                      # nothing wired -> nothing to warn
    if "django.contrib.sitemaps" not in settings.INSTALLED_APPS:
        return [Warning("'sitemap' is wired but django.contrib.sitemaps is not installed",
                        hint="Add 'django.contrib.sitemaps' to INSTALLED_APPS.",
                        id=f"{APP_LABEL}.W001")]
    return []
```

- **E002 resolves without importing** (`apps.get_containing_app_config`, mirroring
  Django's `admin.check_dependencies`). Because no import occurs and the helper
  returns `None` rather than raising, the spec's "must never raise / catch
  ImportError…" concern is satisfied *by construction* — the behavioural import is
  left to Layer 3's `test_configured_backend_instantiates` (the D8 division). No
  separate `is_installed()` call is needed: `get_containing_app_config` already
  only returns installed configs.
- **W001 uses `reverse("sitemap")`** to detect wiring — cheap, side-effect-free,
  and only raises `NoReverseMatch` (caught). This mirrors the Layer-3
  `test_reference_url_reverses` probe rather than parsing the URLconf.
- No `Tags.database`, no `deploy=True` (internal-consistency checks run on every
  `check`/`runserver`/`migrate`).
