# Plan: On-brand course content widgets

Implementation plan for `1. spec.md`. Eight content widgets authored as cotton
components inside markdown, rendered through the existing
`markdown â†’ render â†’ nh3 sanitise â†’ cotton compile â†’ Django render` pipeline,
styled with theme **role tokens** (no hardcoded hex), accessible by default, and
demonstrated in a new `functionality_demo_content_widgets` course.

We follow **TDD**: for each widget write a failing render/sanitisation test first
(in `freedom_ls/markdown_rendering/tests/`), then implement until green. Do **not**
pre-write every test up front.

## 0. How the pipeline actually works (read before coding)

Established by reading `freedom_ls/markdown_rendering/markdown_utils.py`,
`config/settings_base.py`, and `freedom_ls/content_engine/templatetags/content_tags.py`:

1. `render_markdown()` runs `markdown.Markdown(extensions=["fenced_code",
   "mdx_headdown", "tables"])` with the indented-code blockprocessor **disabled**.
2. Every key in `settings.MARKDOWN_ALLOWED_TAGS` is appended to
   `md.block_level_elements`, so `<c-foo>` blocks are treated as block-level HTML.
3. `nh3.clean()` runs with `allowed_tags = nh3.ALLOWED_TAGS âˆª MARKDOWN_ALLOWED_TAGS.keys()`
   and per-tag attributes merged from `MARKDOWN_ALLOWED_TAGS`. **A new cotton tag is
   invisible to authors until both its tag name and every attribute it reads are
   added to `MARKDOWN_ALLOWED_TAGS`.**
4. **Cotton compiles _after_ nh3.** The HTML a cotton template emits
   (`<figure>`, `<blockquote>`, `<table>`, `<button>`, KaTeX output, â€¦) is **never
   re-sanitised**. So we do **not** need to allowlist `figure`, `figcaption`,
   `caption`, etc. â€” only the `c-*` tags and their attributes. Developer-authored
   template markup is trusted; **author-supplied attribute/slot values are
   auto-escaped by Django** and must only ever map to **fixed classes/role tokens**,
   never raw CSS/hex.
5. **Slots re-enter the pipeline.** `{% markdown slot %}` calls `render_markdown`
   again, so slot content **is** sanitised (e.g. `javascript:` links in a pull-quote
   body are stripped by nh3). This is the reuse point for any widget whose body is
   prose/markdown.
6. **Key consequence for raw code/LaTeX:** content inside a block-level `c-*`
   element is preserved verbatim by python-markdown (not markdown-processed), then
   it passes through the **outer** nh3 pass. So any literal `<`, `>` an author types
   inside `c-code-block`/`c-equation` looks like a tag to nh3 and is stripped.
   Authors must escape `&lt; &gt; &amp;` in those two widgets (documented in the
   demo). This is why `c-code-block` does **not** offer markdown-processed code.

Content is rendered on student pages via `{% markdown topic.content %}`:
- `student_interface/course_topic.html` â†’ `_topic_base.html` â†’ `_course_base.html`
- `student_interface/course_form.html`, `course_form_page.html`,
  `partials/course_list.html` also call `{% markdown %}`.

`_course_base.html` already overrides `{% block extra_alpine_components %}` to load
`student_interface/js/alpine-components.js`. `_base.html` loads
`base/js/alpine-components.js`, then the `extra_alpine_components` block, then the
**Alpine CSP** build (`@alpinejs/csp`). Anything we add to `extra_alpine_components`
therefore loads **before** Alpine initialises â€” the correct slot for both our new
component file and the KaTeX library.

### Existing reference patterns to reuse (keep it DRY)
- `cotton/callout.html` â€” `{% load content_tags %}`, `{% markdown slot %}`,
  `<c-icon :name=... />`, role-token classes.
- `cotton/picture.html` â€” `get_file_by_path` lookup + error fallback + lightbox.
- `base/js/alpine-components.js` â€” `pictureModal`, `scrollTableLabels`,
  `dropdownMenu.onEscape` (focus-return pattern), `toast` (motion + focus) â€” these
  are **how** to write CSP-safe Alpine (named methods only, no inline expressions).
- `partials/_toast.html` â€” `motion-reduce:` Tailwind variants for reduced motion.

### Task C â€” `c-equation` (new, client-side KaTeX)

**Vendor KaTeX** (pin a version, e.g. `katex@0.16.x`) into
`freedom_ls/content_engine/static/content_engine/vendor/katex/`:
`katex.min.css`, `katex.min.js`, and the `fonts/` directory (the CSS references
fonts by relative URL, so the folder layout must be preserved). Vendor locally â€”
**no CDN** (offline-safe, smaller sanitiser/network surface). Loaded via Task A.

`cotton/equation.html`. Attribute: `label` (optional, e.g. "3"). LaTeX delivered as
**text** in the slot (per spec security note â€” never pre-parsed HTML).

```django
<c-vars label="" />
<div class="my-6 flex items-baseline gap-3">
  <div x-data="equation"
       tabindex="0"
       role="math"
       aria-label="{% if label %}Equation {{ label }}{% else %}Equation{% endif %}"
       class="grow overflow-x-auto focus:outline-none focus:ring-2 focus:ring-primary rounded-md">
    <span x-ref="src" class="block">{{ slot }}</span>
    <span x-ref="out" class="block" hidden></span>
  </div>
  {% if label %}<span class="text-muted shrink-0">({{ label }})</span>{% endif %}
</div>
```

`equation` component in `content_engine/.../alpine-components.js`:
```js
Alpine.data("equation", () => ({
  init() {
    const source = this.$refs.src.textContent;
    try {
      window.katex.render(source, this.$refs.out, {
        displayMode: true,
        throwOnError: false,   // malformed LaTeX degrades to readable source
        trust: false,          // SECURITY: block \href \url \includegraphics
        strict: "ignore",
        output: "htmlAndMathml" // MathML for screen readers
      });
      this.$refs.src.hidden = true;
      this.$refs.out.hidden = false;
    } catch (e) {
      // Leave the raw LaTeX source visible as the fallback.
    }
  },
}));
```
- Container is **focusable + horizontally scrollable** (WCAG 1.4.10) for long
  equations.
- Widget-scoped render (only `c-equation` containers) â€” **no** global `$$`
  delimiter scan, so currency like "$5"/"R1000" in prose is untouched.
- **Escape round-trip:** the template emits `{{ slot }}` (Django escapes `<`â†’`&lt;`),
  and the JS reads `this.$refs.src.textContent` â€” `textContent` (not `innerHTML`)
  returns the **decoded** characters, so KaTeX receives the real LaTeX. Authors must
  still escape `<`/`>`/`&` in the **source markdown** (or use `\lt`/`\gt`) so the
  outer nh3 pass doesn't eat them before they reach the slot. Document in the demo.

### Task F â€” `c-image-grid` (new, no carousel) â€” reuses `c-picture`

`cotton/image-grid.html`. **Layout wrapper only**; the slot contains `c-picture`
children (or `![[file|caption]]` shorthand, which `content_save` rewrites to
`c-picture`). Each child keeps its own (now keyboard-accessible) lightbox â€” so the
grid **reuses** the picture lightbox with **zero new modal code**.

```django
<c-vars columns="3" />
{% load content_tags %}
{# Fixed class whitelist â€” author input never becomes raw CSS.
   Emit the class directly inside each branch (no {% with %}: its scope ends at
   {% endwith %}, so a captured var would be empty by the time the markup uses it). #}
{% if columns == "2" %}{% with cols="sm:grid-cols-2" %}
  <div class="my-6 grid grid-cols-1 {{ cols }} gap-4 [&_figure]:my-0! [&_figure]:mx-0! [&_figure]:max-w-none!">{% markdown slot %}</div>
{% endwith %}
{% elif columns == "4" %}{% with cols="sm:grid-cols-2 lg:grid-cols-4" %}
  <div class="my-6 grid grid-cols-1 {{ cols }} gap-4 [&_figure]:my-0! [&_figure]:mx-0! [&_figure]:max-w-none!">{% markdown slot %}</div>
{% endwith %}
{% else %}{# default = 3, also the fallback for any unknown value #}
  <div class="my-6 grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4 [&_figure]:my-0! [&_figure]:mx-0! [&_figure]:max-w-none!">{% markdown slot %}</div>
{% endif %}
```
(The whole `<div>â€¦</div>` lives **inside** each branch so `cols` is in scope. Or

- `columns` maps through a **fixed** whitelist (`2`/`3`/`4` â†’ fixed Tailwind class);
  any other value falls through to the default 3-column branch. Never interpolate
  `columns` straight into a class string.
- Arbitrary-variant `[&_figure]:â€¦` neutralises `c-picture`'s `max-w-xl/mx-auto/my-6`
  so figures tile â€” no change to `c-picture` needed.
- A11y inherited from `c-picture`: each thumbnail is a real button with an
  accessible name; grid reflows to 1 column on small screens; no auto-advance, no
  motion.
- Also document the **stacked captioned figures** pattern (just consecutive
  `c-picture`) as the second supported gallery shape.

### Task G â€” `c-table` (new) â€” purpose-built, NOT `c-data-table`

`cotton/table.html`. Author writes a **GitHub-flavoured markdown table** in the slot
(the `tables` extension is already enabled). Widget supplies the `<caption>`, the
focusable scrollable wrapper, and brand styling. **Independent** of the base app's
`c-data-table` / `scrollTableLabels` / `scroll-table-labels.html` (do not import or
reuse them).

New `content_tags` filter `inject_table_caption(rendered_html, caption)` â€” inserts a
`<caption>` as the first child of the first `<table>` so the table has a real,
SR-associated caption (markdown's `tables` extension emits no caption). Small,
string-level, unit-tested.

```django
<c-vars caption="" />
{% load content_tags %}
{% markdown slot as rendered_table %}
<div class="my-6 overflow-x-auto focus:outline-none focus:ring-2 focus:ring-primary rounded-md"
     tabindex="0" role="group" aria-label="{{ caption|default:'Data table' }}">
  {% if caption %}{{ rendered_table|inject_table_caption:caption }}
  {% else %}{{ rendered_table }}{% endif %}
</div>
```
- `inject_table_caption` pseudocode (the **only** new Python â€” give it full type
  hints, no `Any`): find the first `<tableâ€¦>`, splice
  `<caption class="...">{escaped caption}</caption>` immediately after it; if there's
  no `<table>`, return the input unchanged (no-op); escape the caption text (author
  input); return `mark_safe`. Unit-test the no-op and escaping cases.
- Column-header semantics come from the markdown `tables` extension's `<thead><th>`
  (note: that extension emits `<th>` **without** an explicit `scope` and never makes
  the first body cell a row-header).
- To genuinely satisfy success-criterion Â§6 ("tables have caption + `scope`"), the
  **comparison-table demo authors a raw HTML `<table>`** with explicit
  `<th scope="col">` / `<th scope="row">`. nh3's defaults **keep** `scope` on
  `th`/`td` (and `colspan`/`rowspan`), so this survives the outer sanitiser with **no
  settings change** â€” confirm with a render test asserting `scope=` is present. (Only
  if a test ever shows `scope` being stripped, add `"th"/"td" â†’ {"scope"}` to the nh3
  attribute map inside `render_markdown`; not expected to be needed.) The simpler
  markdown-table form remains supported for tables that only need column headers.
- **No** automatic row-stacking (breaks comparison tables / semantics). Sticky
  header / scroll cue via CSS where feasible.
