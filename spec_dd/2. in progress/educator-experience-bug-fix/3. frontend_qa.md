# Frontend QA: Educator Experience Bug Fix

## Prerequisites

- Pick an available port and start the dev server:
  - `PORT=$(.claude/fls/scripts/find_available_port.sh)`
  - `uv run python manage.py runserver $PORT`
- Base URL is `http://127.0.0.1:$PORT`.
- Use the **DemoDev** site for all test data. Verify by running the seed/demo content commands if the site is empty.
- An educator user with access to `/educator/` is available. (Use the standard demo educator account.)
- A cohort exists with **enough students and course registrations to require pagination** — at least 6 students (data-table page size is 5) and at least one course registration. If the demo data does not produce more than one page, register additional demo students into a cohort using the existing demo factories or the `manage.py` shell (look up the DemoDev `Site` by name and attach all data to it).
- A course exists with multiple cohort registrations and multiple direct student registrations (again, more than 5 of each so pagination renders).

## Login

1. Visit `http://127.0.0.1:$PORT/accounts/login/`.
2. Sign in as the educator account.

## Test 1: Cohort details — students table no longer recurses on sort

**Goal:** Verify Bug 1 is fixed for `CohortStudentsPanel`.

1. Navigate to `/educator/cohorts/` and click the cohort that has multiple students.
2. The cohort details page loads with several panels visible. Locate the **Students** panel near the bottom.
3. Confirm the panel has a header with title and search bar, an action button (if applicable), and a table.
4. Click any sortable column header (e.g., **Email**) once to sort ascending.
5. Click the same column header again to sort descending.
6. Click the column header a third time.

**Pass criteria:**
- After every click, only the table rows update.
- There is **no nested panel header, no duplicate search bar, no nested table** appearing inside the panel.
- The page remains usable indefinitely.

## Test 8: Pagination — numbered buttons render across all data tables

**Goal:** Verify Bug 2 — `<c-pagination>` renders consistently.

1. Navigate to `/educator/users/`. Scroll to the bottom of the user table.
2. Confirm pagination shows numbered buttons: First, Previous, 1, 2, …, N, Next, Last (desktop). On mobile width (< 640px), confirm it shows Previous, "Page X of Y", Next.
3. Click page 2. Confirm only the table rows update and the URL/page indicator reflects page 2.
4. Repeat at `/educator/cohorts/` and `/educator/courses/`.
5. On the cohort details page, scroll to the Students panel and confirm the same numbered-button pagination appears under the table.

**Pass criteria:** Identical numbered-button pagination on every list and panel data table. Mobile responsive split still works. HTMX page changes update only the table.

## Test 11: Course progress panel — independent paginators

**Goal:** Clicking one paginator must not reset the other.

1. Same page as Test 10. The course-progress panel has two paginators: one for course items (columns), one for students (rows).
2. Advance the **student paginator** to page 2.
3. Now click the **next** button on the course-items paginator.
4. After the swap completes, confirm the **student paginator is still on page 2** (not reset to page 1).
5. Conversely, reset both to page 1, then advance the **course-items paginator** to page 2, then click student-paginator next. Confirm the course-items paginator stays on page 2.

**Pass criteria:** Each paginator's state is preserved when the other paginator is clicked.
