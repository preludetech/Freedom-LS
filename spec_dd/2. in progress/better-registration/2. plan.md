This plan implements `1. spec.md`. We use TDD throughout: each task starts with one failing test, then adds the minimum implementation, then expands tests as needed. Manual verification steps are out of scope for this file.

Conventions reminder (see `CLAUDE.md`):
- Type hints on all functions; no `Any`.
- ORM only.
- Use `select_related()` / `prefetch_related()` for related-object queries.
- HTMX validation errors return HTTP 422 (not relevant for this feature unless we adopt HTMX in the completion view — we don't).
- After model changes: `uv run manage.py makemigrations` then `uv run manage.py migrate`.
- Never edit existing migration files — create new ones.
- URL names use `snake_case`; URL paths use `kebab-case`.
- Modern Python syntax (`X | None`, `list[str]`, etc.).

## Phase 0 — Spec sanity & scaffolding

### Task 0.1 — Confirm spec reading; introduce shared helpers module

There are no contradictions in the spec to resolve. Proceed.

Create a single internal helper module to keep the rest of the work DRY:

- New file: `freedom_ls/accounts/utils.py`
  - `get_client_ip(request: HttpRequest) -> str` — trusted-proxy aware. Reads `settings.TRUSTED_PROXY_IP_HEADER` (string, default `None`). When set, returns the leftmost IP in that header's value (after stripping whitespace). When `None`, returns `request.META["REMOTE_ADDR"]`. Returns `""` if neither is available. This is the only sanctioned way to derive the client IP.
  - `get_signup_policy_for_request(request: HttpRequest) -> SiteSignupPolicy | None` — returns the policy for the current site or `None`. Used by signup form, completion middleware, legal-doc loader, and the adapter.

Tests (`freedom_ls/accounts/tests/test_utils.py`):
- `get_client_ip` returns `REMOTE_ADDR` when no trusted-proxy header is configured.
- `get_client_ip` returns the leftmost value of the configured header when `TRUSTED_PROXY_IP_HEADER` is set.
- `get_client_ip` falls back to `""` when nothing is set.
- `get_signup_policy_for_request` returns `None` when no policy exists; returns the right `SiteSignupPolicy` when one does.

### Task 2.2 — `legal_docs.py` module

New file: `freedom_ls/accounts/legal_docs.py`. Public API:

```python
LEGAL_DOCS_ROOT: Path = settings.BASE_DIR / "legal_docs"
# Reject leading dot/dash and any `..` sequence as defence-in-depth before the
# resolve()-stays-inside-LEGAL_DOCS_ROOT check below.
SITE_DOMAIN_RE = re.compile(r"^(?!\.|-)(?!.*\.\.)[A-Za-z0-9][A-Za-z0-9.\-]*[A-Za-z0-9]$|^[A-Za-z0-9]$")

@dataclass(frozen=True)
class LegalDoc:
    doc_type: str            # "terms" | "privacy"
    site_domain: str         # actual domain used (may be "_default")
    relative_path: str       # e.g. "legal_docs/example.com/terms.md"
    version: str
    title: str
    effective_date: str
    body_markdown: str
    git_hash: str            # blob SHA at HEAD

def get_legal_doc(site: Site, doc_type: str) -> LegalDoc | None: ...
def render_legal_doc(doc: LegalDoc, request: HttpRequest) -> str: ...
def has_legal_doc(site: Site, doc_type: str) -> bool: ...
```

Pseudocode for `get_legal_doc`:

```
1. Validate doc_type in {"terms", "privacy"}.
2. Validate site.domain matches SITE_DOMAIN_RE; if not, log WARNING and treat as no
   site-specific doc (fall through to _default).
3. Candidate paths, in order:
     legal_docs/<site.domain>/<doc_type>.md
     legal_docs/_default/<doc_type>.md
   For each candidate:
     a. Resolve to an absolute path; assert the resolved path is inside LEGAL_DOCS_ROOT
        (path-traversal guard). Reject otherwise.
     b. Read the file content from the git blob at HEAD using a `read_blob_at_head(rel_path)`
        helper (see below). If the blob does not exist, continue.
     c. Parse with `yaml.safe_load` for frontmatter; on failure log and continue.
     d. Return LegalDoc(...).
4. If no candidate succeeded, return None.
```

Implementation of `read_blob_at_head(rel_path)` — encapsulates the two deployment modes from spec 3.2:

```
1. If `settings.LEGAL_DOCS_MANIFEST_PATH` is set and the file exists, load the JSON manifest:
       { "head_commit": "...", "blobs": { "<rel_path>": {"sha": "...", "content_b64": "..."} } }
   Use the manifest entry directly. The build step writes this file (see Task 2.5).
2. Otherwise (typical dev / git checkout deploy):
       run `git -C BASE_DIR rev-parse HEAD:<rel_path>` for the SHA
       run `git -C BASE_DIR show HEAD:<rel_path>` for the content
   Use `subprocess.run` with `check=True`, no shell, list args.
3. Return (sha, content_text). Raise FileNotFoundError on missing.
```

Markdown rendering uses `freedom_ls.markdown_rendering.markdown_utils.render_markdown(body_markdown, request)` — the only sanctioned route. (This module is extracted from `content_engine` in Task 0.3 specifically so that `accounts` can depend on it without taking a runtime edge on `content_engine`.) Frontmatter is parsed with `yaml.safe_load`. Frontmatter is split on a leading `^---\n...\n---\n` block; reject if not present.

### Task 2.5 — Manifest build step (deferred / documented only)

Add a management command stub `freedom_ls/accounts/management/commands/build_legal_docs_manifest.py` that walks `legal_docs/` and writes the manifest used by `read_blob_at_head` when no `.git` directory exists.

Pseudocode:

```
for path in iter LEGAL_DOCS_ROOT/*/*.md:
    rel = path.relative_to(BASE_DIR)
    sha = subprocess(["git", "-C", BASE_DIR, "rev-parse", f"HEAD:{rel}"]).stdout.strip()
    content = path.read_bytes()  # working tree is fine at build time, since we record HEAD's SHA
    blobs[str(rel)] = {"sha": sha, "content_b64": base64.b64encode(content).decode()}
manifest = {"head_commit": HEAD_SHA, "blobs": blobs}
write JSON to settings.LEGAL_DOCS_MANIFEST_PATH
```

Also add `LEGAL_DOCS_MANIFEST_PATH: str | None = None` in `config/settings_base.py`.

**Manifest trust boundary.** When the manifest mode is active, the manifest file *is* the source of truth — there is no `git show HEAD:...` available to cross-check. The manifest must therefore be treated as part of the immutable build artifact (baked into the container image; not stored on a writable volume; not regenerated at runtime). Operationally this means: (a) document that `LEGAL_DOCS_MANIFEST_PATH` should point inside the read-only image filesystem, (b) the runtime MUST NOT re-run `build_legal_docs_manifest` automatically, and (c) the V8.1 mitigation is weaker in this mode than in the git-checkout mode — an attacker with write access to the manifest file controls both content and recorded hash. Note this trade-off in the manifest command's docstring.

## Phase 3 — Additional registration forms loader & protocol

### Task 3.1 — `registration_forms.py`

New file: `freedom_ls/accounts/registration_forms.py`. Public API:

```python
class RegistrationFormProtocol(Protocol):
    @classmethod
    def applies_to(cls, user) -> bool: ...
    @classmethod
    def is_complete(cls, user) -> bool: ...
    def save(self, user) -> None: ...

def load_registration_form_classes(dotted_paths: list[str]) -> list[type[forms.Form]]:
    """Resolve dotted paths to Form classes, validating protocol and skipping bad entries.

    For each path:
      1. import_string(path) — on ImportError/AttributeError, log WARNING and skip.
      2. Verify it's a class and a subclass of django.forms.Form. If not, log and skip.
      3. Verify it exposes applies_to (callable, classmethod), is_complete (same), save.
      4. Verify no field name in cls.base_fields is in {"user", "user_id", "email"}.
         If any are present, log WARNING and skip.
      5. Append to result list.
    Never raises into the caller. A misconfigured entry degrades gracefully.
    """

def get_incomplete_forms(user, dotted_paths: list[str]) -> list[type[forms.Form]]:
    """Return form classes that apply to `user` and aren't yet complete.

    Short-circuit: if user.is_superuser, return []. (Baseline safety net so a
    misconfigured form can never lock out a superuser.)
    """
```

## Phase 4 — Custom signup form

### Task 4.1 — `SiteAwareSignupForm`

Edit `freedom_ls/accounts/forms.py` (or split into `signup_form.py` if it gets large). Add:

```python
from allauth.account.forms import SignupForm
# ... etc

class SiteAwareSignupForm(SignupForm):
    first_name = forms.CharField(max_length=200, required=True)
    last_name = forms.CharField(max_length=200, required=False)

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        request = self._get_request()
        site = get_cached_site(request) if request else None
        policy = get_signup_policy_for_request(request) if request else None

        # Name handling
        if policy and not policy.require_name:
            self.fields["first_name"].required = False

        # Terms / privacy handling
        if policy and policy.require_terms_acceptance and site is not None:
            from .legal_docs import has_legal_doc
            if has_legal_doc(site, "terms"):
                self.fields["accept_terms"] = forms.BooleanField(
                    required=True,
                    label=_("I accept the Terms and Conditions"),
                )
            else:
                logger.warning("Site %s has require_terms_acceptance=True but no terms doc", site.domain)
            if has_legal_doc(site, "privacy"):
                self.fields["accept_privacy"] = forms.BooleanField(
                    required=True,
                    label=_("I accept the Privacy Policy"),
                )
            else:
                logger.warning(...)

    def custom_signup(self, request, user):
        # allauth calls this after the user is saved.
        site = get_cached_site(request)
        ip = get_client_ip(request)
        # Wrap the consent inserts so a partial failure does not leave a user
        # with one consent row (e.g. terms saved, privacy DB error). The user
        # has already been created by the time custom_signup runs, so we cannot
        # roll the user back here — but we must not record asymmetric consent.
        with transaction.atomic():
            for doc_type, accepted_field in [("terms", "accept_terms"), ("privacy", "accept_privacy")]:
                if not self.cleaned_data.get(accepted_field):
                    continue
                doc = get_legal_doc(site, doc_type)
                if doc is None:
                    # Form would not have asked; defensive
                    continue
                LegalConsent.objects.create(
                    user=user,
                    document_type=doc_type,
                    document_version=doc.version,
                    git_hash=doc.git_hash,
                    ip_address=ip or None,
                    consent_method="signup_checkbox",
                )
```

`_get_request()` should pull the request via `allauth.core.context.request` (used elsewhere in `allauth_account_adapter.py`) and fall back to `None` if not available.

**Fail-closed when request is unavailable.** If `_get_request()` returns `None`, the form cannot determine the current site/policy. In that path, `__init__` MUST NOT silently skip the terms/privacy checkboxes when `require_terms_acceptance` would otherwise apply — that would let a signup proceed without consent capture. The form is constructed in two contexts:

1. Normal request flow — `request` is set; standard logic applies.
2. Out-of-request flow (rare; e.g. management command instantiating the form) — the form should not be submittable as a signup. Detect this and either raise (fail closed) or refuse to render any fields tied to policy. The signup HTTP endpoint always has a request, so this is a defence-in-depth safeguard, not a routine path.

## Phase 6 — Completion view

### Task 6.1 — `complete_registration_view`

Edit `freedom_ls/accounts/views.py`:

```python
@login_required
def complete_registration_view(request: HttpRequest) -> HttpResponse:
    policy = get_signup_policy_for_request(request)
    dotted_paths = policy.additional_registration_forms if policy else []
    incomplete = get_incomplete_forms(request.user, dotted_paths)

    if not incomplete:
        return _safe_post_completion_redirect(request)

    if request.method == "POST":
        bound = [cls(request.POST, prefix=cls.__name__) for cls in incomplete]
        if all(f.is_valid() for f in bound):
            for f in bound:
                f.save(request.user)  # always request.user — never trust submitted user id
            _invalidate_completion_cache(request)
            return _safe_post_completion_redirect(request)
    else:
        bound = [cls(prefix=cls.__name__) for cls in incomplete]

    return render(request, "accounts/complete_registration.html", {"forms": bound})


def _safe_post_completion_redirect(request: HttpRequest) -> HttpResponse:
    next_url = request.GET.get("next") or request.POST.get("next")
    if next_url and url_has_allowed_host_and_scheme(
        next_url, allowed_hosts={request.get_host()}, require_https=request.is_secure()
    ):
        return redirect(next_url)
    return redirect(getattr(settings, "LOGIN_REDIRECT_URL", "/"))
```

Each form uses `prefix=cls.__name__` to avoid field name collisions on a single page.

## Phase 7 — `RegistrationCompletionMiddleware`

### Task 7.1 — Middleware

New file: `freedom_ls/accounts/middleware.py`.

```python
EXEMPT_URL_NAMES: set[str] = {
    "account_login", "account_logout", "account_signup",
    "account_email_verification_sent", "account_confirm_email", "account_email",
    "account_reset_password", "account_reset_password_done",
    "account_reset_password_from_key", "account_reset_password_from_key_done",
    "accounts:legal_doc", "accounts:complete_registration",
}
EXEMPT_PATH_PREFIXES: tuple[str, ...] = ("/static/", "/media/", "/health/")

CACHE_SESSION_KEY = "_registration_completion_state"

class RegistrationCompletionMiddleware:
    def __init__(self, get_response): ...

    def __call__(self, request):
        if not self._should_check(request):
            return self.get_response(request)

        if self._is_exempt(request):
            return self.get_response(request)

        if request.user.is_superuser:
            return self.get_response(request)

        if self._is_complete_cached(request):
            return self.get_response(request)

        policy = get_signup_policy_for_request(request)
        dotted_paths = policy.additional_registration_forms if policy else []
        incomplete = get_incomplete_forms(request.user, dotted_paths)

        if not incomplete:
            self._mark_complete(request, dotted_paths)
            return self.get_response(request)

        return redirect(reverse("accounts:complete_registration"))
```

Exempt-match strategy (per threat-model V1.2):
- `_is_exempt(request)`:
  1. `request.path.startswith(prefix)` for any prefix in `EXEMPT_PATH_PREFIXES`.
  2. `resolve(request.path).view_name` (the `app_name:url_name`) is in `EXEMPT_URL_NAMES`.
  No substring / `in` matching.

Cache key:
- `dotted_paths_hash = hashlib.sha256(json.dumps(sorted(dotted_paths)).encode()).hexdigest()`

Stored in `request.session[CACHE_SESSION_KEY]` as `{"hash": "<dotted_paths_hash>"}`. No user-version component, no signal-driven invalidation.

**Decision (2026-04-27):** admin-edit invalidation is intentionally relaxed (see updated spec 4.4 / threat-model V4.3). If an admin wipes data that a registration form looks at, the affected user can remain "complete" for the rest of the current session and only get redirected to the completion view on next login / next session. Acceptable trade-off in exchange for not adding a `user_version` field, post_save signals, or session-walking invalidation logic.

The cache invalidates explicitly on:
1. Successful submit of the completion view (Task 7.3 below).
2. Site policy change (handled automatically by the dotted-paths hash — a different list produces a different cache key, so the previous "complete" entry is simply ignored).
3. New session (the cache lives in `request.session`, so logout/login starts fresh).
