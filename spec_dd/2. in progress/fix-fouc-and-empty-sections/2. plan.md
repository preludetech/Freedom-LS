# Implementation Plan: Fix FOUC on Alpine modals and hide empty dashboard sections

This plan turns `1. spec.md` into concrete, ordered tasks. Two independent slices:

- **Part A** — add the global `[x-cloak] { display: none !important; }` CSS rule and patch one missing `x-cloak` so Alpine `x-show` elements stop flashing on first paint.
- **Part B** — wrap the `recommended-courses` and `learning-history` partials in `{% if %}` so they emit zero DOM when the queryset is empty.

Both slices follow TDD where there is a meaningful behavioural assertion to make. The Tailwind change is verified by inspecting the built CSS output (no Python test).

## Files involved

### Part A (FOUC)

- `tailwind.components.css` — add the `[x-cloak]` rule inside `@layer base`. This file is already imported from `tailwind.input.css` and already hosts other `@layer base` rules, so it is the most consistent home.
- `static/vendor/tailwind.output.css` — produced by `npm run tailwind_build`. Verified, not edited.
- `freedom_ls/student_interface/templates/student_interface/partials/course_minimal_toc.html` — add `x-cloak` to the nested `<div x-show="expanded" x-collapse>` at line ~127 (the children container of a course part).
- `freedom_ls/base/templates/cotton/dropdown-menu.html` — add `x-cloak` to the dropdown panel `<div x-show="open" …>` at line ~21 (it has an inline `style="display: none"` fallback that can stay or go).
- `freedom_ls/content_engine/templates/cotton/picture.html` — already correctly `x-cloak`-protected. No template edit needed; the CSS rule is the fix.

The other audited templates already match the spec's audit table and need no change.

### Part B (empty dashboard sections)

- `freedom_ls/student_interface/templates/student_interface/partials/course_list.html` — the `partialdef recommended-courses` and `partialdef learning-history` blocks. Wrap each entire `<div class="surface">…</div>` in `{% if recommended_courses %}` / `{% if completed_courses %}` respectively. The inner else-branch (`No recommended courses yet.` / `No completed courses yet`) is removed entirely along with the `<h2>` and the surface wrapper.
- `freedom_ls/student_interface/views.py` — `partial_list_courses` (already passes `recommended_courses` and `completed_courses` into the template context). No view change needed; the template guard is sufficient.
- `freedom_ls/student_interface/tests/test_course_list_views.py` — extend with rendered-HTML assertions for the empty case and the populated case. The file already imports `RecommendedCourseFactory`, `UserCourseRegistrationFactory`, `CourseProgressFactory`, and the `courses` fixture, so we reuse them.

### Existing functionality to reuse (DRY)

- `get_completed_courses`, `get_recommended_courses`, `get_current_courses` already drive the context — no new helpers.
- `RecommendedCourseFactory` and `CourseProgressFactory` (via `mark_complete`-style helpers in the existing tests) already generate the data shapes the new tests need.
- `mock_site_context` fixture is already used by every test in this module.

## Task list

### A1. Add the `[x-cloak]` global CSS rule

Edit `tailwind.components.css`. Inside the existing `@layer base { … }` block (alongside the body / heading / paragraph rules), add:

```css
[x-cloak] { display: none !important; }
```

Rationale for this location: the spec offers `tailwind.input.css` *or* `tailwind.components.css`. The components file already owns `@layer base` and is where every other base-layer rule lives in this project, so it is the consistent home.

### A2. Patch the two `x-show`-without-`x-cloak` cases identified in the audit

- `course_minimal_toc.html` — add `x-cloak` to the `<div x-show="expanded" x-collapse class="ml-6 mt-1 …">` element. Keep `x-collapse`. Pseudocode:

  ```
  <div x-show="expanded"
       x-collapse
       x-cloak
       class="ml-6 mt-1 space-y-1 border-l-2 border-border pl-3">
  ```

- `cotton/dropdown-menu.html` — add `x-cloak` to the dropdown panel `<div x-show="open" …>`. The inline `style="display: none"` fallback can stay; it remains a belt-and-braces guard for any environment where the global rule is unexpectedly stripped, and it does not conflict with `x-cloak`. Pseudocode:

  ```
  <div x-show="open"
       x-cloak
       … (existing transitions / refs / classes) …
       style="display: none">
  ```

The other audited templates are confirmed correct and require no edits:

- `cotton/picture.html` — already has `x-cloak` on the `x-show="open"` overlay.
- `_base_interface.html` — both `x-show` elements already cloaked.
- `cotton/modal.html` — outer wrapper carries `x-cloak`, hiding the entire `x-show="open"` subtree at parse time. No nested cloak needed.
- `panel_framework/partials/sidebar_nav.html` — every `x-show` already cloaked.
- `partials/messages.html` — `x-show="show"` starts open by design (toast enter-animation). Leave as-is, document in the PR description.
- `_base.html` debug branch badge — dev-only chrome with `x-transition.opacity`. Leave as-is, document in the PR description.

### B2. Implement the template guard

Edit `freedom_ls/student_interface/templates/student_interface/partials/course_list.html`:

- Replace the entire `partialdef recommended-courses` body with a guarded version. Pseudocode:

  ```
  {% partialdef recommended-courses %}
      {% if recommended_courses %}
          <div class="surface space-y-4">
              <h2>Recommended Courses</h2>
              <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
                  {% for recommendation in recommended_courses %}
                      {% with course=recommendation.collection %}
                          {% partial course-card %}
                      {% endwith %}

              </div>
          </div>
      {% endif %}
  {% endpartialdef %}
  ```

- `partialdef all-courses` is unchanged. The "No courses available yet." dashboard-wide empty state stays.

- The outer wrapper `<div class="space-y-4">` and the trailing `c-button-group` for the "All Courses" link stay unchanged.

## Risk notes

- **Tailwind purge.** Attribute selectors with no class reference can in principle be stripped by aggressive purging. The spec mitigates this by placing the rule in `@layer base` inside `tailwind.components.css`, which is imported from `tailwind.input.css`. A3 verifies the rule survives the build.
- **Toast / debug-badge regressions.** Both are intentionally left without `x-cloak` per the spec. If a future contributor adds `x-cloak` to either one without redesigning the enter transition, the toast and debug badge would no longer animate on first paint. Document this in the PR description audit table so it is not lost.
- **`x-cloak` on `cotton/dropdown-menu.html`.** The element already starts with inline `style="display:none"`; adding `x-cloak` makes the rule consistent with the rest of the codebase and is harmless.
- **Test fragility on string-match assertions.** Asserting `"Recommended Courses" not in body` is robust because the literal heading text only appears in this partial. If the heading text changes in a future redesign, the assertion will fail loudly — that is the desired behaviour.
