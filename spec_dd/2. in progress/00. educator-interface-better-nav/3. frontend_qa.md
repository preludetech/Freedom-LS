# Frontend QA: Educator Interface Better Navigation

## Setup

```bash
PORT=$(./find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

Login as an educator user. Navigate to `http://127.0.0.1:$PORT/educator-interface/` (the educator interface root).

---

## Test 0: Educator Root Page

### Steps
1. After login, observe the educator interface root page at `http://127.0.0.1:$PORT/educator-interface/`

### Expected
- Breadcrumbs are visible showing only `Educator` (current page, not a link)
- The breadcrumb `Educator` has `aria-current="page"`
- No sidebar item is highlighted as active
- No sidebar section is expanded

---

## Test 1: Sidebar Active Highlighting

### Steps
1. Navigate to `http://127.0.0.1:$PORT/educator-interface/cohorts/`
2. Observe the sidebar

### Expected
- The "Cohorts" sidebar link has a visually distinct active state: background fill, left accent bar, and/or semi-bold text
- "Users" and "Courses" sidebar links are NOT highlighted
- The active link has `aria-current="page"` attribute (inspect in dev tools)

### Steps (continued)
3. Click "Users" in the sidebar
4. Observe the sidebar

### Expected
- "Users" is now highlighted with the active style
- "Cohorts" is no longer highlighted
- Hovering over "Courses" shows a subtle hover effect that is visibly different from the active state
- Note: active and hover states should meet WCAG 2.1 AA contrast requirements (4.5:1 for text, 3:1 for UI components). Use a contrast checker if needed

## Test 2: Breadcrumbs on List Pages

### Steps
1. Navigate to `http://127.0.0.1:$PORT/educator-interface/cohorts/`

### Expected
- Breadcrumbs visible at the top of the main content area: `Educator > Cohorts`
- "Educator" is a clickable link
- "Cohorts" is NOT a link (it's the current page)
- The breadcrumb is inside a `<nav aria-label="Breadcrumb">` element containing an `<ol>` (ordered list, not `<ul>`)
- The last item has `aria-current="page"`
- Separators between breadcrumb items (e.g. `/`) are rendered via CSS `::before` pseudo-elements, NOT as inline text (inspect in dev tools — there should be no separator text in the DOM)
- Tab through the breadcrumb links — each should show a visible focus indicator (outline/ring)

### Steps (continued)
2. Click "Users" in the sidebar

### Expected
- Breadcrumbs update to: `Educator > Users`
- "Users" is the current page (not a link)

## Test 3: Breadcrumbs on Instance Pages

### Steps
1. Navigate to `http://127.0.0.1:$PORT/educator-interface/cohorts/`
2. Click on a specific cohort name in the data table (e.g. "Data Science 2026")

### Expected
- Breadcrumbs show: `Educator > Cohorts > Data Science 2026`
- "Educator" and "Cohorts" are clickable links
- "Data Science 2026" (the instance name) is NOT a link

## Test 4: HTMX Navigation (No Full Page Reload)

### Steps
1. Navigate to `http://127.0.0.1:$PORT/educator-interface/cohorts/`
2. Open browser dev tools → Network tab
3. Click "Users" in the sidebar

### Expected
- The page does NOT fully reload (no full document request in Network tab — only an XHR/fetch request)
- The main content area updates to show the Users list
- The browser URL updates to `.../users/`
- Breadcrumbs update to `Educator > Users`
- Sidebar highlighting moves to "Users"
- No flash of unstyled or stale content during the transition — the swap should appear instant and clean

### Steps (continued)
4. Click on a specific user name in the data table

### Expected
- No full page reload — content swaps via HTMX
- URL updates to `.../users/<id>/`
- Breadcrumbs update to `Educator > Users > <User Name>`

## Test 8: Sidebar Instance Dropdown — Keyboard Navigation

### Steps
1. Navigate to `http://127.0.0.1:$PORT/educator-interface/cohorts/` and click on a specific cohort in the data table to reach an instance page
2. Use Tab key to navigate to the sidebar toggle/expand button next to "Cohorts"
3. Press Enter or Space

### Expected

- The toggle button has `aria-expanded="true"` when expanded and `aria-expanded="false"` when collapsed (inspect in dev tools)
