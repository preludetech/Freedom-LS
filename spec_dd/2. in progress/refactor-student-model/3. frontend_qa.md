# Frontend QA: Remove the Student Model

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- Demo data loaded: `uv run python manage.py create_demo_data`
- Log in as an educator user (check demo data for credentials)

---

## Test 1: Educator Interface — User List

**Goal:** Verify the educator can see users (previously "students") listed correctly.

1. Log in as an educator
2. Navigate to the educator interface
3. Look for a "Users" section (previously "Students")
4. Click into the Users list

**Expected:**
- The URL path should contain `/users/` (not `/students/`)
- The page title/header should say "Users" (not "Students")
- A data table should display with columns: First Name, Last Name, Email, Cohorts, Registered Courses
- All users who are in cohorts or have course registrations should appear
- Clicking a user's name should navigate to a user detail page at a `/users/{id}` path

---

## Test 2: Educator Interface — User Detail

**Goal:** Verify the user detail page works correctly.

1. From the user list (Test 1), click on a user's name
2. The detail view should load

**Expected:**
- URL contains `/users/{id}` (not `/students/{id}`)
- User details panel shows: First Name, Last Name, Email
- The old Student-specific fields (ID Number, Date of Birth, Cellphone) should NOT appear
- Cohorts panel shows the cohorts this user belongs to

---

## Test 3: Educator Interface — Cohort Detail with User Progress

**Goal:** Verify the cohort course progress panel still works after the refactor.

1. Navigate to a cohort that has members and a registered course
2. Click on the course progress panel

**Expected:**
- User names are displayed correctly in the progress table
- Progress percentages/scores load correctly
- Pagination works if there are many users
- No errors in the browser console

---

## Test 4: Student Interface — Course Registration

**Goal:** Verify a user can still register for a course.

1. Log in as a regular user (not an educator)
2. Navigate to the student interface home page
3. Browse available courses
4. Click "Register" on a course

**Expected:**
- Registration succeeds without errors
- The course appears in the user's course list
- No mention of "Student" in any visible UI text or error messages

---

## Test 5: Student Interface — Course Navigation with Deadlines

**Goal:** Verify deadline-related features work after the refactor.

1. Log in as a user who is part of a cohort with deadline-configured courses
2. Navigate to a course with deadlines
3. Browse through course items

**Expected:**
- Locked items (past deadline) show lock indicators
- Unlocked items are accessible
- No errors when navigating between items

---

## Test 6: Admin Interface

**Goal:** Verify admin pages work for renamed models.

1. Log in as a superuser
2. Navigate to `/admin/`
3. Look for the Student Management section

**Expected:**
- "Student" model should NOT appear in admin
- "User Course Registration" should appear (previously "Student Course Registration")
- "User Cohort Deadline Override" should appear (previously "Student Cohort Deadline Override")
- Clicking into each model's list should load without errors
- Creating/editing records should work correctly
- Search functionality should work (search by user name/email)

---

## Test 7: Educator Interface — Cohort Member Count

**Goal:** Verify cohort list shows correct member counts.

1. Navigate to the educator interface cohort list

**Expected:**
- Each cohort shows an "Active Users" count (not "Active Students")
- The count matches the actual number of members
- Clicking into a cohort shows the correct members

---

## Test 8: No Stale "Student" Text in UI

**Goal:** Verify that no user-facing page still says "Student" or "Students" where it should say "User" or "Users".

1. Navigate through all educator interface pages: user list, user detail, cohort list, cohort detail, course list
2. Check page titles, breadcrumbs, column headers, button labels, and panel headings

**Expected:**
- No instance of "Student" or "Students" appears in the educator interface UI (except in app names like "Student Management" which are intentionally kept)
- All references should say "User" or "Users"

---

## Test 9: Management Commands

**Goal:** Verify QA helper commands still work after the refactor.

1. Run `uv run python manage.py create_demo_data` — should complete without errors
2. Run any available `qa_*` commands that create test data

**Expected:**
- Commands complete without errors
- Data is created correctly (verify in admin or by browsing the site)
