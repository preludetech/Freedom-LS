# Frontend QA: Debug Branch Indicator Panel

## Prerequisites

- Dev server running: `uv run python manage.py runserver`
- `DEBUG=True` (default in dev settings)
- Login credentials: `demodev@email.com` / `demodev@email.com`

---

## Test 1: Badge Visibility on Page Load

**Steps:**
1. Navigate to `http://127.0.0.1:8000/`
2. Log in with demo credentials if not already logged in

**Expected:**
- A small colored pill/badge is visible in the bottom-left corner of the page
- The badge displays the current git branch name (e.g., `debug-show-current-branch`)
- The badge has a colored background with readable text (good contrast)
- The badge uses a monospace font
- The badge defaults to the **expanded** state (branch name visible)

---

## Test 2: Badge Appears on All Pages

**Steps:**
1. Navigate to several different pages:
   - Home page: `http://127.0.0.1:8000/`
   - Student course list: `http://127.0.0.1:8000/student/courses/`
   - Educator dashboard: `http://127.0.0.1:8000/educator/`
   - Login page: `http://127.0.0.1:8000/accounts/login/`

**Expected:**
- The badge appears on every page in the same position (bottom-left corner)
- The color and text are consistent across all pages (same branch = same color)

---

## Test 3: Click to Collapse

**Steps:**
1. On any page, click the debug badge

**Expected:**
- The badge collapses to a small colored dot (no text visible)
- The transition is smooth (animated)
- The dot remains in the bottom-left corner

---

## Test 4: Click to Expand

**Steps:**
1. With the badge in collapsed (dot) state, click it again

**Expected:**
- The badge expands back to show the full branch name
- The transition is smooth

---

## Test 5: State Persistence Across Page Loads

**Steps:**
1. Click the badge to collapse it
2. Navigate to a different page (e.g., click a link or type a new URL)
3. Observe the badge on the new page

**Expected:**
- The badge remains collapsed after navigation
- Reload the page (Ctrl+R) — badge should still be collapsed

**Steps (continued):**
4. Click to expand the badge
5. Navigate to another page

**Expected:**
- The badge remains expanded after navigation

---

## Test 6: First Visit Default State

**Steps:**
1. Open browser developer tools (F12)
2. Go to Application > Local Storage > `http://127.0.0.1:8000`
3. Delete the `debug-branch-expanded` key (or clear all local storage)
4. Reload the page

**Expected:**
- The badge appears in the **expanded** state (showing branch name)

---

## Test 7: Badge Does Not Interfere with Page Content

**Steps:**
1. Navigate to pages with content near the bottom-left
2. Scroll to the bottom of long pages
3. Check that the badge overlays content cleanly

**Expected:**
- The badge floats above page content (fixed position)
- The badge does not push or shift any page elements
- The badge has sufficient z-index to appear above normal content

---

## Test 8: Responsive Behavior (Small Screens)

**Steps:**
1. Open browser developer tools (F12)
2. Toggle device toolbar (Ctrl+Shift+M) or resize the browser window to below 480px width

**Expected:**
- The badge is smaller or more compact on small screens
- It does not obstruct significant page content

---

## Test 9: Correct Branch Name Displayed

**Steps:**
1. In a terminal, run `git branch --show-current` (or check `git rev-parse --abbrev-ref HEAD`)
2. Note the branch name
3. Look at the debug badge in the browser

**Expected:**
- The badge text matches the actual git branch name exactly
- This is especially important in worktree setups where each worktree has a different branch

---

## Test 10: Color Determinism

**Steps:**
1. Note the badge color on the current branch
2. Reload the page multiple times
3. Navigate to different pages

**Expected:**
- The badge color is always the same for the same branch name
- The text is always readable against the background
