# Frontend QA: TOC expand/collapse on course detail pages

## Setup

```bash
PORT=$(./find_available_port.sh)
uv run python manage.py runserver $PORT
```

Base URL: `http://127.0.0.1:$PORT`

Log in as any student user who is registered for the "Functionality Demo Course (Parts)" course.

## Test 1: TOC expand/collapse works on course topic page

1. Navigate to a course topic page, e.g. `http://127.0.0.1:$PORT/courses/functionality-demo-course-parts/2/`
2. Wait for the sidebar TOC to load (it loads via HTMX)
3. In the right-hand sidebar, find a collapsible TOC section (a course part with children)
4. Click on the expand/collapse toggle for that section
5. **Expected:** The section expands to show its children. No console errors.
6. Click the toggle again
7. **Expected:** The section collapses. No console errors.

## Test 2: TOC expand/collapse works on course form page

1. Navigate to a course form page within the same course
2. Wait for the sidebar TOC to load
3. Click to expand a TOC section
4. **Expected:** The section expands without any `Alpine Expression Error` in the console

## Test 3: Expand/collapse state persists across navigation

1. On any course detail page, expand a TOC section
2. Navigate to a different course detail page (e.g. click a different topic in the TOC)
3. **Expected:** The previously expanded section remains expanded (state is stored in localStorage)

## Test 4: No regression on course home page

1. Navigate to the course home page: `http://127.0.0.1:$PORT/courses/functionality-demo-course-parts/`
2. The TOC sections in the main content area should still expand/collapse correctly
3. **Expected:** No console errors, expand/collapse works as before

## Console check

For all tests above, open browser DevTools (F12) and check the Console tab. There should be **no** `Alpine Expression Error` messages about `toggleExpanded` being undefined.
