# Plan: Bugfix — TOC expand/collapse broken on course detail pages

## Summary

The `coursePart` Alpine.js component (which provides `toggleExpanded`) is only loaded on `course_home.html` via the `extra_alpine_components` block. All course detail pages (`course_topic.html`, `course_form.html`, `course_form_page.html`, `course_finish.html`, `course_form_complete.html`) extend `_course_base.html`, which does not override this block. The fix is to add the block override to `_course_base.html`.

## Skills to use

- **fls:playwright-tests** — for writing the E2E behavior test (TDD)
- **fls:template** — for editing `_course_base.html` with correct template conventions
- **fls:alpine-js** — for understanding Alpine.js component loading patterns

## Tasks

### Task 1: Write a failing Playwright test for TOC expand/collapse behavior

**Skill:** `fls:playwright-tests`

Write an E2E test that exercises the actual user-visible behavior: on a course detail page, clicking a course-part toggle in the sidebar TOC expands it to reveal its children, and clicking again collapses it. This tests the interface (does expand/collapse work?), not the implementation (is a specific `<script>` tag present?).

**File to create:** `freedom_ls/student_interface/tests/e2e/test_course_toc.py` (or add to an existing `tests/e2e/` module if one already covers course pages — check `freedom_ls/student_interface/tests/` first).

**Test outline:**

```python
import pytest
from playwright.sync_api import Page, expect

from conftest import reverse_url
# ... factory imports for Course, CoursePart, Topic, student User, UserCourseRegistration

@pytest.mark.playwright
@pytest.mark.django_db(transaction=True)
def test_toc_expands_and_collapses_on_course_detail_page(page: Page, live_server):
    # Arrange: build a course containing at least one CoursePart with a
    # child Topic, register a student, log them in.
    # Navigate to a course DETAIL page (a topic page) — NOT course_home —
    # since the bug only manifests on pages that extend _course_base.html.

    # Act + Assert: locate the expandable course-part toggle button in the
    # sidebar TOC (the <button x-on:click="toggleExpanded"> inside the
    # x-data="coursePart" block). Use a semantic selector such as the part
    # title text.
    #
    # 1. The child list under the part is hidden initially (or expanded,
    #    depending on persisted localStorage state — start from a known
    #    state by clearing storage or by asserting the initial state and
    #    then toggling twice).
    # 2. Click the toggle. The nested children div (x-show="expanded")
    #    becomes visible — assert a child topic's title is visible.
    # 3. Click again. The nested children div is hidden — assert the child
    #    topic's title is not visible.
    #
    # Prefer Playwright's `expect(locator).to_be_visible()` /
    # `to_be_hidden()` — these auto-wait and don't assert on CSS classes.
```

Keys points for the author:
- Use `page` and `live_server` fixtures; `reverse_url` for the detail URL.
- Select the toggle by the course-part's title text, not by CSS class.
- Assert visibility of a child item's text, not on Alpine's internal `expanded` state or on CSS classes — that keeps the test at the interface level and aligned with the feedback to avoid style assertions.
- Clear localStorage at the start of the test (e.g. `page.goto(...)` then `page.evaluate("localStorage.clear()")` then reload) so the expanded/collapsed starting state is deterministic.

Run with `uv run pytest -m playwright` and confirm the test fails before the fix is applied (the Alpine component isn't registered on detail pages, so clicking the toggle does nothing and the children never become visible).

### Task 2: Add the `extra_alpine_components` block to `_course_base.html`

**Skills:** `fls:template`, `fls:alpine-js`

**File to edit:** `freedom_ls/student_interface/templates/student_interface/_course_base.html`

Add `{% load static %}` at the top (after the `{% extends %}` tag) and the `extra_alpine_components` block, following the same pattern as `course_home.html`:

```django
{% load static %}

{% block extra_alpine_components %}
    <script defer src="{% static 'student_interface/js/alpine-components.js' %}"></script>
{% endblock extra_alpine_components %}
```

Run `uv run pytest -m playwright` and confirm the test from Task 1 now passes.
