# Plan: Split the monolithic plugin into focused plugins under `claude_plugins/`

Implements `1. spec.md`. This plan **only** touches Claude Code plugin authoring artifacts
(commands, skills, agents, hooks, scripts, templates, manifests, configs). It touches **no** Django
application code, no models, no migrations, no runtime behaviour of the FLS product. There is no
pytest suite for this work and no browser surface; verification is by `grep` checklist,
`claude plugin validate`, and a post-cutover plugin-load smoke test (§Validation).

The spec is the design source of truth. This plan turns it into an ordered, commit-by-commit,
grep-verifiable task list. Every task cites the spec section it implements.

---

## 0. Existing code to reuse / not duplicate (DRY)

The whole job is a re-organisation of existing artifacts, so "reuse" here means **move, don't rewrite**
and **overlay, don't duplicate**:

- **Move via `git mv`, never recreate.** Every destination file already exists under
  `fls-claude-plugin/`. Use `git mv` so history follows and diffs stay reviewable. Recount, don't
  re-author, the per-plugin READMEs from disk (§11 — the current README's "13 skills / 16 commands" is
  stale; disk has 17 skills / 32 command files).
- **Overlay skills reference `ds`, never copy its body.** The 10 split skills/resources in spec §7
  keep their generic core in `ds` and leave a *thin* `fls-dev` overlay that points at the `ds` skill
  and adds only the FLS delta. No duplicated prose (Decision 5.1(1)).
- **`agent_memory_guidelines.md`** is the one generic resource deliberately **duplicated** (into `ds`,
  `fls-dev`, and `sdd`) because `${CLAUDE_PLUGIN_ROOT}` is per-plugin and cross-plugin resource reads
  cannot resolve (§6.5). This is an intentional exception to DRY, justified in the spec.
- **`protected/` helpers are read-and-followed by literal path** (via Read), shared across plugins —
  they are the existing cross-plugin reuse mechanism and stay that way; we only rewrite their paths.
- **Existing scripts are reused as-is** (`find_available_port.sh`, `generate_app_map.py`,
  `fetch_pr_comments.sh`, etc.) — they move to their owning plugin unchanged except where the spec
  calls out a parameterization (`install_dev.sh`, `dev_db_delete.sh`).

## 1. Execution-safety protocol (do this FIRST — spec §10.1)

This split is **self-modifying**: the SDD driver tools (`/implement_plan`, `/next`,
`/finish_worktree`, `sdd-mechanic`, `sdd-worker`, hooks, launcher, `.claude/settings.json`) are the
very files being moved/renamed/deleted. In-place migration would move files out from under the running
session. **The migration must be driven from a frozen external copy of the plugin.**

### T1.1 — Snapshot the plugin outside the repo (operator, manual, one-time)
Copy the current plugin to a location **outside every worktree and outside the bare-repo dir**
(`freedom-ls-worktrees/` is the git-common-dir — not there):

```bash
cp -r <this-worktree>/fls-claude-plugin <lms>/fls-plugin-frozen-split
```
The frozen copy keeps `name: fls` and its own `hooks.json`, so it stays consistent with the
still-unmodified `.claude/settings.json` (`enabledPlugins: {fls:true}`, `Skill(fls:*)`).

### T1.2 — Relaunch the driver against the frozen copy with BOTH sentinels set
```bash
FLS_PLUGIN=1 CLAUDE_PLUGINS_LOADED=1 claude --plugin-dir <lms>/fls-plugin-frozen-split
```
Both sentinels set so a restart passes the `SessionStart` check under either the old or new name.
All implementation work below runs in this session.

### T1.3 — Hold the ordering invariant until cutover
Build all four plugins under `claude_plugins/` and rewrite every reference **inside plugin files**
first, but keep these **project-level** artifacts on the **old `fls` / `$FLS_PLUGIN` scheme until the
final cutover commit** (they are what the frozen driver needs for a clean restart):
- repo-root **`claude.sh`**
- **`.claude/settings.json`** (sentinel, `Skill(fls:*)` glob, `enabledPlugins`, `mcp__plugin_fls_playwright__*`, `Bash(.claude/fls/scripts/*.sh:*)` perms)
- the **`.claude/fls/`** config dir

> Rollback path at every step = the frozen copy + the pre-cutover `claude.sh`/`settings.json`.
> Do **not** delete the frozen copy until Validation is green.

  dir, `.gitignore`, and `settings.json` perms at once — do all three in cutover.
- **`fls-content` sync writing to the old path** → fix all 6 literals; test the sync once post-move.
- **Self-modifying driver** → the frozen-copy protocol (Workstream 1) is mandatory, not optional.
