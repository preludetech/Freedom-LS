# Plan: Fix Malformed Allauth Email Confirmation Links

## Overview

Override `send_mail` in our `AccountAdapter` to set `Content-Transfer-Encoding: 8bit` on all email messages before sending. This prevents Python's `email` library from applying quoted-printable encoding, which wraps long URLs at 76 characters and corrupts them.

## Approach

The fix is in a single method override. Allauth's `DefaultAccountAdapter.send_mail()` calls `self.render_mail()` to build an `EmailMultiAlternatives` (or `EmailMessage`), then calls `msg.send()`. We override `send_mail` to intercept between render and send, setting 8bit encoding on both the main body and any HTML alternative.

from email.mime.text import MIMEText

def send_mail(self, template_prefix: str, email: str, context: dict) -> None:
    # Use allauth's render_mail to build the message (reuses allauth's
    # template rendering, from_email, subject formatting, etc.)
    from django.contrib.sites.shortcuts import get_current_site
    from allauth.account import app_settings as allauth_app_settings

    request = ... # get request from allauth context
    ctx = {
        "request": request,
        "email": email,
        "current_site": get_current_site(request),
    }
    ctx.update(context)
    msg = self.render_mail(template_prefix, email, ctx)

    # Set 8bit encoding on all message parts to prevent quoted-printable

    _set_8bit_encoding(msg)

The `_set_8bit_encoding` helper function (module-level in `allauth_account_adapter.py`):

```python
def _set_8bit_encoding(msg):
    """Set Content-Transfer-Encoding to 8bit on an EmailMessage.

    This prevents Python's email library from using quoted-printable encoding,
    which wraps lines at 76 characters and corrupts long URLs.
    """
    # Django's EmailMessage uses a lazy message generation approach.
    # We need to set encoding on the underlying MIMEText parts.
    # Override the encoding property that Django uses when generating the message.
    msg.encoding = msg.encoding  # ensure encoding attr exists

    # Monkey-patch the message method to set 8bit after generation
    original_message = msg.message
    def patched_message():
        mime_msg = original_message()
        for part in mime_msg.walk():
            if part.get_content_type() in ('text/plain', 'text/html'):
                del part['Content-Transfer-Encoding']
                part['Content-Transfer-Encoding'] = '8bit'
        return mime_msg
    msg.message = patched_message
```

Note: The exact implementation may need adjustment based on how Django's `EmailMultiAlternatives` generates MIME messages. The key insight is that we need to modify the MIME message *after* Django builds it but *before* it's sent. Research the Django source during implementation to find the cleanest hook point.
