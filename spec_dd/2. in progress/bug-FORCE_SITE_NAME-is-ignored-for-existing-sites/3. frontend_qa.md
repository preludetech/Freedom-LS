# Frontend QA: FORCE_SITE_NAME bug fix

## Prerequisites

- `FORCE_SITE_NAME=DemoDev` is set in dev settings
- A DemoDev site exists in the database with a user `demodev@email.com`
- The Bloom site is configured with domain `127.0.0.1:8001`

## Test 1: Login works on non-matching port (port 8010)

1. Start the dev server: `uv run python manage.py runserver 8010`
2. Visit `http://127.0.0.1:8010/accounts/login/`
3. Verify the page shows DemoDev branding
4. Log in with `demodev@email.com` and the correct password
5. **Expected:** Login succeeds and you are redirected to the home page

## Test 2: No IntegrityError on duplicate signup (port 8010)

1. With the server still running on port 8010, visit `http://127.0.0.1:8010/accounts/signup/`
2. Attempt to sign up with `demodev@email.com`
3. **Expected:** A form validation error saying the email is already taken — NOT an IntegrityError / 500 error

## Test 3: Forced site overrides domain-matched site (port 8001)

1. Start the dev server: `uv run python manage.py runserver 8001`
2. Visit `http://127.0.0.1:8001/`
3. **Expected:** The page shows **DemoDev** site, NOT Bloom (even though Bloom has domain `127.0.0.1:8001`)
4. Log in with `demodev@email.com` and the correct password
5. **Expected:** Login succeeds
