# In-App Feedback System — Implementation Plan

## Overview

Create a new `feedback` Django app (`freedom_ls/feedback/`) that provides a configurable, site-aware feedback system. The system uses Django signals to trigger feedback prompts at natural pause points, stores responses via the admin-configurable forms, and uses session state + middleware to display modals on both full page loads and HTMX partial responses.

## Task 2: Create the trigger point registry

**Skill:** `fls:testing`

Create `freedom_ls/feedback/registry.py` — a simple module-level registry for valid trigger points.

```python
# Pseudocode
_registry: dict[str, str] = {}  # key: trigger_point name, value: description

def register_trigger_point(name: str, description: str) -> None:
    """Register a trigger point. Called by apps in their AppConfig.ready()."""
    _registry[name] = description

def get_trigger_points() -> dict[str, str]:
    """Return all registered trigger points."""
    return dict(_registry)

def is_valid_trigger_point(name: str) -> bool:
    return name in _registry
```

Register the standard FLS trigger points in `FeedbackConfig.ready()`:
```python
register_trigger_point("course_completed", "Fired when a user completes a course")
register_trigger_point("topic_completed", "Fired when a user completes a topic")
```

**Skill:** `fls:testing`

### Signal (`freedom_ls/feedback/signals.py`)

```python
import django.dispatch

feedback_trigger = django.dispatch.Signal()
# Sent with: sender=<trigger_point_string>, user=<User>, context_object=<model instance>
```

### Receiver (`freedom_ls/feedback/receivers.py`)

Create a signal receiver that connects to `feedback_trigger`.

```python
# Pseudocode for the receiver function:
def handle_feedback_trigger(sender, user, context_object, request, **kwargs):
    trigger_point = sender  # sender is the trigger point string

    # 1. Increment FeedbackTriggerLog
    log, _ = FeedbackTriggerLog.objects.get_or_create(
        user=user, trigger_point=trigger_point, site=current_site
    )
    log.count = F("count") + 1
    log.save(update_fields=["count"])
    log.refresh_from_db()

    # 2. Find active form for this trigger point
    form = FeedbackForm.objects.filter(trigger_point=trigger_point, is_active=True).first()
    if not form:
        return

    # 3. Check eligibility
    #    a. log.count >= form.min_occurrences
    #    b. No existing FeedbackResponse for this user + form + content_object
    #    c. FeedbackDismissal count for this user + form < 3
    #    d. Session flag "feedback_shown_this_session" is not set
    #    e. Cooldown: no FeedbackResponse or FeedbackDismissal for this form + user
    #       within the last cooldown_days

    # 4. If eligible, store in session:
    #    request.session["pending_feedback"] = {
    #        "form_id": str(form.id),
    #        "content_type_id": ContentType.objects.get_for_model(context_object).id,
    #        "object_id": context_object.pk,
    #    }
```

**Important:** The receiver needs access to the `request` object to set session data. The signal must be sent with `request` as a keyword argument.

**Note on site context for `get_or_create`:** Since `SiteAwareManager` filters by site automatically, but `get_or_create` needs to include `site` explicitly in the defaults for creation. Use the `site` from the thread-local request context (`_thread_locals.request`).

**Tests (TDD):**
- Test trigger log increment
- Test eligible form is found
- Test all eligibility checks individually:
  - min_occurrences not met → no session data
  - Already responded for same context object → no session data
  - 3+ dismissals → no session data
  - Session flag already set → no session data
  - Cooldown not elapsed → no session data
- Test session data is set when all checks pass

## Task 6: Create the feedback middleware

**Skill:** `fls:htmx`

Create `freedom_ls/feedback/middleware.py` — `FeedbackMiddleware`.

This middleware checks the session after each response. If `pending_feedback` is in the session and the response is an HTMX partial (has `HX-Request` header), it adds an `HX-Trigger` header to tell the client to show the feedback modal.

```python
# Pseudocode
class FeedbackMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        response = self.get_response(request)

        # Only act on HTMX requests
        if request.headers.get("HX-Request") == "true":
            pending = request.session.get("pending_feedback")
            if pending:
                # Merge with any existing HX-Trigger header
                import json
                existing = response.get("HX-Trigger", "")
                if existing:
                    try:
                        triggers = json.loads(existing)
                    except (json.JSONDecodeError, TypeError):
                        triggers = {existing: None}
                else:
                    triggers = {}
                triggers["show-feedback-modal"] = pending
                response["HX-Trigger"] = json.dumps(triggers)

        return response
```

Add `"freedom_ls.feedback.middleware.FeedbackMiddleware"` to `MIDDLEWARE` in `config/settings_base.py` — after `CurrentSiteMiddleware`.

**Tests (TDD):**
- Test middleware adds HX-Trigger header on HTMX requests when session has pending_feedback
- Test middleware does NOT add header on non-HTMX requests
- Test middleware merges with existing HX-Trigger headers

## Task 7: Create the feedback views

**Skill:** `fls:htmx`, `fls:testing`

Create views in `freedom_ls/feedback/views.py` and wire them up in `freedom_ls/feedback/urls.py`.

### `feedback_form_view` (GET)

URL: `feedback/form/<form_id>/` name=`"feedback_form"`

- Receives form_id, content_type_id, object_id as query params
- Returns the rendered feedback form partial (for HTMX lazy-load)
- Clears `pending_feedback` from session
- Sets `feedback_shown_this_session = True` in session

### `feedback_submit_view` (POST)

URL: `feedback/submit/<form_id>/` name=`"feedback_submit"`

- Validates rating (1-5), saves `FeedbackResponse`
- Sets `feedback_shown_this_session = True` in session
- Returns the thank-you message partial

### `feedback_dismiss_view` (POST)

URL: `feedback/dismiss/<form_id>/` name=`"feedback_dismiss"`

- Creates `FeedbackDismissal` record
- Sets `feedback_shown_this_session = True` in session
- Returns HTTP 204

All views require `@login_required`.

## Task 8: Create the feedback modal template and star rating component

**Skill:** `fls:template`, `fls:alpine-js`, `fls:frontend-styling`, `fls:icon-usage`

### Alpine components

Create `freedom_ls/feedback/static/feedback/js/alpine-components.js` — the feedback app owns its own Alpine components. Load this file via the `{% block extra_alpine_components %}` block in `_base.html`.

**Important:** All Alpine.data() registrations must be wrapped in the `alpine:init` event listener. Add `x-cloak` to any elements hidden by default and use `x-transition` for show/hide animations. If `init()` adds event listeners, add a `destroy()` to clean them up.

```javascript
document.addEventListener("alpine:init", () => {
    Alpine.data("starRating", () => ({
        rating: 0,
        hoverRating: 0,
        setRating(value) {
            this.rating = value;
        },
        setHover(value) {
            this.hoverRating = value;
        },
        clearHover() {
            this.hoverRating = 0;
        },
        starClass(index) {
            return (this.hoverRating || this.rating) >= index ? "text-warning" : "text-muted";
        },
    }));

    Alpine.data("feedbackModal", () => ({
        open: false,
        formId: null,
        contentTypeId: null,
        objectId: null,
        submitted: false,
        _handler: null,
        init() {
            // Listen for HX-Trigger event from HTMX partial responses
            this._handler = (event) => {
                const data = event.detail;
                if (data) {
                    this.formId = data.form_id;
                    this.contentTypeId = data.content_type_id;
                    this.objectId = data.object_id;
                    this.loadForm();
                }
            };
            document.body.addEventListener("show-feedback-modal", this._handler);
        },
        destroy() {
            if (this._handler) {
                document.body.removeEventListener("show-feedback-modal", this._handler);
            }
        },
        loadForm() {
            // Trigger HTMX request to load the form
            this.open = true;
        },
        close() {
            if (!this.submitted) {
                // Fire dismiss request via htmx.ajax()
            }
            this.open = false;
        },
        onEscape() {
            this.close();
        },
        onSubmitted() {
            this.submitted = true;
            setTimeout(() => { this.open = false; }, 2000);
        },
    }));
});
```
