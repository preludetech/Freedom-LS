# Spec: Course page small fixes

Two small standalone tweaks to the course content templates. Bundled together because each is too small to justify its own spec/branch and they touch the same area of the codebase.

## Background

Two minor UX issues exist on the student-facing course pages:

1. The Next/Previous buttons on the topic page place the directional icon on the *wrong* side of the label. Users read direction from icon position, so a "Next" button with the arrow on the left subtly fights the user's mental model. A right-pointing arrow should sit *after* "Next"; a left-pointing arrow should sit *before* "Previous".
2. The course-finish ("Congratulations") page shows two CTAs side-by-side: "Return to Home" and "View Course". Once a learner has finished a course, sending them back into the course pages is rarely the intent — the more useful path is back to their home dashboard. The second button adds noise and dilutes the primary action.

Both changes are presentational, scoped to two templates, and require no model, view, URL, or migration changes.

## Scope

### 1. Topic page — Next / Previous icon position

**File:** `freedom_ls/student_interface/templates/student_interface/course_topic.html`

The page currently uses the existing `c-button` cotton component, which already supports both `icon_left` and `icon_right` attributes. The fix is to switch the navigation buttons to use the correct attribute for direction:

- **Previous** button (line 26–29) — keep the icon on the left (use `icon_left="previous"`). The current markup nests `<c-icon>` as a child instead of using the attribute; align it with the rest of the codebase by using `icon_left`.
- **Next** button (when shown via the `mark_complete` form, line 36) — currently `icon-left="check"`. The "check" icon for a forward-progress action is fine for "Finish Course" but for "Next" it should be a right-side `next` arrow. Split the two cases:
  - When `is_last_item` → keep `icon_left="check"` for the "Finish Course" label.
  - Otherwise → use `icon_right="next"` for the "Next" label.
- **Next** button (when shown as a link, line 45) — currently `icon-left="next"`. Change to `icon_right="next"`.
- **Finish Course** fallback link (line 56) — leave `icon_left="check"` as-is (terminal action, not directional).
- **Home** fallback link (line 50) — leave `icon_left="home"` as-is (not directional).

Use the existing `next` and `previous` icon names already registered in the project's icon component. Do not introduce new icon names.

The `course_form_page.html` "Next"/"Back" buttons are *out of scope* — the idea explicitly references only the topic page URL. Leave that template untouched.

### 2. Finish page — remove "View Course" button

**File:** `freedom_ls/student_interface/templates/student_interface/course_finish.html`

Delete the second `<c-button>` block (the one linking to `student_interface:course_home` with the `topic` icon and label "View Course"). Keep the "Return to Home" button as the single primary CTA. The wrapping `flex justify-center gap-4` stays — it still centres the remaining button correctly.

## Out of scope

- `course_form_page.html` Back/Next buttons.
- Any styling, copy, or layout changes beyond the two items above.
- Adding new icons or modifying the `c-button` cotton component.
- Changes to views, URLs, models, or migrations.

## Success criteria

1. On `/courses/<slug>/<topic-num>/`, the **Previous** button renders with a left-pointing arrow to the *left* of the word "Previous".
2. On the same page, the **Next** button (whether submit or link) renders with a right-pointing arrow to the *right* of the word "Next".
3. The "Finish Course" submit button still uses the `check` icon on the left.
4. On `/courses/<slug>/finish/`, only the **Return to Home** button is visible — the **View Course** button is gone.
5. Layout remains visually clean (no broken alignment, no orphaned spacing) at desktop, tablet, and mobile widths.
6. No regressions in any other template — only the two files above are modified.
7. Existing tests still pass; no new tests required (presentational change, no behaviour change).

## Decisions and rationale

- **Use the existing `icon_left` / `icon_right` attributes on `c-button` rather than nesting `<c-icon>` manually.** The component already supports both sides natively, keeps spacing consistent (`mr-2` / `ml-2`), and matches the style used elsewhere in the same file. Manual nesting (as the current Previous button does) is an inconsistency worth fixing while we're in there.
- **Keep the `check` icon for "Finish Course".** It signals completion, not direction, so a left-side check mark is appropriate. Only the "Next" case needs the directional right-arrow.
- **Remove "View Course" rather than demote it.** The idea is explicit ("remove the View Course button"). A learner who *does* want to revisit the course can navigate from the home dashboard; we don't need a secondary CTA on the celebration page.
- **Leave `course_form_page.html` alone.** The idea only mentions the topic page URL. Form-page Back/Next is a separate UX surface (form navigation vs. course navigation) and out of scope for this small bundle.
