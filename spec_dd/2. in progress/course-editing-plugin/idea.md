We need to allow people who are making courses to use claude code to help them structure things correctly. We'll do this using a plugin.

The plugin should:
- explain the different types of files and how to use them (eg content files, form pages, etc)
- explain how to organise those files, numbering conventions
- explain how the different widgets are used (usage syntax and what they are for, simple best practices)
- explain that UUIDs should never be touched

The projects using this plugin will not have access to this source code.

We will need to use the skills in this plugin to take nasty markdown content and turn it into something we can use.

If we change course authoring functionality at all (eg adding new widgets, frontmatter, content types, etc) then we'll need to update the plugin to match
