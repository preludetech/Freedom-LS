Currently we have an icon component that fetches icons by name from a data structure defined in base/icons. It relies on hero icons.

We need to set things up so it is easy to override this behavior. Different applications that build on top of freedom ls will have different visual styles and will therefore have different icons.

We have 2 options:

1. Include a few different open-source icon sets and have a setting that sets which icon set we are using. Or
2. Make it easy to override completely.

I'm leaning towards 1.

It could be good to install one package that includes a lot of differnt icon libraries internally.
