# Session Timeout Strategy

## Problem

FLS currently uses Django's default `SESSION_COOKIE_AGE` of 2 weeks. This is reasonable for general use, but sensitive actions (changing password, changing email, accessing admin) should not rely solely on a session that could be hours old.

## Options to explore

1. **Re-authentication for sensitive actions** -- Keep 2-week sessions but require the user to re-enter their password before performing sensitive actions. django-allauth already has a `reauthenticate` rate limit setting, which suggests it has built-in support for this flow.

2. **Shorter session timeout with "remember me"** -- Reduce the default session length (e.g. to 1 hour) and offer a "remember me" checkbox at login that extends it.

3. **Idle timeout vs absolute timeout** -- An idle timeout expires the session after a period of inactivity, while an absolute timeout expires it after a fixed duration regardless of activity. These can be combined.

## Initial thinking

Option 1 is likely the best fit: it keeps the user experience simple for everyday use while adding a security gate where it matters. Investigate django-allauth's re-authentication support before designing anything custom.
