# Educator Interface IDOR Fixes

Found during the security audit. These are concrete IDOR (Insecure Direct Object Reference) vulnerabilities that need fixing independently of the broader educator permissions spec.

## Bugs

1. **`ListViewConfig.__class_getitem__` lacks permission checks** - Objects are fetched by PK without verifying the requesting user has access. Any authenticated user can access any object by guessing/enumerating PKs (e.g. `/educator/cohorts/{any_uuid}/`).

2. **`CourseDataTable` missing guardian permission filtering** - Unlike the other educator data tables, `CourseDataTable` does not filter by object-level permissions via django-guardian. Any authenticated educator can see all courses across the system.
