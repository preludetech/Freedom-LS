# Non-Mobile Bug Fixes

## Goal

Fix non-mobile bugs discovered during QA reports that are unrelated to mobile responsiveness.

## Bugs

### 1. TOC shows original deadline instead of student's override

**Source:** Deadlines QA Report

When a student has a `StudentCohortDeadlineOverride` that extends a course-level deadline (e.g., from Mar 15 to Mar 20), the TOC still displays the original cohort deadline date rather than the overridden date. This is misleading for students who have been given an extension.

**Severity:** Medium


### 4. Admin content type dropdown too verbose

**Source:** Deadlines QA Report

The `content_type` dropdown in deadline admin inlines shows ALL content types in the system instead of only relevant ones (Topic, Form, Activity, CoursePart, Course).

**Severity:** Low (admin UX)

### 5. Password change signs user out

**Source:** Email Templates QA Report

After changing a password, the user is signed out and redirected to the homepage. May be intentional for security — needs verification and either documentation or a fix.

**Severity:** Low

## Verification Items

### 6. CoursePart deadline display never tested

Test course lacked CourseParts so this was never verified. Needs a test course with CourseParts to confirm deadline display works correctly in the TOC.

### 7. Deadline override priority with incomplete items

Only partially verified because all affected items were already completed. Needs testing with an incomplete item governed only by a course-level deadline (with no item-level deadline) to confirm override behavior.

## Approach

- Backend/logic bugs (#1, #4, #5): Write a failing unit test first (TDD), then fix
- Frontend bugs (#2, #3): Verify with Playwright MCP, then fix
- Verification items (#6, #7): Set up test data and verify behavior; fix if broken
