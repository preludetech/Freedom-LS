Django premailer is causing errors in production:

```
2026-04-06T15:46:18.854053707Z /usr/local/lib/python3.13/site-packages/premailer/premailer.py:105: SyntaxWarning: invalid escape sequence '\s'
```

## Root Cause

The warning is in the **premailer** library (v3.0.0, from 2016), not django-premailer itself. Line 105 uses `re.compile('\s*!important')` — a non-raw string with `\s`, which Python 3.12+ flags as a SyntaxWarning (and will become a SyntaxError in future Python).

We're stuck on premailer 3.0.0 even though **3.10.0** is available and maintained. django-premailer (the Django template tag wrapper) is unmaintained since 2016 but doesn't pin the premailer version.

## Proposed Fix

**Upgrade premailer to 3.10.0** — this should fix the warning since newer versions use raw strings for regex patterns.

- django-premailer's usage of premailer is via the stable public API: `Premailer(html, **kwargs).transform()` — very unlikely to break
- Dry-run confirms clean upgrade: only adds `cachetools` as new dependency

## Follow-up Consideration

django-premailer itself is unmaintained (last release 2016, officially supports Django 1.6-1.9). It's only 38 lines of code — a single template tag. If it causes further issues, we should replace it with our own template tag and remove the django-premailer dependency entirely. But that's not needed for this fix.
