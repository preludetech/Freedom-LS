# Exam Feature for CAA-Approved Courses

## Overview

Build an exam system into FLS that supports both official SACAA-administered exams (at accredited examination centres) and internal ATO course assessments (mock exams, progress checks, course completion exams). The system must be capable of meeting CAA regulatory requirements while being flexible enough for general exam use.

CAA regulations reference: https://caa.mylexisnexis.co.za/#

## Core Capabilities

### 1. Exam Definition (built on existing course forms system)

- Exams are specified using the existing course forms system, extended with exam-specific settings
- Exam-specific settings include: time limits, pass percentage (CAA standard is 75%), whether manual grading is required, retake policy configuration
- Support for question randomisation (shuffle question order and/or answer option order per attempt)
- To support randomisation without breaking context: FormPage gets a content field that is displayed alongside every question on that page. This way questions can be freely shuffled while their associated context (e.g. an image, a scenario description) always appears with them
- Support for question pools (draw N questions from a larger bank per attempt) - useful for integrity and CAA-style exam variety, and for mock exams
- One-question-at-a-time display mode (important for exam integrity)

### 2. Exam-Taking Experience

- Countdown timer (visible, colour-coded warnings at thresholds, auto-submit on expiry)
- Question navigation panel showing answered/unanswered/flagged status
- Flag/mark-for-review functionality
- Auto-save on every question navigation + periodic saves
- Explicit submission with confirmation dialog showing unanswered/flagged question count
- Post-submission confirmation with receipt number and timestamp

### 3. Retake Rules Engine

- Educators can define retake policies per exam with configurable conditions:
  - Maximum number of attempts (or unlimited)
  - Time-based delays (e.g. can retake after N days/weeks/months)
  - Score-based rules (e.g. if mark < 40% then wait 3 months, else wait 2 weeks)
  - Consecutive failure escalation (e.g. 3 consecutive failures = 6 month wait)
  - Educator approval required before retake
- CAA-compliant retake rules as a preset configuration:
  - PPL-level: 7 calendar days after failure (no escalation)
  - CPL/ATPL-level: 7 days for 1st/2nd failure, 2 months for 3rd+ failure or score below 50%
- Student-facing: clear display of attempt history, scores, when next attempt is available (with countdown), how many attempts remain
- Educator-facing: ability to grant additional attempts or override waiting periods per student

### 4. Online Proctoring

- Online proctoring for remote exam-taking
- Approach: Start with lightweight integrity measures, with the architecture to support heavier proctoring later
- Lightweight measures: question/answer randomisation, time limits, one-at-a-time display, IP logging
- Browser-based integrity (no extensions needed):
  - Detect and log tab/window focus changes (visibilitychange + blur/focus events), warn the student and record occurrences
  - Record all browser integrity events (tab switches, copy attempts, right-click attempts, focus loss) with timestamps. Batch-send to backend alongside auto-saves. Provides an event timeline per attempt for examiner review and misconduct investigations
- Configurable answer reveal: for mock/practice exams, correct answers can be shown after submission to aid learning. For formal exams (especially CAA), correct answers should never be revealed to protect the question bank from being memorised and shared
- Note: screenshots and screen recording cannot be blocked from the browser — that requires a native lockdown browser app, which is a future consideration
- Future: webcam monitoring, lockdown browser integration, AI-based behaviour flagging

### 5. Exam Scheduling and Access Control

- Exam availability windows: exams open at a configured start time and close at a configured end time
- Concurrent session prevention: ensure a student cannot open the same exam attempt in multiple tabs or devices simultaneously
- Identity verification: require a selfie/photo at exam start that is stored with the attempt record for later verification by an invigilator
- IP/device fingerprinting: log IP and device info per attempt, flag if a student's IP or device changes significantly during an attempt

### 6. Network Resilience

- Auto-save ensures answers are persisted frequently
- On connection loss: display a clear warning to the student, continue allowing them to answer questions locally
- On reconnection: batch-sync locally cached answers to the server
- Grace period: if a student disconnects near the end of the timer, allow a configurable grace period (e.g. 2 minutes) for reconnection before auto-submitting what was saved
- Examiner visibility: flag attempts that experienced connectivity issues so examiners can make informed decisions

### 7. Examiner Live Dashboard

- Real-time view of active exam sessions: who is currently taking the exam, time remaining, number of questions answered
- Live integrity event feed: surface tab switches, focus loss, and other flagged events as they happen
- Ability to flag or terminate an attempt from the dashboard (e.g. in cases of obvious misconduct)

### 8. Audit Trail and Record-Keeping (CAA Compliance)

- CAA requires detailed student records retained for minimum 5 years
- Log all exam events: start time, answer submissions, navigation, submission time, score, grader actions
- Exam results history per student with full attempt details

## What This Is NOT

- This is not a replacement for the SACAA central question bank - ATOs using FLS for accredited centre exams would still source questions from SACAA
- This is not a full AI proctoring system initially - start with lightweight integrity and build up
- This does not handle exam booking/scheduling with SACAA - that remains a separate administrative process

## Key CAA Regulatory Constraints to Keep in Mind

- 75% pass mark for all CAA theoretical knowledge exams
- 18-month window to complete all exam subjects (credits expire)
- 36-month window to complete skills test after passing all subjects
- Exams in English only
- MCQ format with 3-4 choices, no negative marking
- Strict prohibited items rules for in-person exams
- Misconduct can result in 36-month disqualification
- Student records must be retained for minimum 5 years
