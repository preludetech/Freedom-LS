When moving around the educator interface, it's easy to lose track of where we are. The three improvements below work together to give the user clear orientation at all times.

## Breadcrumbs

Add hierarchy-based breadcrumbs at the top of the main content area showing the user's position in the site structure.

Example: `Educator > Cohorts > Data Science 2026`

Key considerations:
- Hierarchy-based (not path/history-based) - reflects site structure, not the user's click path
- All items except the current page are clickable links
- Accessible: use `<nav aria-label="Breadcrumb">` with `<ol>` and `aria-current="page"` on the last item. CSS-only separators (not text) to avoid screen reader noise
- Update breadcrumbs via `hx-swap-oob` when HTMX swaps main content, so they stay in sync without a full page reload
- Keep to 2-4 levels max
- Must appear consistently on every educator interface page

## Left panel navigation highlight

Highlight the active link in the left panel so the user can see which section they're in at a glance.

Key considerations:
- Use `aria-current="page"` as the source of truth, styled with a CSS attribute selector
- Visual treatment: background fill + left accent bar + bold/semi-bold text (combine 2-3 of these)
- Hover state should be visually distinct from (and subtler than) the active state
- Server controls which item is active via OOB swap when content changes

## Left panel navigation dropdowns

When the user navigates to a specific instance (eg a specific cohort), expand that section in the left panel to show the selected item. Use expand-in-place (not replace-menu drill-down) since our hierarchy is shallow and educators frequently switch between sibling items.

Example:
```
- Cohorts v
    - Data Science 2026   <-- currently selected
- Users
- Courses
```

Key considerations:
- Use the W3C disclosure pattern: `<button aria-expanded="true|false">` controlling a nested `<ul>`, animated with Alpine.js `x-collapse`
- Server determines which section is expanded and which item is active (via OOB swap); Alpine.js handles the expand/collapse animation
- Keyboard support: Tab to navigate, Enter/Space to toggle, Escape to close
- Only show the currently selected instance under the section heading (not a full list of all instances)
