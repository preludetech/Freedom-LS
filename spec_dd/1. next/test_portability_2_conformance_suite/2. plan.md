## Layer 3 — `freedom_ls.contrib.conformance` package

New package `freedom_ls/contrib/conformance/`. Auto-shipped by the existing
`include = ["freedom_ls*"]` glob. Probe submodules are `test_*`-named so FLS's own
run collects them under `config.settings_dev` (success criterion #2).

### T3.1 — Package scaffolding + override-friendliness core

Create:

- `freedom_ls/contrib/__init__.py` — empty.
- `freedom_ls/contrib/conformance/_registry.py` — the drop() mechanism and
  install-gating helpers (the one non-`test_*` module; defines **no `test_*`
  callables**).

`_registry.py` pseudocode:

```python
from __future__ import annotations
from django.conf import settings

# Downstream-mutable set of internal-tier probe ids to skip when a project keeps
# an app but customises one of its routes.
_DROPPED: set[str] = set()

def drop(*probe_ids: str) -> None:
    """Prune internal-tier namespace probes a downstream has customised.

    Call from the downstream's conftest/test module before the run. Contract-tier
    ids cannot be pruned; the probe body enforces that.
    """
    _DROPPED.update(probe_ids)

def _is_dropped(probe_id: str) -> bool:
    return probe_id in _DROPPED

def _app_installed(app_label_dotted: str) -> bool:
    # e.g. "freedom_ls.student_interface"
    return app_label_dotted in settings.INSTALLED_APPS
```

**Note for the real code (both here and every probe module below):** the shipped
code MUST carry full type hints on every function (no `Any`, `X | None` syntax)
per CLAUDE.md, and MUST NOT contain any spec-citation comments (no `D9`, `§`,
"criterion #7"). The pseudocode's decision-letter references are for the plan
reader only.

- Runtime membership check (not collection-time): probes stay **collected** (node
  IDs appear in `--collect-only`, criterion #1) but **skip** at run when their app
  is absent or an internal id is dropped — satisfying criterion #6 ("skipped, not
  failed") regardless of import ordering.

### T3.2 — Namespace + reference URL probes (`test_urls.py`)

`freedom_ls/contrib/conformance/test_urls.py`. Data tables + two probes.

```python
from __future__ import annotations
import pytest
from dataclasses import dataclass, field
from django.conf import settings
from django.urls import reverse, NoReverseMatch
from ._registry import _app_installed, _is_dropped

@dataclass(frozen=True)
class _Probe:
    app: str          # dotted INSTALLED_APPS label, e.g. "freedom_ls.student_interface"
    viewname: str     # e.g. "student_interface:dashboard"
    contract: bool    # True = hard-fail while app installed; False = prunable
    kwargs: dict[str, str] = field(default_factory=dict)   # placeholder reverse kwargs

# viewname ids MUST match the acceptance node IDs.
FLS_NAMESPACE_PROBES: list[_Probe] = [
    _Probe("freedom_ls.student_interface", "student_interface:dashboard", True),
    _Probe("freedom_ls.student_interface", "student_interface:course_detail", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:course_home", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:initiate_course_access", True, {"course_slug": "x"}),
    _Probe("freedom_ls.student_interface", "student_interface:courses", False),
    _Probe("freedom_ls.course_applications", "course_applications:apply", True, {"course_slug": "x"}),
    _Probe("freedom_ls.course_applications", "course_applications:status", True, {"pk": "00000000-0000-0000-0000-000000000000"}),
    _Probe("freedom_ls.course_interest", "course_interest:express_interest", True, {"course_slug": "x"}),
    _Probe("freedom_ls.educator_interface", "educator_interface:interface", True, {"path_string": ""}),
    _Probe("freedom_ls.accounts", "accounts:account_profile", True),
    _Probe("freedom_ls.accounts", "accounts:legal_doc", True, {"doc_type": "terms"}),
]

REFERENCE_URL_NAMES: list[str] = ["sitemap", "robots_txt"]

@pytest.mark.parametrize(
    "probe", FLS_NAMESPACE_PROBES,
    ids=[p.viewname for p in FLS_NAMESPACE_PROBES],   # -> "student_interface:dashboard" etc.
)
def test_fls_namespace_reverses(probe: _Probe) -> None:
    if not _app_installed(probe.app):
        pytest.skip(f"{probe.app} not installed")
    if not probe.contract and _is_dropped(probe.viewname):
        pytest.skip(f"{probe.viewname} dropped by downstream")
    reverse(probe.viewname, kwargs=probe.kwargs or None)   # NoReverseMatch => fail

@pytest.mark.parametrize("name", REFERENCE_URL_NAMES, ids=REFERENCE_URL_NAMES)
def test_reference_url_reverses(name: str) -> None:
    reverse(name)   # required (D1); NoReverseMatch => hard fail
```

Notes:
- Parametrize `ids` produce exactly the acceptance node IDs.
- `educator_interface:interface` reverses with `path_string=""` against the
  catch-all `re_path`.
- The probe asserts *resolvability*, never that a target object exists — no DB.

### T3.5 — Migration-state probe (`test_migrations.py`)

`freedom_ls/contrib/conformance/test_migrations.py`.

```python
def test_migration_state_consistent() -> None:
    from django.core.management import call_command
    # --check --dry-run diffs models against migration files and writes NOTHING
    # to disk. Raises SystemExit if a migration is missing.
    call_command("makemigrations", check=True, dry_run=True, verbosity=0)
```

- `call_command(..., check=True)` raises `SystemExit` on model/migration drift;
  pytest reports that as a failure. `--dry-run` guarantees **no migration file is
  written**. Needs **no `django_db` marker and no test-DB records**.
- **Accuracy note:** `makemigrations` internally calls
  `MigrationLoader.check_consistent_history()`, which *may open* a DB connection to
  read `django_migrations`; Django guards that call and swallows the
  `OperationalError` if the DB is unreachable, so the probe stays green with or
  without a live DB. The probe does **not** strictly "never touch a socket", it
  tolerates the DB being absent. T3.8 adds a meta-test asserting the probe still
  passes when the DB is unreachable, so this tolerance is proven, not assumed.

### T3.8 — Conformance probe meta-tests (teeth; TDD-first)

`freedom_ls/contrib/conformance/tests/test_conformance_meta.py` (a real `tests/`
dir → inherits the existing ruff/bandit relaxation). **Also create
`freedom_ls/contrib/conformance/tests/__init__.py`** (empty). These are the
**failing-first** tests written before the probes. All use FLS settings, so they
are `fls_internal`-safe.

The probes themselves contain intentional `if …: pytest.skip(...)` gates in their
bodies (runtime skip, required by criterion #6's "skipped not failed" design). This
is a deliberate, documented exception to `fls:testing`'s "no conditionals in test
bodies" guidance — the meta-tests below prove the gates behave correctly.

Cover:
- `test_fls_namespace_reverses` **fails** when a bad table entry is injected — call
  the probe directly with a `_Probe` whose `viewname` does not exist, assert raises.
- **Skip-when-absent (criterion #6a):** with an app removed from `INSTALLED_APPS`
  (via `@override_settings`), the probe for that app raises `pytest.skip.Exception`.
- **Drop-internal (criterion #6b):** `conformance.drop("student_interface:courses")`
  → the internal probe skips, while a contract-tier probe with a (simulated) broken
  include still fails. Reset `_DROPPED` in a fixture teardown.
- `test_configured_backend_instantiates` raises when
  `@override_settings(COURSE_ACCESS_BACKEND="does.not.Exist")` +
  `get_course_access_backend.cache_clear()` (autouse fixture already clears).
- `test_migration_state_consistent` passes on the clean tree (positive), and
  (optionally) a monkeypatched `call_command` raising `SystemExit` surfaces as
  failure.
- `test_migration_state_consistent` still **passes when the DB is unreachable** —
  proving the `--check --dry-run` tolerance claimed in T3.5, not assuming it.
- `test_active_theme_resolves` **fails** when `FLS_THEMES_DIRS` is overridden to
  dirs that don't contain the active theme.

## Task order (TDD)

1. **T3.1** conformance scaffolding + `_registry.py` (drop/gating core).
2. **T3.8** write the conformance meta-tests **first** (failing) — proves teeth.
3. **T3.2–T3.6** implement probes + `__init__` re-export until meta-tests + FLS's

   T3.5 probe asserts.
