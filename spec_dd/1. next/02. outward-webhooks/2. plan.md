# Outward Webhooks: Implementation Plan

## Overview

Create a new `webhooks` Django app (`freedom_ls/webhooks/`) that provides outbound webhook delivery for platform events. The system uses custom Django signals for decoupling, `django-tasks` for background delivery, HMAC-SHA256 signing per the Standard Webhooks spec, and exponential backoff with circuit breaker resilience.

**Key dependencies to add:** `django-tasks[django_tasks.backends.database]`, `httpx`

```python
class WebhookDeliveryAttempt(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    endpoint = models.ForeignKey(WebhookEndpoint, on_delete=models.CASCADE, related_name="delivery_attempts")
    event = models.ForeignKey(WebhookEvent, on_delete=models.CASCADE, related_name="delivery_attempts")

    # Delivery status
    status = models.CharField(max_length=20, choices=[
        ("pending", "Pending"),
        ("success", "Success"),
        ("failed", "Failed"),           # Transient failure, will retry
        ("permanent_failure", "Permanent Failure"),  # 4xx, won't retry
        ("dead_letter", "Dead Letter"),  # Exhausted all retries
    ], default="pending")

    # Response details
    status_code = models.PositiveIntegerField(null=True, blank=True)
    response_body = models.TextField(blank=True, default="")  # First 1024 chars
    latency_ms = models.PositiveIntegerField(null=True, blank=True)

    # Retry tracking
    attempt_number = models.PositiveIntegerField(default=1)
    next_retry_at = models.DateTimeField(null=True, blank=True)
    max_retries = models.PositiveIntegerField(default=5)

    created_at = models.DateTimeField(auto_now_add=True)
```

- Add `@property def is_retriable(self) -> bool` — checks status is "failed" and attempt_number < max_retries

```python
from django.contrib.sites.models import Site
from django.db import transaction

def fire_webhook_event(event_type: str, payload: dict, site: Site | None = None) -> None:
    """
    Fire a webhook event. Called by other apps to trigger webhooks.
    Wraps in transaction.on_commit to ensure data is committed before delivery.
    If site is None, uses the current site from thread locals.
    """
    def _dispatch() -> None:
        from freedom_ls.webhooks.signals import webhook_event
        webhook_event.send(
            sender=None,
            event_type=event_type,
            payload=payload,
            site=site or _get_current_site(),
        )
    transaction.on_commit(_dispatch)
```

Write tests:

- Test that signal is NOT sent if transaction rolls back
- Test that current site is used when site param is None

### Task 5.1: HTTP delivery function

In `delivery.py`:

```python
def deliver_webhook(delivery_attempt_id: uuid.UUID) -> None:
    """
    Execute a single webhook delivery attempt.
    - Build payload envelope: {id, type, timestamp, data}
    - Sign with endpoint secret
    - POST to endpoint URL with standard headers
    - Record response status, body (truncated to 1024 chars), latency
    - Classify response and update delivery attempt status
    - Update endpoint circuit breaker state
    """
```

Use `httpx` for HTTP POST with:
- Timeout: 15 seconds (connect: 5s, read: 15s)
- User-Agent: `FreedomLS-Webhooks/1.0`
- Content-Type: `application/json`

Response classification:
- 2xx → status="success", reset endpoint consecutive_failures to 0
- 410 Gone → status="permanent_failure", deactivate endpoint (set is_active=False)
- 4xx (except 429) → status="permanent_failure"
- 429 → status="failed", respect Retry-After header for next_retry_at
- 5xx / timeout / connection error → status="failed", schedule retry

Write tests using `httpx` mock/transport or `respx` library:
- Test successful delivery (2xx)
- Test permanent failure (4xx)
- Test retriable failure (5xx)
- Test timeout handling
