We need to keep track of everything that users on the platform do.

We will base our approach on Experience API  (also called xapi).

There is a good chance that we'll generate a lot of data, so we will need to carefully consider what database to use. Is postgres sufficient? Or should we use something else?

Ideally we wont add a lot of infrastructural complexity. 

Create all functionality in an xapi app.
We'll need quick and easy helper functions that we can use for tracking user actions. 

These actions will typically happen in the backend, but we will also likely need some js functions to track certain activities that dont result in API requests

We will likely need to maintain our own store of event types and suchlike instead of relying fully on existing xapi repositories
