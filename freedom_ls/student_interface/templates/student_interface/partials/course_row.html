{% comment %} The single all-courses list row. Renders any course state from one template, branching on course.listing_status (CourseListingStatus StrEnum) so there is no per-status leaf template. Horizontal sibling of course_card.html — same field contract and shared partials, different shell/layout. Standard fields land in standard places: icon + colour -> accent panel (accent_slot_key, icon) status -> eyebrow slot (course_status_eyebrow.html — the single place status is ever shown; no duplicate chip/badge) details -> course_details_link.html, right-aligned inline with the status in the eyebrow slot (top-right of the row) title + link -> stretched title link (card_title_link.html) progress -> footer slot (), registered/in_progress only Title-link target by status: complete -> course_finish registered / in_progress -> view_course_item index=1 coming_soon / not_registered -> course_detail Anonymous, not-registered visitors see the access badge (Free / By application) in place of the registration eyebrow. Removing the old "Coming soon" chip (status is now only in the eyebrow) also frees the horizontal space that clipped "Details" off narrow viewports.
