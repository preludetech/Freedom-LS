Create a GitHub issue for this repository.

1. Gather context: understand what the issue is about from the user's request and any relevant code
2. Determine a clear, concise title
3. Draft the issue body with:
   - Description of the problem or feature request
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior (for bugs)
   - Any relevant code references
4. Ask the user to confirm the title and body before creating
5. Create the issue: `gh issue create --title "<title>" --body "<body>"`
6. Report the issue URL back to the user
